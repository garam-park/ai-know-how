# 수동 검증: 페이지네이션 및 검색 최적화 동작 확인

## 개요
새롭게 추가된 `Pagination` 컴포넌트와 `useDebounce` 기반 검색 최적화가 목록 페이지에서 정상적으로 동작하는지 개발자 관점(브라우저)에서 확인합니다.

## 검증 대상 페이지
- 프로젝트 목록 페이지 (`ProjectListPage`)
- 회사 목록 페이지 (`CompanyListPage`)

## 검증 시나리오
1. **페이지 이동 검증**
   - 백엔드에 더미 데이터가 충분히 있어 페이지네이션 버튼(1, 2, 다음 등)이 표시되는지 확인합니다.
   - 각 페이지 번호를 클릭 시 즉각적으로 해당 페이지의 데이터가 노출되는지 확인합니다.
   
2. **페이지네이션 예외 검증**
   - 현재 페이지가 1일 때 `이전` 버튼이 비활성화(disabled/클릭불가) 되어 있는지 확인합니다.
   - 현재 페이지가 마지막 페이지일 때 `다음` 버튼이 비활성화되는지 확인합니다.

3. **Debounce 자동 검색 검증**
   - 회사 목록 페이지에서 검색어 입력창에 키워드를 빠르게 여러 번 타이핑해 봅니다.
   - 매 타이핑마다 API 핑(ping)이 발생하지 않고, 타이핑 후 `0.3초(300ms)` 가량 지난 시점에 단 1번의 통합 API 요청이 발생하여 목록이 필터링되는지 네트워크 탭에서 확인합니다.

4. **1페이지 초기화 검증**
   - 페이지네이션에서 임의의 페이지(ex: 3페이지)로 이동합니다.
   - 그 상태에서 새로운 검색어를 입력합니다.
   - 검색 직후 현재 페이지 상태가 다시 `1페이지`로 초기화되는지 점검합니다.
