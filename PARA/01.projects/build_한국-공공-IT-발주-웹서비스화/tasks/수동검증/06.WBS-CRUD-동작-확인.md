# WBS CRUD 로직 및 트리 렌더링 검증 가이드

## 1. WBS 루트 카테고리 구성 (`WbsTab.tsx`)
1. **경로**: 프로젝트 상세 페이지의 `WBS 및 공정` 탭
2. **트리 초기화 여부**:
   - `WBS가 비어 있습니다`라는 EmptyState가 노출되는지 확인
   - 아이콘, 문구, `루트 카테고리 추가` 액션 버튼 정렬 여부
3. **카테고리 생성 버튼 (루트 레벨)**:
   - 우측 상단 `+ 루트 카테고리 추가` 버튼 클릭
   - 최상위에 회색 Inline 양식이 나타나는지 확인
   - `카테고리` 옵션 선택, 제목을 `분석설계`로 기입하고 저장 클릭 -> 정상 저장과 함께 화면에 트리가 노출되는지 점검

## 2. 하위 태스크 추가 및 인라인 갱신
1. **노드간 계층 구조 확인**:
   - 추가된 `분석설계` 항목 우측 `+` 아이콘을 클릭하여 하위 인라인 입력기를 연다.
   - 드롭다운을 `태스크(TASK)`로 변경하고, "요구사항 명세서 작성", 가중치(Weight) `3` 입력 후 추가 여부 확인
2. **이름/가중치 인라인 편집 테스트 (Update)**:
   - 방금 추가한 `요구사항 명세서 작성` 텍스트 항목을 **더블클릭**하여 텍스트 상자로 전환되는지 점검
   - 커서를 올려 `요구사항 명세서 초안 작성`으로 고치고 엔터 (또는 다른 영역 포커스 아웃)
   - 성공 Toast가 뜨고 화면이 자동 갱신되는지 확인
3. **태스크 진척률 편집 테스트 (Progress)**:
   - 태스크 행 우측 영역에 **진척률 바 혹은 `0%` 텍스트** 클릭
   - 텍스트가 Input 박스로 전환되었는지 점검
   - `50`을 입력하고 Enter 입력 시 다시 텍스트 `50%`로 돌아가고, 상위 노드(카테고리)의 진척률이 백엔드 로직에 의해 자동 재계산되어 표시되는지 확인
   - 퍼센테이지 바(Bar UI)가 진척도에 비례해 파란색(100%일 경우 초록색 등) 영역을 채우며 직관적으로 변경되는지 점검

## 3. 삭제 (Delete)
1. **단일 태스크 삭제**:
   - 태스크 노드 우측 끝 `x` 마크 클릭 -> "삭제하시겠습니까?" 경고 및 삭제 동작 확인
2. **카테고리 (하위 노드 포함) 폭포수 삭제**:
   - 내부에 태스크나 자식 카테고리를 가진 상태에서 최상단 상위 노드의 `x` 클릭
   - "하위 항목도 모두 삭제됩니다"라는 경고 메시지가 나오는지 필히 확인
   - 수락 시 트리 내부에서 하위 노드들이 모두 정상적으로 지워지는지 점검
