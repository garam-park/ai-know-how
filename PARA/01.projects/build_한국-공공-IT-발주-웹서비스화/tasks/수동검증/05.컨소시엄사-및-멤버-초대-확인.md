# 컨소시엄사 구조 적용 및 멤버 초대 수동 검증 가이드

## 1. 컨소시엄 트리 관리 검증 (`CompaniesTab.tsx` / `AddConsortiumForm.tsx`)
1. **경로**: `/projects/:id` 에 들어간 후, `컨소시엄 및 멤버` -> 하위 **컨소시엄사 탭** 이동
2. **"참여사 추가" UI 확인**:
   - `AddConsortiumForm`이 정상적으로 위에 렌더링 되어있는지 점검
   - **회사명** 드롭다운에서 등록된 전체 회사 리스트가 잘 나오는지 확인 (단, 이미 프로젝트에 추가된 주관사나 타 참여사는 보이지 않음에 유의)
3. **추가 및 삭제 동작**:
   - 상위사(부모 노드), 역할(PARTNER/SUB) 선택 뒤 "추가" 버튼 클릭
   - 즉각적으로 하단 CompanyTree에서 계층(Depth)이 적용되어 그려지는지 확인 ("참여사", "하청" 배지 색상 확인)
   - 프로젝트 최초 주관사(PRIME) 배지를 가진 행에는 삭제(x) 버튼이 없는지 확인
   - 새롭게 추가한 노드의 우측에 위치한 "삭제" 버튼 클릭 -> 경고 Alert 수락 시 정상 삭제되는지, 자식 노드가 있다면 함께 CASCADE 제거됨 안내가 표출되는지 테스트

## 2. 멤버 초대 UI 및 플로우 검증 (`MembersTab.tsx` / `InviteMemberForm.tsx`)
1. **경로**: 같은 페이지 내 `멤버 관리` 탭
2. **"멤버 초대" Toggle 동작**:
   - 우측 상단 `+ 멤버 초대` 클릭 시 토글 형태로 폼이 아래로 열리고, 다시 취소 누르면 닫히는지 점검
3. **초대 폼 유효성(Validation)**:
   - **소속 컨소시엄사** 드롭다운에 방금 컨소시엄 탭에서 추가한 (또는 주관사인) 회사들이 정상적으로 Flat(1차원 배열)하게 모두 노출되는지 확인
   - 이메일, 역할군, 초기 투입률 등을 입력하고 "초대 발송"
   - 회사나 역할을 지정하지 않았을 때는 버튼이 `disabled` 되어있는지 점검
4. **멤버 관리 인라인 리스트**:
   - 멤버가 성공적으로 추가된 뒤 탭 하단 표기 리스트에 추가되는가
   - 각 멤버의 **역할군 열**에서 `<select>` 태그를 통해 다른 역할(Developer -> QA 등)로 변경 시 성공 Toast가 울리는지 점검
   - **투입률 열**의 `<input>` 에 커서를 두고 숫자를 바꾼 뒤, 칸 밖을 클릭(onBlur) 했을 때 갱신 Toast 확인
   - **제거 동작**: 우측 끝 제거 링크 클릭 -> 정말 프로젝트에서 배제할지 묻는 Confirm 노출 및 수락 시 즉각 리스트에서 사라지는지 점검
