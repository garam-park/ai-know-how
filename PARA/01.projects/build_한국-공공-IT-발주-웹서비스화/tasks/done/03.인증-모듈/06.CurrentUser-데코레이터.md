# CurrentUser 데코레이터

## 예상 소요시간
| 구분 | 시간 |
|------|------|
| 사람 (숙련 개발자) | ~20분 |
| AI 에이전트 | ~8분 |

## What (무엇을)
JWT 토큰에서 사용자 정보를 추출하는 @CurrentUser() 커스텀 파라미터 데코레이터와 JwtAuthGuard를 구현합니다. 이를 통해 모든 인증 필요 API에서 현재 로그인 사용자 정보를 쉽게 가져올 수 있습니다.

## Why (왜)
매 API마다 Request에서 토큰을 추출하고 검증하는 반복적인 코드를 제거하고, 선언적으로 @UseGuards(JwtAuthGuard) @CurrentUser() user: User 형태로 사용자를 받을 수 있도록 합니다.

## 수행 가이드

### Step 1: JwtStrategy 구현
- 대상 파일: `apps/backend/src/modules/auth/strategies/jwt.strategy.ts`
- 작업 내용: Passport JWT 전략 구현
  ```typescript
  import { Injectable } from '@nestjs/common';
  import { PassportStrategy } from '@nestjs/passport';
  import { ExtractJwt, Strategy } from 'passport-jwt';
  import { ConfigService } from '@nestjs/config';
  import { PrismaService } from '../../prisma/prisma.service';
  import { JwtPayload } from '../types/jwt-payload.type';

  @Injectable()
  export class JwtStrategy extends PassportStrategy(Strategy) {
    constructor(
      configService: ConfigService,
      private prisma: PrismaService,
    ) {
      super({
        jwtFromRequest: ExtractJwt.fromAuthHeaderAsBearerToken(),
        ignoreExpiration: false,
        secretOrKey: configService.get<string>('JWT_SECRET') || 'dev-secret-key',
      });
    }

    async validate(payload: JwtPayload) {
      // JWT 검증 후 사용자 정보 조회
      const user = await this.prisma.user.findUnique({
        where: { id: payload.sub },
      });

      if (!user) {
        throw new Error('User not found');
      }

      return user; // request.user에 할당됨
    }
  }
  ```

### Step 3: @CurrentUser 데코레이터 구현
- 대상 파일: `apps/backend/src/common/decorators/current-user.decorator.ts`
- 작업 내용: 현재 사용자를 주입하는 파라미터 데코레이터
  ```typescript
  import { createParamDecorator, ExecutionContext } from '@nestjs/common';

  export const CurrentUser = createParamDecorator(
    (data: unknown, ctx: ExecutionContext) => {
      const request = ctx.switchToHttp().getRequest();
      return request.user;
    },
  );
  ```

### Step 4: AuthModule에 JwtStrategy 제공자 추가
- 대상 파일: `apps/backend/src/modules/auth/auth.module.ts`
- 작업 내용: JwtStrategy를 provider로 등록
  ```typescript
  import { Module } from '@nestjs/common';
  import { JwtModule } from '@nestjs/jwt';
  import { PassportModule } from '@nestjs/passport';
  import { ConfigModule, ConfigService } from '@nestjs/config';
  import { AuthController } from './auth.controller';
  import { AuthService } from './auth.service';
  import { JwtStrategy } from './strategies/jwt.strategy';
  import { PrismaModule } from '../prisma/prisma.module';

  @Module({
    imports: [
      PassportModule,
      JwtModule.registerAsync({
        imports: [ConfigModule],
        inject: [ConfigService],
        useFactory: (configService: ConfigService) => ({
          secret: configService.get<string>('JWT_SECRET') || 'dev-secret-key',
          signOptions: {
            expiresIn: configService.get<string>('JWT_EXPIRATION') || '15m',
          },
        }),
      }),
      PrismaModule,
    ],
    controllers: [AuthController],
    providers: [AuthService, JwtStrategy],
    exports: [AuthService, JwtModule],
  })
  export class AuthModule {}
  ```

### Step 5: @CurrentUser 데코레이터 테스트용 엔드포인트 추가
- 대상 파일: `apps/backend/src/modules/auth/auth.controller.ts`
- 작업 내용: 현재 사용자 정보를 반환하는 GET /me 엔드포인트 추가
  ```typescript
  import { Controller, Post, Get, Body, UseGuards } from '@nestjs/common';
  import { AuthService } from './auth.service';
  import { JwtAuthGuard } from '../../common/guards/jwt-auth.guard';
  import { CurrentUser } from '../../common/decorators/current-user.decorator';
  import { CreateUserDto } from './dto/create-user.dto';
  import { LoginDto } from './dto/login.dto';
  import { RefreshDto } from './dto/refresh.dto';

  @Controller('api/v1/auth')
  export class AuthController {
    constructor(private readonly authService: AuthService) {}

    @Post('register')
    async register(@Body() createUserDto: CreateUserDto) {
      const user = await this.authService.register(createUserDto);
      return {
        code: 201000,
        result: { user },
      };
    }

    @Post('login')
    async login(@Body() loginDto: LoginDto) {
      const { accessToken, refreshToken, user } = await this.authService.login(loginDto);
      return {
        code: 200000,
        result: { accessToken, refreshToken, user },
      };
    }

    @Post('refresh')
    async refresh(@Body() refreshDto: RefreshDto) {
      const { accessToken, refreshToken } = await this.authService.refresh(
        refreshDto.refreshToken,
      );
      return {
        code: 200000,
        result: { accessToken, refreshToken },
      };
    }

    @Post('logout')
    async logout() {
      const result = await this.authService.logout('');
      return {
        code: 200000,
        result,
      };
    }

    @UseGuards(JwtAuthGuard)
    @Get('me')
    async getCurrentUser(@CurrentUser() user: any) {
      return {
        code: 200000,
        result: {
          user: {
            id: user.id,
            email: user.email,
            name: user.name,
            tel: user.tel,
            createdAt: user.createdAt,
          },
        },
      };
    }
  }
  ```

### Step 6: API 테스트
- 작업 내용: @CurrentUser 데코레이터 동작 확인

  **Step 1: 로그인하여 accessToken 획득**
  ```bash
  curl -X POST http://localhost:3000/api/v1/auth/login \
    -H "Content-Type: application/json" \
    -d '{
      "email": "test@example.com",
      "password": "password123"
    }'
  ```

  응답에서 accessToken을 복사합니다.

  **Step 2: GET /me 요청 (인증 필요)**
  ```bash
  curl -X GET http://localhost:3000/api/v1/auth/me \
    -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
  ```

  정상 응답:
  ```json
  {
    "code": 200000,
    "result": {
      "user": {
        "id": "uuid...",
        "email": "test@example.com",
        "name": "테스트 사용자",
        "tel": null,
        "createdAt": "2024-01-15T10:30:00Z"
      }
    }
  }
  ```

  **Step 3: Authorization 헤더 없이 요청 (실패)**
  ```bash
  curl -X GET http://localhost:3000/api/v1/auth/me
  ```

  에러 응답:
  ```json
  {
    "statusCode": 401,
    "message": "Unauthorized"
  }
  ```

  **Step 4: 잘못된 토큰으로 요청 (실패)**
  ```bash
  curl -X GET http://localhost:3000/api/v1/auth/me \
    -H "Authorization: Bearer invalid.token"
  ```

  에러 응답:
  ```json
  {
    "statusCode": 401,
    "message": "Unauthorized"
  }
  ```

## 참고
- JwtAuthGuard는 Passport의 'jwt' 전략을 기반으로 동작합니다.
- @CurrentUser() 데코레이터는 request.user에 할당된 사용자 정보를 반환합니다.
- JwtStrategy의 validate() 메서드에서 반환한 값이 request.user에 할당됩니다.
- 모든 인증 필요 API에 @UseGuards(JwtAuthGuard)와 @CurrentUser() 조합으로 사용합니다.
- 추후 권한 체크를 위해 PermissionGuard도 함께 사용하게 됩니다.
