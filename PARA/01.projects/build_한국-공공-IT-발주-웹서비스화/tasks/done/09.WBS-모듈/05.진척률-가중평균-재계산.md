# 진척률 가중평균 재계산

## 예상 소요시간
| 구분 | 시간 |
|------|------|
| 사람 (숙련 개발자) | ~60분 |
| AI 에이전트 | ~20분 |

## What (무엇을)
PATCH /api/v1/projects/:id/wbs-nodes/:nodeId/progress 엔드포인트를 구현하고, TASK 노드의 진척률 변경 시 상위 CATEGORY 노드의 진척률을 자동으로 가중평균으로 재계산합니다.

## Why (왜)
이것은 이 프로젝트의 핵심 비즈니스 로직입니다. WBS의 전체 진행률의 정확성을 보장하기 위해, 개별 TASK의 진척률 변경이 즉시 상위 노드의 진척률에 반영되어야 합니다. 각 노드의 weight(가중치) 값을 고려하여 공정한 진척률 계산을 수행합니다.

## 수행 가이드

### Step 1: UpdateProgressDto 정의
`apps/backend/src/modules/wbs/dto/update-progress.dto.ts`:
```typescript
import { IsNumber, Min, Max } from 'class-validator';
import { Type } from 'class-transformer';

export class UpdateProgressDto {
  @IsNumber()
  @Min(0)
  @Max(100)
  @Type(() => Number)
  progress: number;
}
```

### Step 2: WbsService에 진척률 관련 로직 추가
`apps/backend/src/modules/wbs/wbs.service.ts`:
```typescript
async updateProgress(
  nodeId: number,
  updateDto: UpdateProgressDto,
): Promise<{ node: any; affectedNodes: any[] }> {
  // TASK만 직접 진척률 입력 가능
  const node = await this.prisma.wbsNode.findUnique({
    where: { id: nodeId },
  });

  if (!node) {
    throw new NotFoundException('Node not found');
  }

  if (node.type !== 'TASK') {
    throw new BadRequestException('CATEGORY 노드의 진척률은 자동 계산됩니다');
  }

  // TASK 진척률 업데이트
  const updatedNode = await this.prisma.wbsNode.update({
    where: { id: nodeId },
    data: { progress: updateDto.progress },
    include: {
      assignee: true,
      projectCompany: true,
    },
  });

  // 상위 노드 진척률 재계산
  const affectedNodes: any[] = [];
  if (node.parentId) {
    const affected = await this.recalculateProgress(node.parentId);
    affectedNodes.push(...affected);
  }

  return {
    node: updatedNode,
    affectedNodes,
  };
}

async recalculateProgress(nodeId: number): Promise<any[]> {
  const node = await this.prisma.wbsNode.findUnique({
    where: { id: nodeId },
  });

  if (!node || !node.parentId) {
    return [];
  }

  // 현재 노드의 부모를 찾음
  const parent = await this.prisma.wbsNode.findUnique({
    where: { id: node.parentId },
  });

  if (!parent) {
    return [];
  }

  // 부모의 자식 노드들 조회 (삭제된 노드 제외)
  const siblings = await this.prisma.wbsNode.findMany({
    where: {
      parentId: parent.id,
      deletedAt: null,
    },
  });

  if (siblings.length === 0) {
    return [];
  }

  // 가중평균 계산
  // 공식: Σ(progress × weight) / Σ(weight)
  const totalWeight = siblings.reduce((sum, sibling) => sum + sibling.weight, 0);
  const weightedSum = siblings.reduce(
    (sum, sibling) => sum + sibling.progress * sibling.weight,
    0,
  );

  const parentProgress = totalWeight > 0 
    ? Math.round(weightedSum / totalWeight) 
    : 0;

  // 부모 노드의 진척률 업데이트
  const updatedParent = await this.prisma.wbsNode.update({
    where: { id: parent.id },
    data: { progress: parentProgress },
    include: {
      assignee: true,
      projectCompany: true,
    },
  });

  const affectedNodes = [
    {
      id: parent.id,
      progress: parentProgress,
    },
  ];

  // 재귀: 부모의 부모로 올라감
  if (parent.parentId) {
    const upperAffected = await this.recalculateProgress(parent.id);
    affectedNodes.push(...upperAffected);
  }

  return affectedNodes;
}
```

### Step 3: WbsController에 진척률 업데이트 엔드포인트 추가
`apps/backend/src/modules/wbs/wbs.controller.ts`:
```typescript
@Patch(':nodeId/progress')
@RequirePermission(Resource.WBS, Action.EDIT)
async updateProgress(
  @Param('id') projectId: string,
  @Param('nodeId') nodeId: string,
  @Body() updateDto: UpdateProgressDto,
) {
  const result = await this.wbsService.updateProgress(
    parseInt(nodeId),
    updateDto,
  );

  return {
    code: 200000,
    result,
  };
}
```

### Step 4: 진척률 계산 예시

**예시 상황:**
```
CATEGORY A (weight=1)
├── TASK 1 (weight=1, progress=50)
├── TASK 2 (weight=2, progress=100)
└── TASK 3 (weight=1, progress=0)
```

**TASK 1의 progress를 50에서 80으로 변경하면:**
- totalWeight = 1 + 2 + 1 = 4
- weightedSum = (80 × 1) + (100 × 2) + (0 × 1) = 80 + 200 + 0 = 280
- CATEGORY A progress = Math.round(280 / 4) = 70

## 완료 기준
- [ ] UpdateProgressDto 정의 및 검증 완료
- [ ] TASK 노드의 진척률 직접 입력 가능
- [ ] CATEGORY 노드는 직접 입력 불가 (자동 계산만)
- [ ] 진척률 변경 시 상위 CATEGORY 가중평균 자동 재계산
- [ ] 재귀적으로 root까지 올라가며 재계산 수행
- [ ] weight 값이 다를 때 정확한 가중평균 계산
- [ ] weight 전부 0인 경우 처리 (progress = 0)
- [ ] affectedNodes 배열 반환 (프론트엔드 실시간 업데이트용)
- [ ] 200000 응답 반환

## 참고
- 진척률: 0~100 범위
- 가중평균 공식: Σ(progress × weight) / Σ(weight)
- Math.round() 사용하여 정수로 반올림
- Soft delete된 노드(deletedAt != null) 제외
