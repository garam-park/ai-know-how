# 환경변수 (.env) 설정

## 예상 소요시간
| 구분 | 시간 |
|------|------|
| 사람 (숙련 개발자) | ~10분 |
| AI 에이전트 | ~5분 |

## What (무엇을)
프로젝트 루트에 환경변수 파일(`.env`)과 템플릿 파일(`.env.example`)을 생성하여 데이터베이스 접속 정보, JWT 시크릿, API 포트 등 민감한 설정을 코드와 분리합니다. NestJS `ConfigModule`을 설정하여 모든 환경변수를 중앙에서 관리합니다.

## Why (왜)
환경변수를 코드에 하드코딩하면 보안 위험이 높고, 배포 환경(개발/스테이징/프로덕션)에 따라 설정을 변경하기 어렵습니다. `.env` 파일로 관리하면 각 환경별 설정을 쉽게 관리할 수 있고, 민감 정보를 버전 관리 시스템에서 제외할 수 있습니다.

## 수행 가이드

### Step 1: .gitignore 확인 및 업데이트
- 대상 파일: `project-root/.gitignore`
- 다음 항목 추가 (이미 있으면 확인):

```
# Environment variables
.env
.env.local
.env.*.local

# Dependencies
node_modules

# Build output
dist
build
.tsbuildinfo

# IDE
.vscode
.idea
.DS_Store

# Logs
*.log
npm-debug.log*
pnpm-debug.log*
```

### Step 2: .env.example 파일 생성 (템플릿)
- 대상 파일: `project-root/.env.example`
- 작업 내용:

```env
# Database Configuration
# 형식: postgresql://[username]:[password]@[host]:[port]/[database]
DATABASE_URL="postgresql://user:password@localhost:5432/korean_public_it?schema=public"

# JWT Configuration
JWT_SECRET="your-secret-key-change-in-production"
JWT_EXPIRES_IN="1h"
JWT_REFRESH_EXPIRES_IN="7d"

# Server Configuration
PORT=3000
NODE_ENV=development

# Frontend Configuration
VITE_API_BASE_URL="http://localhost:3000/api"
```

**설명:**
- `DATABASE_URL` — PostgreSQL 접속 정보
- `JWT_SECRET` — JWT 토큰 서명에 사용되는 비밀키 (최소 32자 이상 권장)
- `JWT_EXPIRES_IN` — 액세스 토큰 만료 시간 (기본값: 1시간)
- `JWT_REFRESH_EXPIRES_IN` — 리프레시 토큰 만료 시간 (기본값: 7일)
- `PORT` — NestJS 서버 포트 (기본값: 3000)
- `NODE_ENV` — 실행 환경 (development, production 등)
- `VITE_API_BASE_URL` — 프론트엔드에서 사용할 백엔드 API 기본 URL

### Step 3: .env 파일 생성 (실제 사용)
- 대상 파일: `project-root/.env`
- 작업 내용 (개발 환경용):

```env
# Database Configuration
DATABASE_URL="postgresql://postgres:postgres@localhost:5432/korean_public_it?schema=public"

# JWT Configuration
JWT_SECRET="dev-secret-key-change-in-production-12345678"
JWT_EXPIRES_IN="1h"
JWT_REFRESH_EXPIRES_IN="7d"

# Server Configuration
PORT=3000
NODE_ENV=development

# Frontend Configuration
VITE_API_BASE_URL="http://localhost:3000/api"
```

**주의:** 이 파일은 `.gitignore`에 포함되므로 Git에 커밋되지 않습니다.

### Step 4: NestJS ConfigModule 설치
```bash
cd project-root/apps/backend
pnpm add @nestjs/config
```

### Step 5: AppModule에 ConfigModule 설정 확인
- 대상 파일: `project-root/apps/backend/src/app.module.ts`
- 확인 사항:

```typescript
import { Module } from '@nestjs/common';
import { ConfigModule } from '@nestjs/config';
import { PrismaModule } from './prisma/prisma.module';
import { AppController } from './app.controller';
import { AppService } from './app.service';

@Module({
  imports: [
    ConfigModule.forRoot({
      isGlobal: true,
      envFilePath: '../../.env', // 루트의 .env 파일을 읽음
    }),
    PrismaModule,
  ],
  controllers: [AppController],
  providers: [AppService],
})
export class AppModule {}
```

**설명:**
- `isGlobal: true` — 모든 모듈에서 ConfigModule 사용 가능
- `envFilePath: '../../.env'` — 루트 디렉토리의 .env 파일을 읽음

### Step 6: 환경변수 사용 예시
예를 들어, 데이터베이스 URL을 사용하는 서비스 파일:

- 대상 파일: `project-root/apps/backend/src/config/database.config.ts` (새로 생성)

```typescript
import { Injectable } from '@nestjs/common';
import { ConfigService } from '@nestjs/config';

@Injectable()
export class DatabaseConfig {
  constructor(private configService: ConfigService) {}

  getDatabaseUrl(): string {
    return this.configService.get<string>('DATABASE_URL');
  }

  getJwtSecret(): string {
    return this.configService.get<string>('JWT_SECRET');
  }

  getJwtExpires(): string {
    return this.configService.get<string>('JWT_EXPIRES_IN', '1h');
  }

  getPort(): number {
    return this.configService.get<number>('PORT', 3000);
  }

  getNodeEnv(): string {
    return this.configService.get<string>('NODE_ENV', 'development');
  }
}
```

### Step 7: 환경변수 로드 확인
개발 서버 시작:

```bash
cd project-root/apps/backend
pnpm run dev
```

콘솔에서 다음과 같은 환경정보가 로깅되는지 확인:

```
Backend server listening on http://localhost:3000
```

에러가 없으면 환경변수 로딩 성공.

### Step 8: 프론트엔드 환경변수 설정 (선택)
- 대상 파일: `project-root/apps/frontend/.env`

```env
VITE_API_BASE_URL=http://localhost:3000/api
```

Vite에서 환경변수는 `import.meta.env`로 접근:

```typescript
// src/config/api.ts
export const API_BASE_URL = import.meta.env.VITE_API_BASE_URL || 'http://localhost:3000/api';
```

## 참고
- 프로덕션 배포 시 `.env` 파일을 환경에 맞게 수정해야 함
- `JWT_SECRET`은 매우 중요하므로 프로덕션에서는 강력한 비밀키 사용 (최소 32자)
- 여러 환경을 관리하려면 `.env.development`, `.env.production` 등을 따로 만들 수 있음
- Docker를 사용하면 런타임에 환경변수 주입 가능
- 민감 정보(비밀키, DB 비밀번호)는 절대 버전 관리에 포함되지 않도록 주의
