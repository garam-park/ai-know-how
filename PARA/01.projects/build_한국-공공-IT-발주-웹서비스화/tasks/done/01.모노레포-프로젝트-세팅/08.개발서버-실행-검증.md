## 수행 가이드

### Step 1: 루트 디렉토리 확인
```bash
cd project-root
ls -la
```

다음 항목 확인:
- `package.json` (루트)
- `pnpm-workspace.yaml`
- `packages/shared/` (공유 패키지)
- `apps/backend/` (NestJS 백엔드)
- `apps/frontend/` (React 프론트엔드)
- `.env` (환경변수)
- `.eslintrc.json`, `.prettierrc` (Lint/Format 설정)

### Step 2: 모든 의존성 설치 확인
```bash
cd project-root
pnpm install
```

모든 패키지 node_modules가 정상적으로 설치되는지 확인.

### Step 3: 공유 패키지 빌드 확인
```bash
cd project-root/packages/shared
pnpm run build
```

에러 없이 완료되고 `dist/` 디렉토리가 생성되는지 확인.

### Step 4: 백엔드만 실행 테스트
새 터미널 탭에서:

```bash
cd project-root/apps/backend
pnpm run dev
```

콘솔에 다음과 같은 메시지 확인:
```
[Nest] [timestamp] [NestFactory] [PackageVersion]
Backend server listening on http://localhost:3000
```

브라우저에서 `http://localhost:3000/api/health` 접속:
```json
{
  "status": "ok",
  "timestamp": "2026-04-01T..."
}
```

정상 응답 확인 후 서버 종료 (Ctrl+C).

### Step 5: 공유 패키지 Import 테스트 (백엔드)
백엔드의 AppService 또는 임시 파일에서 공유 패키지 타입을 Import:

- 대상 파일: `project-root/apps/backend/src/app.service.ts` (테스트 코드 추가)

```typescript
import { Injectable } from '@nestjs/common';
import { Resource, Action, Scope } from '@shared/types';

@Injectable()
export class AppService {
  getHealth() {
    // 공유 패키지 타입 사용 테스트
    const testResource: Resource = Resource.PROJECT;
    const testAction: Action = Action.VIEW;
    const testScope: Scope = Scope.OWN_COMPANY;

    return {
      status: 'ok',
      timestamp: new Date().toISOString(),
      testTypes: {
        resource: testResource,
        action: testAction,
        scope: testScope,
      },
    };
  }
}
```

다시 `pnpm run dev` 실행하여 TypeScript 컴파일 에러가 없는지 확인.

### Step 6: 프론트엔드만 실행 테스트
새 터미널 탭에서:

```bash
cd project-root/apps/frontend
pnpm run dev
```

콘솔에 다음과 같은 메시지 확인:
```
VITE v5.0.0 ready in 1000 ms

➜  Local:   http://localhost:5173/
```

브라우저에서 `http://localhost:5173` 접속하여 React 애플리케이션이 로드되는지 확인:
- 헤더: "한국 공공 IT 발주 웹서비스"
- 네비게이션: 홈, 프로젝트, 회사 링크
- 홈 페이지 내용 표시

정상 확인 후 서버 종료 (Ctrl+C).

### Step 7: 공유 패키지 Import 테스트 (프론트엔드)
프론트엔드의 App.tsx에서 공유 패키지 타입 Import 확인:

- 대상 파일: `project-root/apps/frontend/src/App.tsx` (이미 포함됨)

```typescript
import { Resource } from '@shared/types';
```

다시 `pnpm run dev` 실행하여 TypeScript 컴파일 에러가 없는지 확인.

### Step 8: PostgreSQL 연결 테스트
백엔드 서버 실행 중 PostgreSQL이 정상 연결되는지 확인:

```bash
cd project-root/apps/backend
pnpm run dev
```

콘솔에서 Prisma 로그나 에러 메시지 확인. 에러가 없으면 DB 연결 정상.

Prisma Studio로 데이터베이스 확인 (선택):

```bash

npx prisma studio
```

브라우저에서 `http://localhost:5555` 열려서 데이터베이스 테이블 확인 가능.

### Step 9: 동시 실행 (병렬 개발)
루트에서 모든 개발 서버를 동시에 실행:

```bash
cd project-root
pnpm dev
```

콘솔에서 다음 메시지 확인:
```

frontend: VITE v5.0.0 ready in ... 

➜  Local:   http://localhost:5173/
```

두 서버가 동시에 구동되는지 확인.

### Step 10: 브라우저 테스트
1. `http://localhost:5173` (프론트엔드) 접속
2. `http://localhost:3000/api/health` (백엔드) 접속
3. 네비게이션 링크 클릭하여 라우팅 정상 작동 확인


## 문제 해결 가이드

### 포트 충돌
- 백엔드 포트 3000 충돌: `.env` 파일의 `PORT` 변수 변경
- 프론트엔드 포트 5173 충돌: `apps/frontend/vite.config.ts`의 `server.port` 변수 변경

### 공유 패키지 Import 실패
- `packages/shared/package.json`에 `"name": "@shared/types"` 확인
- `tsconfig.json`의 `paths` 설정 확인:
  ```json
  "paths": {
    "@shared/*": ["../../packages/shared/src/*"]
  }
  ```

### PostgreSQL 연결 실패
- `.env`의 `DATABASE_URL` 확인
- PostgreSQL 서버 실행 확인
- 데이터베이스명 확인 (기본값: `korean_public_it`)

### Prisma 마이그레이션 실패
- `apps/backend` 디렉토리에서 `npx prisma validate` 실행
- `npx prisma migrate dev --name init` 다시 실행

### 의존성 설치 실패
- `pnpm store prune` 실행하여 캐시 정리
- `rm -rf node_modules && pnpm install` 재설치
