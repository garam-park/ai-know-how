# 중복투입 감지 API

## 예상 소요시간
| 구분 | 시간 |
|------|------|
| 사람 (숙련 개발자) | ~40분 |
| AI 에이전트 | ~15분 |

## What (무엇을)
GET `/api/v1/projects/:projectId/members/overlap` 엔드포인트를 구현합니다. 동일 인물이 여러 프로젝트에 투입되어 있을 때, 기간 겹침 여부를 확인하고 투입비율 합계가 100%를 초과하는 멤버를 감지합니다.

## Why (왜)
인력 자원은 한정되어 있습니다. 같은 사람이 여러 프로젝트에서 기간이 겹쳐 동시 투입될 때, 실제 가용성 초과를 경고합니다. 프로젝트 계획 단계에서 리소스 충돌을 조기에 발견할 수 있습니다.

### Step 2: ProjectMemberService에 기간 겹침 판별 헬퍼 메서드 구현
- 대상 파일: `apps/backend/src/modules/project-member/project-member.service.ts`
- 작업 내용:
  ```typescript
  private isDateRangeOverlap(
    start1: Date,
    end1: Date,
    start2: Date,
    end2: Date,
  ): boolean {
    // 범위 겹침: start1 < end2 AND end1 > start2
    return start1 < end2 && end1 > start2;
  }
  ```

### Step 3: ProjectMemberService에 findOverlappingMembers 메서드 구현
- 대상 파일: `apps/backend/src/modules/project-member/project-member.service.ts`
- 작업 내용:
  ```typescript
  async findOverlappingMembers(
    projectId: string,
  ): Promise<{
    overlaps: MemberOverlapDto[];
    totalCount: number;
  }> {
    // 대상 프로젝트 조회
    const project = await this.prisma.project.findFirst({
      where: { id: projectId, deletedAt: null },
    });

    if (!project) {
      throw new NotFoundException({
        code: 404000,
        message: '프로젝트를 찾을 수 없습니다.',
      });
    }

    // 해당 프로젝트의 모든 멤버
    const projectMembers = await this.prisma.projectMember.findMany({
      where: { projectId, deletedAt: null },
      include: { user: true },
    });

    const overlaps: MemberOverlapDto[] = [];
    const processedUserIds = new Set<string>();

    for (const member of projectMembers) {
      const userId = member.userId;

      // 이미 처리한 사용자는 건너뛰기
      if (processedUserIds.has(userId)) {
        continue;
      }

      // 해당 사용자의 모든 프로젝트 멤버 조회
      const userProjects = await this.prisma.projectMember.findMany({
        where: { userId, deletedAt: null },
        include: { project: true },
      });

      // 기간 겹침 확인
      const overlappingProjects: {
        project: { id: string; name: string };
        inputRate: number;
        startDate: Date;
        endDate: Date;
      }[] = [];
      let totalInputRate = 0;

      for (let i = 0; i < userProjects.length; i++) {
        for (let j = i + 1; j < userProjects.length; j++) {
          const p1 = userProjects[i].project;
          const p2 = userProjects[j].project;

          if (this.isDateRangeOverlap(p1.startDate, p1.endDate, p2.startDate, p2.endDate)) {
            // 겹치는 범위에서 투입비율 합산
            const overlap1InputRate = userProjects[i].inputRate;
            const overlap2InputRate = userProjects[j].inputRate;

            // 중복되는 기간에서의 총 투입비율
            if (overlap1InputRate + overlap2InputRate > 100) {
              // 중복되는 기간 동안의 프로젝트들만 포함
              if (!overlappingProjects.some((p) => p.project.id === p1.id)) {
                overlappingProjects.push({
                  project: { id: p1.id, name: p1.name },
                  inputRate: overlap1InputRate,
                  startDate: p1.startDate,
                  endDate: p1.endDate,
                });
              }
              if (!overlappingProjects.some((p) => p.project.id === p2.id)) {
                overlappingProjects.push({
                  project: { id: p2.id, name: p2.name },
                  inputRate: overlap2InputRate,
                  startDate: p2.startDate,
                  endDate: p2.endDate,
                });
              }
            }
          }
        }
      }

      // 중복이 있는 경우만 결과에 추가
      if (overlappingProjects.length > 0) {
        totalInputRate = overlappingProjects.reduce(
          (sum, p) => sum + p.inputRate,
          0,
        );

        overlaps.push({
          user: {
            id: member.user.id,
            email: member.user.email,
            name: member.user.name,
          },
          projects: overlappingProjects,
          totalInputRate,
        });
      }

      processedUserIds.add(userId);
    }

    return {
      overlaps,
      totalCount: overlaps.length,
    };
  }
  ```

### Step 4: ProjectMemberController에 GET /overlap 엔드포인트 추가
- 대상 파일: `apps/backend/src/modules/project-member/project-member.controller.ts`
- 작업 내용:
  ```typescript
  @Get('overlap')
  @UseGuards(JwtAuthGuard, PermissionGuard)
  @RequirePermission('MEMBER', 'VIEW')
  async findOverlappingMembers(
    @Param('projectId') projectId: string,
  ): Promise<{
    code: number;
    result: {
      overlaps: MemberOverlapDto[];
      totalCount: number;
    };
  }> {
    const result = await this.projectMemberService.findOverlappingMembers(
      projectId,
    );
    return {
      code: 200000,
      result,
    };
  }
  ```

## 완료 기준
- [ ] GET `/api/v1/projects/:projectId/members/overlap` 엔드포인트 동작 확인
- [ ] 동일 인물의 모든 프로젝트 멤버십 조회
- [ ] 기간 겹침 판별 정상 동작 (start1 < end2 AND end1 > start2)
- [ ] 겹치는 기간의 투입비율 합계 계산
- [ ] 100% 초과 멤버만 결과에 포함
- [ ] 응답 형식: `{code:200000, result:{overlaps: MemberOverlapDto[], totalCount}}`
- [ ] 중복투입 없으면 빈 배열 반환 (totalCount: 0)

## 참고
- isDateRangeOverlap: 두 기간의 겹침 판별 (수학적 범위 교집합)
- totalInputRate: 겹치는 기간 동안 누적 투입비율 (100% 초과 경고)
- 중복 가능성이 있는 멤버들의 프로젝트를 모두 나열하여 시각화 지원
- 시간 복잡도: O(m² × n) (m: 프로젝트 내 멤버 수, n: 사용자 전체 프로젝트 수)
