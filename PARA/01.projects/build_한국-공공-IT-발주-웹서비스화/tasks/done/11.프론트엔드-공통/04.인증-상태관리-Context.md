# 인증 상태관리 (Context)

## 예상 소요시간
| 구분 | 시간 |
|------|------|
| 사람 (숙련 개발자) | ~30분 |
| AI 에이전트 | ~10분 |

## What (무엇을)
React Context를 이용하여 로그인 상태, 사용자 정보, 토큰을 관리합니다. AuthProvider를 통해 앱 전역에서 useAuth 훅으로 인증 상태에 접근하고, 초기 로드 시 refresh 토큰으로 자동 로그인을 시도합니다.

## Why (왜)
앱 전역에서 인증 상태와 사용자 정보에 접근하기 위한 기반입니다. 페이지 새로고침 후에도 저장된 refresh 토큰으로 자동 로그인을 시도하여 UX를 개선합니다.

## 수행 가이드

### Step 1: AuthContext 정의
`apps/frontend/src/contexts/AuthContext.tsx` 파일을 생성합니다:

```typescript
import React, { createContext, useContext, ReactNode, useEffect, useState } from 'react';
import { UserDto } from '../types/api';
import { authService } from '../services/authService';
import { tokenStore } from '../services/tokenStore';

interface AuthContextType {
  user: UserDto | null;
  isAuthenticated: boolean;
  isLoading: boolean;
  error: string | null;
  login: (email: string, password: string) => Promise<void>;
  register: (email: string, password: string, name: string, tel?: string) => Promise<void>;
  logout: () => Promise<void>;
  refreshToken: () => Promise<void>;
}

const AuthContext = createContext<AuthContextType | undefined>(undefined);

interface AuthProviderProps {
  children: ReactNode;
}

export const AuthProvider: React.FC<AuthProviderProps> = ({ children }) => {
  const [user, setUser] = useState<UserDto | null>(null);
  const [isLoading, setIsLoading] = useState(true);
  const [error, setError] = useState<string | null>(null);

  // 초기 로드: 저장된 refresh 토큰으로 자동 로그인 시도
  useEffect(() => {
    const initializeAuth = async () => {
      try {
        setIsLoading(true);
        const refreshToken = tokenStore.getRefreshToken();

        if (refreshToken) {
          // Refresh 토큰으로 새 액세스 토큰 획득
          const response = await authService.refresh(refreshToken);
          tokenStore.setAccessToken(response.accessToken);
          tokenStore.setRefreshToken(response.refreshToken);

          // 사용자 정보 조회 (선택)
          // const userData = await userService.getMe();
          // setUser(userData);
        }
      } catch (err) {
        console.error('초기 인증 실패:', err);
        setError(err instanceof Error ? err.message : '인증 초기화 실패');
        tokenStore.clear();
      } finally {
        setIsLoading(false);
      }
    };

    initializeAuth();
  }, []);

  const login = async (email: string, password: string) => {
    try {
      setIsLoading(true);
      setError(null);

      const response = await authService.login({ email, password });
      setUser(response.user);
    } catch (err) {
      const errorMessage = err instanceof Error ? err.message : '로그인 실패';
      setError(errorMessage);
      throw err;
    } finally {
      setIsLoading(false);
    }
  };

  const register = async (
    email: string,
    password: string,
    name: string,
    tel?: string
  ) => {
    try {
      setIsLoading(true);
      setError(null);

      const response = await authService.register({ email, password, name, tel });
      setUser(response.user);
    } catch (err) {
      const errorMessage = err instanceof Error ? err.message : '회원가입 실패';
      setError(errorMessage);
      throw err;
    } finally {
      setIsLoading(false);
    }
  };

  const logout = async () => {
    try {
      setIsLoading(true);
      authService.logout();
      setUser(null);
      setError(null);
    } catch (err) {
      console.error('로그아웃 오류:', err);
    } finally {
      setIsLoading(false);
    }
  };

  const refreshTokenFn = async () => {
    try {
      const refreshToken = tokenStore.getRefreshToken();
      if (!refreshToken) {
        throw new Error('No refresh token');
      }

      const response = await authService.refresh(refreshToken);
      tokenStore.setAccessToken(response.accessToken);
      tokenStore.setRefreshToken(response.refreshToken);
    } catch (err) {
      console.error('토큰 갱신 실패:', err);
      tokenStore.clear();
      setUser(null);
      throw err;
    }
  };

  const value: AuthContextType = {
    user,
    isAuthenticated: !!user,
    isLoading,
    error,
    login,
    register,
    logout,
    refreshToken: refreshTokenFn,
  };

  return <AuthContext.Provider value={value}>{children}</AuthContext.Provider>;
};

export const useAuth = () => {
  const context = useContext(AuthContext);
  if (!context) {
    throw new Error('useAuth는 AuthProvider 내부에서만 사용 가능합니다');
  }
  return context;
};
```

### Step 2: useAuth 훅 활용 예제
컴포넌트에서 사용:

```typescript
import React from 'react';
import { useAuth } from '../hooks/useAuth';

function MyComponent() {
  const { user, isAuthenticated, isLoading, login, logout } = useAuth();

  if (isLoading) {
    return <div>로딩 중...</div>;
  }

  return (
    <div>
      {isAuthenticated ? (
        <>
          <p>환영합니다, {user?.name}</p>
          <button onClick={logout}>로그아웃</button>
        </>
      ) : (
        <button onClick={() => login('user@example.com', 'password')}>
          로그인
        </button>
      )}
    </div>
  );
}

export default MyComponent;
```

### Step 4: 고급 예제 - ProtectedRoute에서 활용
`apps/frontend/src/components/ProtectedRoute.tsx`를 수정합니다:

```typescript
import React from 'react';
import { Navigate } from 'react-router-dom';
import { useAuth } from '../hooks/useAuth';

interface ProtectedRouteProps {
  children: React.ReactNode;
  requiredRole?: string;
}

export const ProtectedRoute: React.FC<ProtectedRouteProps> = ({
  children,
  requiredRole,
}) => {
  const { isAuthenticated, isLoading, user } = useAuth();

  if (isLoading) {
    return <div className="loading">로딩 중...</div>;
  }

  if (!isAuthenticated) {
    return <Navigate to="/login" replace />;
  }

  // 특정 역할 필요 시 확인
  if (requiredRole && user?.role !== requiredRole) {
    return <Navigate to="/projects" replace />;
  }

  return <>{children}</>;
};
```

### Step 5: useUser 커스텀 훅 (선택)
`apps/frontend/src/hooks/useUser.ts` 파일을 생성합니다:

```typescript
import { useAuth } from '../contexts/AuthContext';

export const useUser = () => {
  const { user, isAuthenticated } = useAuth();

  const isAdmin = user?.role === 'ADMIN';
  const isPM = user?.role === 'PM';
  const isStaff = user?.role === 'STAFF';

  return {
    user,
    isAuthenticated,
    isAdmin,
    isPM,
    isStaff,
  };
};
```
