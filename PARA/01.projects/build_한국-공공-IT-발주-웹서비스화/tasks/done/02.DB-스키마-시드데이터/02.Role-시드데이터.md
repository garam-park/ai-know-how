# Role 시드데이터

## 예상 소요시간
| 구분 | 시간 |
|------|------|
| 사람 (숙련 개발자) | ~15분 |
| AI 에이전트 | ~5분 |

## What (무엇을)
Role 테이블에 프로젝트 멤버 관리에 필요한 4개 프리셋 Role을 삽입합니다: 주관사PM, 참여사PM, 실무자, 뷰어

## Why (왜)
멤버 초대 시 Role 선택이 필수이며, RolePermission 매핑의 기반이 되므로 먼저 Role을 시드데이터로 생성해야 합니다. 이후 모든 권한 설정이 이 4개 Role을 중심으로 이루어집니다.

## 선행 작업
- `02.DB-스키마-시드데이터/01.Prisma-마이그레이션-실행.md`

## 수행 가이드

### Step 1: Prisma Seed 파일 생성
- 대상 파일: `apps/backend/prisma/seed.ts`
- 작업 내용: seed.ts 파일을 새로 생성하여 Role 시드데이터 삽입 로직 작성
  ```typescript
  import { PrismaClient } from '@prisma/client';

  const prisma = new PrismaClient();

  async function main() {
    // Role 데이터 삽입
    await prisma.role.upsert({
      where: { id: 1 },
      update: {},
      create: {
        id: 1,
        name: '주관사PM',
        description: '프로젝트 주관사의 프로젝트 관리자',
        isSystem: true,
      },
    });

    await prisma.role.upsert({
      where: { id: 2 },
      update: {},
      create: {
        id: 2,
        name: '참여사PM',
        description: '참여 회사의 프로젝트 관리자',
        isSystem: true,
      },
    });

    await prisma.role.upsert({
      where: { id: 3 },
      update: {},
      create: {
        id: 3,
        name: '실무자',
        description: '프로젝트 실무 담당자',
        isSystem: true,
      },
    });

    await prisma.role.upsert({
      where: { id: 4 },
      update: {},
      create: {
        id: 4,
        name: '뷰어',
        description: '프로젝트 조회 권한만 보유',
        isSystem: true,
      },
    });

    console.log('Role 시드데이터 삽입 완료');
  }

  main()
    .catch((e) => {
      console.error(e);
      process.exit(1);
    })
    .finally(async () => {
      await prisma.$disconnect();
    });
  ```

### Step 2: package.json에 prisma seed 설정 추가
- 대상 파일: `apps/backend/package.json`
- 작업 내용: package.json에 prisma.seed 설정 추가
  ```json
  {
    "prisma": {
      "seed": "ts-node prisma/seed.ts"
    }
  }
  ```

### Step 3: ts-node 설치 (필요시)
- 작업 내용: prisma seed를 TypeScript로 실행하려면 ts-node 설치 필요
  ```bash
  cd apps/backend
  npm install -D ts-node @types/node
  ```

### Step 4: Prisma Seed 실행
- 작업 내용: 준비된 seed.ts를 실행하여 Role 데이터 삽입
  ```bash
  cd apps/backend
  npx prisma db seed
  ```

### Step 5: Role 테이블 데이터 확인
- 작업 내용: PostgreSQL에서 실제로 4개 Role이 삽입되었는지 확인
  ```bash
  psql -U postgres -d public_it_service_db
  # psql 프롬프트에서
  SELECT id, name, description, "isSystem" FROM "Role";
  ```
  다음과 같은 결과를 확인해야 합니다:
  ```
   id |   name   |              description              | isSystem
  ----+----------+--------------------------------------+----------
    1 | 주관사PM | 프로젝트 주관사의 프로젝트 관리자      |    t
    2 | 참여사PM | 참여 회사의 프로젝트 관리자            |    t
    3 | 실무자   | 프로젝트 실무 담당자                   |    t
    4 | 뷰어     | 프로젝트 조회 권한만 보유               |    t
  ```

## 참고
- upsert를 사용하여 이미 존재하는 데이터가 있어도 안전하게 처리합니다.
- isSystem: true 속성은 시스템 기본 Role임을 나타내며, 추후 Role 삭제 시 보호됩니다.
- 시드 파일은 개발 환경에서 여러 번 실행해도 안전하도록 설계되었습니다.
