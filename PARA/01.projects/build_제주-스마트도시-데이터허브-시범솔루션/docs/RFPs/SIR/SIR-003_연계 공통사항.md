# SIR-003 연계 공통사항

| 항목                  | 내용                |
| --------------------- | ------------------- |
| **요구사항 분류**     | 인터페이스 요구사항 |
| **요구사항 고유번호** | SIR-003             |
| **요구사항 명칭**     | 연계 공통사항       |

## 정의

내·외부 연계 관련 공통으로 요구되는 내용

## 세부내용

- 내·외부 시스템 연계와 관련하여, 정보 유형별(민원, 영상, 음향, 센서, 이벤트, 행정, 로그 등), 기관별(제주도가 지정하는 관내 공영주차장 9개소, 만덕콜센터, 도·시청, 스마트시티 데이터허브 등), 송·수신 정보, 연계 방식(API, DB, 배치 등), 연계 목적, 정보 교환 주기, 예상 이용 규모 등을 현장 사전 조사·분석하고, 그 결과를 기반으로 구체적인 연계 설계 방안을 제시하여야 함
- 현장분석 결과를 바탕으로 내부 시스템 간, 외부 기관 간, 업무 프로세스 간 연계 구조를 포함한 통합 연계 아키텍처를 설계하고, 적용 방안을 제시하여야 함
- 내·외부 시스템과 연계 방안을 제시하고 구축
  - 내·외부간, 업무간, 시스템간 정보연계
  - 내·외부 연계 시 정보 일관성, 정합성 유지 및 보안성 확보
  - 외부연계는 연계근거, 기술표준 적용, 자료 무결성, 연계관리 항목을 정확히 파악하여 구축
  - 송·수신 현황 모니터링이 가능하도록 구축
  - 배치성 연계의 경우 데이터 검증 후 오류데이터의 재송·수신 용이 하도록 구축
- 내·외부 시스템 연계 시 DCAT, DCAT-AP, NGSI-LD 등 표준 데이터 모델과 인터페이스 규격을 적용하여 상호운용성을 확보하도록 설계하여야 하며, 타 지자체 데이터허브, 공공데이터 포털, 외부 플랫폼과의 연계를 고려한 구조를 포함하여야 함
- 외부 기관·시스템과의 데이터 제공·연계를 위해 표준 기반 REST API(Open API) 형태의 외부 인터페이스를 제공할 수 있도록 설계하고, 인증·권한 관리, 접근통제, 호출 제한(Throttle), 로그 관리 등을 포함한 개방형 연계 구조를 적용하여야 함
- 카탈로그 기반 메타데이터, 데이터셋, 이벤트 정보 등은 DCAT RDF(JSON-LD, Turtle 등) 형식으로 외부 조회 및 제공이 가능하도록 인터페이스를 설계하여야 함
- 향후 연계 대상 기관 확대 및 신규 시스템 추가를 고려하여, API 기반 개방형 구조와 표준 인터페이스를 적용하고, 별도 개발 없이 설정 중심으로 연계 확장이 가능하도록 확장성 있는 구조로 설계하여야 함
