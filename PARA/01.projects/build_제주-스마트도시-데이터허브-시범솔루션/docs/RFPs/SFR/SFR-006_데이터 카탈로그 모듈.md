# SFR-006 데이터 카탈로그 모듈

| 항목                  | 내용                 |
| --------------------- | -------------------- |
| **요구사항 분류**     | 기능 요구사항        |
| **요구사항 고유번호** | SFR-006              |
| **요구사항 명칭**     | 데이터 카탈로그 모듈 |

## 정의

DCAT 데이터 카탈로그 제공 및 NGSI-LD 기반 스마트시티 데이터허브 연계 기능 구현

## 세부내용

내용

- 데이터 카탈로그는 웹 포털 기반 사용자 인터페이스를 제공해야 하고, 통합 검색·필터링·탐색 기능을 지원해야하며, 기존 구성요소 및 검증된 컴포넌트를 활용하여 커스터마이징을 최소화할 수 있는 기술 구조가 제시되어야 함
- 스마트시티 데이터허브의 NGSI-LD 표준 인터페이스(/entities 등)를 활용하여 데이터셋 메타데이터를 조회·수집할 수 있어야 함
- 데이터 카탈로그는 AI 기반 자동화 지원 구조로 구현되어야 함
- 표준용어 사전 관리 기능을 제공하고, 메타데이터 작성 시 AI 추천 기준 데이터로 활용할 수 있어야 함
- 업로드 된 인터페이스 정의서 및 데이터 모델 분석을 통해 메타데이터 요소를 자동 추출·초안 생성하고, 사용자가 검토·수정할 수 있어야 함
- 신규 용어 입력 시 표준용어 후보 등록 및 관리자 승인 반영 기능을 제공해야 함
- 데이터허브에 등록된 데이터 모델 자동 탐색을 통해 데이터셋 등록 후보를 생성할 수 있어야 하며, AI 기반 추천·분석 결과는 로그로 기록되고 리포트로 확인 가능해야 함
- 자연어 기반 질의 입력 방식을 제공하여 데이터 검색이 가능해야 하며, 질의 의도를 분석하여 데이터셋 목록, 요약 정보, 추천 근거, 바로가기를 제공해야 함. 검색 이력은 저장되어 검색 품질 개선 및 운영 분석에 활용 가능해야 함
- 데이터 등록 시 설명, 키워드, 제공기관, 담당부서, 라이선스, 공개등급 등 필수항목 검증 기능을 제공해야 하며, 데이터셋은 조직/분야/주제 기반 분류 구조로 관리 가능을 제공해야 함
- 카탈로그 데이터는 DCAT 또는 DCAT-AP 규격(필요시 DCAT-AP-KR 포함) 에 기반하여 외부 제공될 수 있어야 함
- 데이터셋 메타데이터는 DCAT RDF(JSON-LD, Turtle 등) 형식으로 조회 가능한 기능을 제공해야 함
- 데이터 카탈로그는 오픈소스화를 고려하여 확장성·호환성·업데이트·보안 패치 전략을 명확히 제시해야 함
