# 멤버 중복투입 경고 UI

## 예상 소요시간
| 구분 | 시간 |
|------|------|
| 사람 (숙련 개발자) | ~25분 |
| AI 에이전트 | ~10분 |

## What (무엇을)
멤버 초대 시 해당 사용자의 다른 프로젝트 투입률 합계가 100%를 초과하면 경고를 표시합니다. 투입률 변경 시에도 같은 검증을 수행합니다. 백엔드의 overlap detection API 응답을 프론트엔드에서 시각적으로 표현합니다.

## Why (왜)
한 사람이 여러 프로젝트에 참여할 때 총 투입률이 100%를 넘으면 과투입 상태입니다. 현재 백엔드에서 overlap 감지 로직은 있지만, 프론트엔드에서 이를 사용자에게 알려주지 않습니다. PM이 멤버 배정 시 과투입 여부를 확인할 수 없으면 리소스 관리가 불가능합니다.

## 선행 작업
- `15.UX-핵심플로우/04.멤버-초대-UI.md` (멤버 초대/투입률 변경 UI 필요)

## 수행 가이드

### Step 1: 멤버 초대 시 과투입 경고
- 대상 파일: `apps/frontend/src/components/InviteMemberForm.tsx`
- 작업 내용:
  - 이메일 입력 후 blur 시 해당 사용자의 기존 투입률 조회
  - 백엔드 응답에 overlap 정보가 포함되면 경고 표시
  ```typescript
  const [overlapWarning, setOverlapWarning] = useState('');

  // 이메일 blur 시 기존 투입률 확인
  const checkOverlap = async (email: string) => {
    if (!email) return;
    try {
      // 백엔드에서 멤버 초대 시 overlap 정보를 응답에 포함
      // 또는 별도 API: GET /users/overlap?email=xxx
      setOverlapWarning('');
    } catch {
      // 무시
    }
  };

  // JSX: 경고 표시
  {overlapWarning && (
    <div style={{ padding: '8px 12px', background: '#fffff0', border: '1px solid #d69e2e', borderRadius: 4, fontSize: 13, color: '#744210', marginBottom: 8 }}>
      ⚠️ {overlapWarning}
    </div>
  )}
  ```

### Step 2: 멤버 목록에서 과투입 멤버 강조
- 대상 파일: `apps/frontend/src/pages/MembersTab.tsx`
- 작업 내용:
  - 투입률이 100% 초과인 행에 경고 아이콘/배경색 추가 (기존 빨간 글씨에 추가)
  - 툴팁으로 "다른 프로젝트 포함 총 투입률: N%" 표시
  ```typescript
  // 투입률 셀
  <td style={{
    padding: '10px 12px',
    textAlign: 'right',
    color: m.inputRate > 100 ? '#e53e3e' : 'inherit',
    background: m.inputRate > 100 ? '#fff5f5' : 'transparent',
  }}>
    {m.inputRate > 100 && '⚠️ '}
    {m.inputRate}%
  </td>
  ```

### Step 3: 투입률 변경 시 실시간 검증
- 대상 파일: `apps/frontend/src/pages/MembersTab.tsx`
- 작업 내용:
  - 투입률 인라인 편집 중 값이 변경되면 전체 프로젝트 합산 투입률 표시
  - 백엔드 PATCH 응답에 overlap 정보 포함 시 경고 토스트 표시
  ```typescript
  const handleInputRateChange = async (memberId: number, newRate: number) => {
    try {
      const res = await api.patch(`/projects/${projectId}/members/${memberId}`, {
        inputRate: newRate,
      });
      // 응답에 totalInputRate 필드가 있으면 확인
      if (res.result?.totalInputRate > 100) {
        addToast('warning', `해당 멤버의 전체 프로젝트 투입률이 ${res.result.totalInputRate}%입니다.`);
      }
      fetchMembers();
    } catch (err: any) {
      addToast('error', err?.message || '수정에 실패했습니다.');
    }
  };
  ```

### Step 4: 백엔드 overlap 정보 응답 보강 (필요 시)
- 대상 파일: `apps/backend/src/modules/project-member/project-member.service.ts`
- 작업 내용:
  - 멤버 초대/수정 응답에 해당 사용자의 전체 프로젝트 투입률 합계 포함
  ```typescript
  // invite 또는 update 응답에 추가
  const totalInputRate = await this.prisma.projectMember.aggregate({
    where: { userId: member.userId, deletedAt: null },
    _sum: { inputRate: true },
  });

  return {
    ...member,
    totalInputRate: totalInputRate._sum.inputRate || 0,
  };
  ```

## 완료 기준
- [ ] 멤버 초대 시 과투입(>100%) 상태면 경고 메시지 표시
- [ ] 경고는 초대를 막지 않음 (경고만, PM 판단에 맡김)
- [ ] 멤버 목록에서 투입률 100% 초과 행 시각적 강조
- [ ] 투입률 변경 시 전체 합산 결과 안내
- [ ] 경고 메시지에 구체적 수치 포함 (총 투입률 N%)
- [ ] Toast 알림과 연동하여 비침투적 경고
