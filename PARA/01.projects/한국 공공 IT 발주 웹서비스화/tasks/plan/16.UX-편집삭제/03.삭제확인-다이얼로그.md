# 삭제 확인 다이얼로그 공통 컴포넌트

## 예상 소요시간
| 구분 | 시간 |
|------|------|
| 사람 (숙련 개발자) | ~20분 |
| AI 에이전트 | ~8분 |

## What (무엇을)
`window.confirm` 대신 사용할 커스텀 확인 다이얼로그 공통 컴포넌트를 만들고, 기존 삭제 로직에서 사용하는 `confirm()`을 이 컴포넌트로 교체합니다.

## Why (왜)
`window.confirm()`은 브라우저 기본 UI로 커스터마이징이 불가능하고, 사용자 경험이 좋지 않습니다. "삭제" 같은 위험한 작업에는 명확한 경고 메시지와 시각적 강조(빨간 버튼 등)가 필요합니다. 재사용 가능한 공통 컴포넌트로 만들어 전체 앱에서 일관된 UX를 제공합니다.

## 수행 가이드

### Step 1: ConfirmDialog 컴포넌트 생성
- 대상 파일: `apps/frontend/src/components/ConfirmDialog.tsx` (신규)
- 작업 내용:
  ```typescript
  interface Props {
    open: boolean;
    title: string;
    message: string;
    confirmLabel?: string;     // 기본값: "삭제"
    cancelLabel?: string;      // 기본값: "취소"
    variant?: 'danger' | 'warning' | 'info';  // 버튼 색상
    onConfirm: () => void;
    onCancel: () => void;
  }

  export default function ConfirmDialog({
    open, title, message,
    confirmLabel = '삭제', cancelLabel = '취소',
    variant = 'danger',
    onConfirm, onCancel,
  }: Props) {
    if (!open) return null;

    const confirmColors = {
      danger: { bg: '#e53e3e', hover: '#c53030' },
      warning: { bg: '#d69e2e', hover: '#b7791f' },
      info: { bg: '#2b6cb0', hover: '#2c5282' },
    };

    return (
      <div style={{ position: 'fixed', inset: 0, background: 'rgba(0,0,0,0.4)', display: 'flex', alignItems: 'center', justifyContent: 'center', zIndex: 1100 }}>
        <div style={{ background: '#fff', borderRadius: 8, padding: 24, width: 400, maxWidth: '90vw' }}>
          <h3 style={{ fontSize: 16, fontWeight: 600, marginBottom: 8 }}>{title}</h3>
          <p style={{ color: '#4a5568', fontSize: 14, marginBottom: 20 }}>{message}</p>
          <div style={{ display: 'flex', justifyContent: 'flex-end', gap: 8 }}>
            <button onClick={onCancel}
              style={{ padding: '8px 16px', background: '#edf2f7', border: 'none', borderRadius: 4, cursor: 'pointer' }}>
              {cancelLabel}
            </button>
            <button onClick={onConfirm}
              style={{ padding: '8px 16px', background: confirmColors[variant].bg, color: '#fff', border: 'none', borderRadius: 4, cursor: 'pointer' }}>
              {confirmLabel}
            </button>
          </div>
        </div>
      </div>
    );
  }
  ```

### Step 2: useConfirm 커스텀 훅 (선택적)
- 대상 파일: `apps/frontend/src/hooks/useConfirm.ts` (신규)
- 작업 내용:
  ```typescript
  import { useState, useCallback } from 'react';

  interface ConfirmState {
    open: boolean;
    title: string;
    message: string;
    onConfirm: () => void;
  }

  export function useConfirm() {
    const [state, setState] = useState<ConfirmState>({
      open: false, title: '', message: '', onConfirm: () => {},
    });

    const confirm = useCallback((title: string, message: string): Promise<boolean> => {
      return new Promise((resolve) => {
        setState({
          open: true,
          title,
          message,
          onConfirm: () => { setState((s) => ({ ...s, open: false })); resolve(true); },
        });
      });
    }, []);

    const cancel = useCallback(() => {
      setState((s) => ({ ...s, open: false }));
    }, []);

    return { ...state, confirm, cancel };
  }
  ```
