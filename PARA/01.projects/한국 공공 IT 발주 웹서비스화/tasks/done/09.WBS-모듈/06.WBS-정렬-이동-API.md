# WBS 정렬/이동 API

## 예상 소요시간
| 구분 | 시간 |
|------|------|
| 사람 (숙련 개발자) | ~40분 |
| AI 에이전트 | ~15분 |

## What (무엇을)
PATCH /api/v1/projects/:id/wbs-nodes/reorder 엔드포인트를 구현하여 노드의 순서를 변경하거나 부모 노드를 이동할 수 있게 합니다.

## Why (왜)
사용자가 WBS를 드래그&드롭으로 재구성할 수 있어야 하며, 부모 노드 변경 시 depth가 자동으로 재계산되어야 합니다. 이동으로 인한 진척률 변경도 반영해야 합니다.

## 선행 작업
- `09.WBS-모듈/05.진척률-가중평균-재계산.md`

## 수행 가이드

### Step 1: ReorderWbsRequest DTO 정의
`apps/backend/src/modules/wbs/dto/reorder-wbs.dto.ts`:
```typescript
import { IsNumber, IsOptional, IsArray, ValidateNested } from 'class-validator';
import { Type } from 'class-transformer';

class ReorderItem {
  @IsNumber()
  nodeId: number;

  @IsNumber()
  order: number;

  @IsOptional()
  @IsNumber()
  parentId?: number;
}

export class ReorderWbsDto {
  @IsArray()
  @ValidateNested({ each: true })
  @Type(() => ReorderItem)
  orders: ReorderItem[];
}
```

### Step 2: WbsService에 재정렬 로직 추가
`apps/backend/src/modules/wbs/wbs.service.ts`:
```typescript
async reorderWbsNodes(
  projectId: number,
  reorderDto: ReorderWbsDto,
): Promise<any[]> {
  const affectedParents = new Set<number>();
  const updatedNodes: any[] = [];

  // 트랜잭션으로 모든 변경사항 처리
  await this.prisma.$transaction(async (tx) => {
    for (const item of reorderDto.orders) {
      const node = await tx.wbsNode.findUnique({
        where: { id: item.nodeId },
      });

      if (!node) {
        throw new NotFoundException(`Node ${item.nodeId} not found`);
      }

      // 기존 부모 기록 (진척률 재계산 대상)
      if (node.parentId) {
        affectedParents.add(node.parentId);
      }

      // 새 부모가 있으면 기록
      if (item.parentId !== undefined && item.parentId !== null) {
        affectedParents.add(item.parentId);
      }

      // parentId가 변경되면 depth도 재계산
      let newDepth = node.depth;
      if (item.parentId !== undefined && item.parentId !== node.parentId) {
        const newParent = item.parentId
          ? await tx.wbsNode.findUnique({
              where: { id: item.parentId },
            })
          : null;

        // 새 부모가 CATEGORY인지 확인
        if (newParent && newParent.type !== 'CATEGORY') {
          throw new BadRequestException('TASK 노드 하위에는 노드를 이동할 수 없습니다');
        }

        // 새 depth 계산
        newDepth = newParent ? newParent.depth + 1 : 0;

        // maxWbsDepth 초과 검증 (하위 노드 포함)
        const project = await this.prisma.project.findUnique({
          where: { id: projectId },
        });
        const maxDepth = project.maxWbsDepth || 5;
        if (newDepth > maxDepth) {
          throw new BadRequestException('WBS depth 초과');
        }

        // 하위 노드들의 depth도 연쇄 업데이트
        await this.updateChildrenDepth(
          tx,
          item.nodeId,
          newDepth,
          maxDepth,
        );
      }

      // 노드 업데이트
      const updatedNode = await tx.wbsNode.update({
        where: { id: item.nodeId },
        data: {
          order: item.order,
          parentId: item.parentId !== undefined ? item.parentId : undefined,
          depth: newDepth,
        },
        include: {
          assignee: true,
          projectCompany: true,
        },
      });

      updatedNodes.push(updatedNode);
    }

    // 영향받은 부모 노드들의 진척률 재계산
    for (const parentId of affectedParents) {
      await this.recalculateProgress(parentId);
    }
  });

  return updatedNodes;
}

private async updateChildrenDepth(
  tx: any,
  parentId: number,
  parentDepth: number,
  maxDepth: number,
) {
  const children = await tx.wbsNode.findMany({
    where: {
      parentId,
      deletedAt: null,
    },
  });

  for (const child of children) {
    const newChildDepth = parentDepth + 1;

    if (newChildDepth > maxDepth) {
      throw new BadRequestException('WBS depth 초과');
    }

    await tx.wbsNode.update({
      where: { id: child.id },
      data: { depth: newChildDepth },
    });

    // 재귀적으로 하위 노드도 업데이트
    await this.updateChildrenDepth(tx, child.id, newChildDepth, maxDepth);
  }
}
```

### Step 3: WbsController에 재정렬 엔드포인트 추가
`apps/backend/src/modules/wbs/wbs.controller.ts`:
```typescript
@Patch('reorder')
@RequirePermission(Resource.WBS, Action.EDIT)
async reorderWbsNodes(
  @Param('id') projectId: string,
  @Body() reorderDto: ReorderWbsDto,
) {
  const updatedNodes = await this.wbsService.reorderWbsNodes(
    parseInt(projectId),
    reorderDto,
  );

  return {
    code: 200000,
    result: {
      nodes: updatedNodes,
    },
  };
}
```

## 완료 기준
- [ ] ReorderWbsDto 정의 및 검증 완료
- [ ] order 값으로 순서 변경 가능
- [ ] parentId 값으로 부모 이동 가능
- [ ] 부모 이동 시 depth 자동 재계산
- [ ] 하위 노드의 depth도 연쇄 업데이트
- [ ] maxWbsDepth 초과 시 에러 반환
- [ ] TASK 노드 하위로의 이동 시도 시 에러 반환
- [ ] 이동 후 이전/새 부모의 진척률 재계산
- [ ] 트랜잭션으로 원자성(atomicity) 보장
- [ ] 200000 응답 반환

## 참고
- 트랜잭션 사용: prisma.$transaction
- depth 연쇄 업데이트: 재귀 함수 사용
- 진척률 재계산: 이전 부모와 새 부모 모두 대상
