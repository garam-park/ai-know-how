# 공유 패키지 (shared) 설정

## 예상 소요시간
| 구분 | 시간 |
|------|------|
| 사람 (숙련 개발자) | ~20분 |
| AI 에이전트 | ~8분 |

## What (무엇을)
`packages/shared` 패키지를 생성하여 프론트엔드와 백엔드에서 공유할 TypeScript 타입, Enum, 상수를 정의합니다. 이 패키지는 `@shared/types` 이름으로 workspace 내 모든 앱에서 참조 가능하도록 설정됩니다.

## Why (왜)
프론트엔드와 백엔드 간 데이터 모델(User, Company, Project 등)의 정의가 일치하지 않으면 타입 오류가 발생합니다. 공유 패키지를 사용하면 한 곳에서 타입과 Enum을 관리하여 불일치를 방지하고, 양쪽 코드의 신뢰성을 높입니다.

## 수행 가이드

### Step 1: 공유 패키지 디렉토리 구조 생성
```bash
cd project-root
mkdir -p packages/shared/src/{types,enums,constants}
```

### Step 2: 공유 패키지 package.json 생성
- 대상 파일: `project-root/packages/shared/package.json`
- 작업 내용:

```json
{
  "name": "@shared/types",
  "version": "1.0.0",
  "description": "공유 타입 정의 및 Enum",
  "main": "dist/index.js",
  "types": "dist/index.d.ts",
  "scripts": {
    "build": "tsc",
    "typecheck": "tsc --noEmit"
  },
  "devDependencies": {
    "typescript": "^5.3.0"
  }
}
```

## 참고
- 공유 패키지는 외부 의존성을 최소화하도록 설계 (기본 TypeScript만 사용)
- 향후 추가 타입은 이 패키지에 계속 추가됨
- 백엔드와 프론트엔드는 모두 `@shared/types` 패키지를 workspace 프로토콜로 의존성 추가
