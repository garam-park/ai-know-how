# Company 모듈 구조생성

## 예상 소요시간
| 구분 | 시간 |
|------|------|
| 사람 (숙련 개발자) | ~15분 |
| AI 에이전트 | ~5분 |

## What (무엇을)
`apps/backend/src/modules/company/` 디렉토리를 생성하고 NestJS 모듈 구조를 세팅합니다. Company 모듈의 기본 프레임워크(module, controller, service, DTO)를 완성하고 AppModule에 등록합니다.

## Why (왜)
회사 CRUD 작업의 기반 구조가 필요합니다. 프로젝트에 참여사나 하청사를 등록할 때 기존 회사를 검색하거나 새로 생성해야 하는데, 이를 위한 API 엔드포인트의 기초입니다.

## 선행 작업
- `04.권한-시스템/03.PermissionGuard-구현.md`

## 수행 가이드

### Step 1: Company 모듈 디렉토리 및 파일 생성
- 대상 경로: `apps/backend/src/modules/company/`
- 작업 내용:
  ```bash
  mkdir -p apps/backend/src/modules/company/dto
  touch apps/backend/src/modules/company/company.module.ts
  touch apps/backend/src/modules/company/company.controller.ts
  touch apps/backend/src/modules/company/company.service.ts
  ```

### Step 2: company.module.ts 작성
- 대상 파일: `apps/backend/src/modules/company/company.module.ts`
- 작업 내용:
  ```typescript
  import { Module } from '@nestjs/common';
  import { PrismaService } from '@/prisma/prisma.service';
  import { CompanyController } from './company.controller';
  import { CompanyService } from './company.service';

  @Module({
    controllers: [CompanyController],
    providers: [CompanyService, PrismaService],
    exports: [CompanyService],
  })
  export class CompanyModule {}
  ```

- 작업 내용:
  ```typescript
  import { Injectable } from '@nestjs/common';
  import { PrismaService } from '@/prisma/prisma.service';

  @Injectable()
  export class CompanyService {
    constructor(private prisma: PrismaService) {}
  ```

### Step 5: dto/ 폴더 및 기본 DTO 작성
- 대상 파일: `apps/backend/src/modules/company/dto/company.dto.ts`

### Step 6: AppModule에 CompanyModule 등록
- 대상 파일: `apps/backend/src/app.module.ts`
- 작업 내용:
  - imports 배열에 `CompanyModule` 추가
  - 예시:
    ```typescript
    import { CompanyModule } from '@/modules/company/company.module';

    @Module({
      imports: [
        // ... 기타 모듈들
        CompanyModule,
      ],
    })
    export class AppModule {}
    ```

## 참고
- NestJS 모듈 구조: @Module 데코레이터로 컨트롤러, 프로바이더, 익스포트 항목 선언
- PrismaService: 데이터베이스 접근용 의존성 주입
- DTO (Data Transfer Object): API 요청/응답 타입 정의
