# Project 모듈 구조생성

## 예상 소요시간
| 구분 | 시간 |
|------|------|
| 사람 (숙련 개발자) | ~15분 |
| AI 에이전트 | ~5분 |

## What (무엇을)
`apps/backend/src/modules/project/` 디렉토리를 생성하고 NestJS 모듈 구조를 세팅합니다. Project 모듈의 기본 프레임워크(module, controller, service, DTO)를 완성하고 AppModule에 등록합니다.

## Why (왜)
프로젝트(사업) 관리의 기반 구조입니다. 모든 협력 관리 기능은 프로젝트를 중심으로 구성됩니다. 프로젝트 생성, 조회, 수정, 삭제 API의 기초입니다.

## 선행 작업
- `04.권한-시스템/03.PermissionGuard-구현.md`

## 수행 가이드

### Step 1: Project 모듈 디렉토리 및 파일 생성
- 대상 경로: `apps/backend/src/modules/project/`
- 작업 내용:
  ```bash
  mkdir -p apps/backend/src/modules/project/dto
  touch apps/backend/src/modules/project/project.module.ts
  touch apps/backend/src/modules/project/project.controller.ts
  touch apps/backend/src/modules/project/project.service.ts
  ```

### Step 2: project.module.ts 작성
- 대상 파일: `apps/backend/src/modules/project/project.module.ts`
- 작업 내용:
  ```typescript
  import { Module } from '@nestjs/common';
  import { PrismaService } from '@/prisma/prisma.service';
  import { ProjectController } from './project.controller';
  import { ProjectService } from './project.service';

  @Module({
    controllers: [ProjectController],
    providers: [ProjectService, PrismaService],
    exports: [ProjectService],
  })
  export class ProjectModule {}
  ```

### Step 3: project.service.ts 작성 (스켈레톤)
- 대상 파일: `apps/backend/src/modules/project/project.service.ts`
- 작업 내용:
  ```typescript
  import { Injectable } from '@nestjs/common';
  import { PrismaService } from '@/prisma/prisma.service';

  @Injectable()
  export class ProjectService {
    constructor(private prisma: PrismaService) {}

    // 추후 추가될 메서드들
  }
  ```

### Step 4: project.controller.ts 작성 (스켈레톤)
- 대상 파일: `apps/backend/src/modules/project/project.controller.ts`
- 작업 내용:
  ```typescript
  import { Controller } from '@nestjs/common';
  import { ProjectService } from './project.service';

  @Controller('api/v1/projects')
  export class ProjectController {
    constructor(private readonly projectService: ProjectService) {}

    // 추후 추가될 엔드포인트들
  }
  ```

### Step 5: dto/ 폴더 및 기본 DTO 작성
- 대상 파일: `apps/backend/src/modules/project/dto/project.dto.ts`
- 작업 내용:
  ```typescript
  export class ProjectDto {
    id: string;
    name: string;
    code?: string;
    description?: string;
    startDate: Date;
    endDate: Date;
    budget?: number;
    maxWbsDepth: number;
    createdAt: Date;
    updatedAt: Date;
  }

  export class CreateProjectDto {
    name: string;
    code?: string;
    description?: string;
    startDate: Date;
    endDate: Date;
    budget?: number;
    maxWbsDepth?: number;
  }

  export class UpdateProjectDto {
    name?: string;
    code?: string;
    description?: string;
    startDate?: Date;
    endDate?: Date;
    budget?: number;
    maxWbsDepth?: number;
  }

  export class ProjectSummaryDto {
    id: string;
    name: string;
    code?: string;
    startDate: Date;
    endDate: Date;
    daysLeft: number;
    progress: number;
    myRole: string;
    myCompany: string;
  }
  ```

### Step 6: AppModule에 ProjectModule 등록
- 대상 파일: `apps/backend/src/app.module.ts`
- 작업 내용:
  - imports 배열에 `ProjectModule` 추가
  - 예시:
    ```typescript
    import { ProjectModule } from '@/modules/project/project.module';

    @Module({
      imports: [
        // ... 기타 모듈들
        ProjectModule,
      ],
    })
    export class AppModule {}
    ```

## 참고
- NestJS 모듈 구조: @Module 데코레이터로 컨트롤러, 프로바이더, 익스포트 항목 선언
- maxWbsDepth: WBS (Work Breakdown Structure) 최대 깊이, 기본값 5
- 프로젝트의 기간(startDate~endDate)은 참여 멤버 관리에서 중복투입 감지에 사용
