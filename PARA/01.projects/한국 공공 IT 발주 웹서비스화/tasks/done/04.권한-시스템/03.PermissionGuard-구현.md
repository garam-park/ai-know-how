# PermissionGuard 구현

## 예상 소요시간
| 구분 | 시간 |
|------|------|
| 사람 (숙련 개발자) | ~30분 |
| AI 에이전트 | ~10분 |

## What (무엇을)
PermissionGuard를 구현하여 @RequirePermission에 지정된 권한을 검증하고, 권한에 맞는 Scope을 추출하여 컨트롤러가 접근할 수 있도록 합니다.

## Why (왜)
모든 인증 필요 API에서 자동으로 권한 체크를 수행하고, 사용자의 접근 범위(Scope)를 전달하여 컨트롤러가 올바른 데이터만 조회/조작하도록 합니다.

## 수행 가이드

### Step 1: PermissionGuard 생성
- 대상 파일: `apps/backend/src/common/guards/permission.guard.ts`
- 작업 내용: 권한 검증 Guard 구현
  ```typescript
  import {
    Injectable,
    CanActivate,
    ExecutionContext,
    ForbiddenException,
    BadRequestException,
  } from '@nestjs/common';
  import { Reflector } from '@nestjs/core';
  import { PermissionService } from '../../modules/permission/permission.service';
  import { RequirePermissionMetadata } from '../decorators/require-permission.decorator';
  import { Scope } from '@shared/enums';

  @Injectable()
  export class PermissionGuard implements CanActivate {
    constructor(
      private reflector: Reflector,
      private permissionService: PermissionService,
    ) {}

    async canActivate(context: ExecutionContext): Promise<boolean> {
      const request = context.switchToHttp().getRequest();
      const handler = context.getHandler();

      // 1. 메타데이터에서 필요 권한 추출
      const permission = this.reflector.get<RequirePermissionMetadata>(
        'permission',
        handler,
      );

      // @RequirePermission이 없으면 권한 체크 없이 통과
      if (!permission) {
        return true;
      }

      // 2. Request에서 사용자, 프로젝트 정보 추출
      const user = request.user;
      if (!user) {
        throw new ForbiddenException({
          code: 403000,
          message: '인증되지 않은 사용자입니다',
        });
      }

      // projectId를 param이나 body에서 추출
      // 일반적으로 /api/v1/projects/:projectId/... 형태이므로 params.projectId 사용
      const projectId = request.params.projectId || request.params.id;
      if (!projectId) {
        throw new BadRequestException({
          code: 400000,
          message: 'projectId가 필요합니다',
        });
      }

      // 3. PermissionService로 Scope 조회
      const scope = await this.permissionService.getScope(
        projectId,
        user.id,
        permission.resource,
        permission.action,
      );

      // 4. Scope이 NONE이면 권한 없음
      if (scope === Scope.NONE) {
        throw new ForbiddenException({
          code: 403001,
          message: `${permission.resource} ${permission.action} 권한이 없습니다`,
        });
      }

      // 5. Request에 scope 저장 (컨트롤러에서 접근 가능)
      request.scope = scope;

      return true;
    }
  }
  ```

### Step 3: 글로벌 또는 선택적 Guard 등록
- 작업 내용: Guard를 전역으로 등록하거나 필요한 컨트롤러에만 사용

  **방법 1: 글로벌 Guard 등록 (권장 아님 - 성능상 모든 요청 검사)**

  ```typescript
  import { NestFactory } from '@nestjs/core';
  import { AppModule } from './app.module';
  import { APP_GUARD } from '@nestjs/core';

  async function bootstrap() {
    const app = await NestFactory.create(AppModule);
    // 글로벌 Guard 등록은 권장하지 않음
  ```

  **방법 2: 필요한 컨트롤러마다 등록 (권장)**
  ```typescript
  import { Controller, Get, Patch, UseGuards } from '@nestjs/common';
  import { JwtAuthGuard } from '../../common/guards/jwt-auth.guard';
  import { PermissionGuard } from '../../common/guards/permission.guard';
  import { RequirePermission } from '../../common/decorators/require-permission.decorator';
  import { CurrentUser } from '../../common/decorators/current-user.decorator';
  import { Resource, Action } from '@shared/enums';

  @Controller('api/v1/projects/:projectId/wbs')
  export class WbsController {
    @UseGuards(JwtAuthGuard, PermissionGuard)
    @RequirePermission(Resource.WBS, Action.VIEW)
    @Get()
    async getWbs(@CurrentUser() user: any, @Request() req: any) {
      const scope = req.scope; // PermissionGuard에서 저장한 scope
      
      // scope에 따라 쿼리 필터링
      // ALL: 모든 WBS 조회
      // OWN_COMPANY: 자사 WBS만 조회
      // OWN_TASK: 할당된 작업 WBS만 조회
      
      return {
        code: 200000,
        result: {
          wbs: [],
          scope,
        },
      };
    }

    @UseGuards(JwtAuthGuard, PermissionGuard)
    @RequirePermission(Resource.WBS, Action.EDIT)
    @Patch(':wbsId')
    async updateWbs(@CurrentUser() user: any, @Request() req: any) {
      const scope = req.scope;
      
      // scope 기반으로 수정 권한 재검증 가능
      if (scope === Scope.NONE) {
        throw new ForbiddenException('편집 권한이 없습니다');
      }
      
      return {
        code: 200000,
        result: { wbs: {} },
      };
    }
  }
  ```

### Step 4: 컨트롤러에서 Scope 활용
- 작업 내용: Scope을 이용한 쿼리 필터링 예시
  ```typescript
  import { Request } from 'express';
  import { Controller, Get, Param, UseGuards, Req } from '@nestjs/common';
  import { Scope } from '@shared/enums';

  @Controller('api/v1/projects/:projectId/wbs')
  export class WbsController {
    constructor(private wbsService: WbsService) {}

    @UseGuards(JwtAuthGuard, PermissionGuard)
    @RequirePermission(Resource.WBS, Action.VIEW)
    @Get()
    async getWbs(
      @Param('projectId') projectId: string,
      @CurrentUser() user: any,
      @Req() req: Request,
    ) {
      const scope = (req as any).scope; // PermissionGuard에서 저장한 scope

      // scope에 따라 적절한 쿼리 실행
      let wbs;
      if (scope === Scope.ALL) {
        wbs = await this.wbsService.findAll(projectId);
      } else if (scope === Scope.OWN_COMPANY) {
        wbs = await this.wbsService.findByCompany(projectId, user.companyId);
      } else if (scope === Scope.OWN_TASK) {
        wbs = await this.wbsService.findByAssignee(projectId, user.id);
      } else {
        throw new ForbiddenException('WBS 조회 권한이 없습니다');
      }

      return {
        code: 200000,
        result: { wbs, scope },
      };
    }
  }
  ```

### Step 5: API 테스트
- 작업 내용: PermissionGuard 동작 검증

  **테스트 설정**
  1. 테스트 프로젝트 생성
  2. 서로 다른 Role의 사용자를 ProjectMember로 추가
  3. 각 사용자가 API를 호출하여 응답 확인

  **시나리오 1: 권한 있음 (주관사PM - WBS VIEW)**
  ```bash
  curl -X GET http://localhost:3000/api/v1/projects/project-1/wbs \
    -H "Authorization: Bearer <accessToken>"
  ```
  
  응답:
  ```json
  {
    "code": 200000,
    "result": {
      "wbs": [...],
      "scope": "ALL"
    }
  }
  ```

  **시나리오 2: 권한 없음 (뷰어 - WBS EDIT)**
  ```bash
  curl -X PATCH http://localhost:3000/api/v1/projects/project-1/wbs/wbs-1 \
    -H "Authorization: Bearer <accessToken>" \
    -H "Content-Type: application/json" \
    -d '{}'
  ```

  응답:
  ```json
  {
    "code": 403001,
    "message": "WBS EDIT 권한이 없습니다"
  }
  ```

  **시나리오 3: 범위 제한 (참여사PM - WBS VIEW → OWN_COMPANY)**

## 참고
- PermissionGuard는 JwtAuthGuard 이후에 실행되어야 user 정보가 있습니다.
- request.scope에 저장된 값으로 컨트롤러가 쿼리를 필터링해야 합니다.
- Scope.NONE은 Guard에서 이미 거부하므로, 컨트롤러에서는 ALL/OWN_COMPANY/OWN_TASK만 처리하면 됩니다.
- 향후 Scope별 필터링 로직은 Service나 Repository에 구현하는 것이 좋습니다.
