## 핵심 개념: FE 개발자를 위한 유추

FE 개발자 관점에서 배포 스크립트를 이해하면:

```
빌드 스크립트 (package.json)    배포 스크립트 (deploy.sh)
├─ npm run build             ├─ docker pull
├─ npm run test              ├─ docker-compose down
├─ npm run lint              ├─ docker-compose up -d
└─ npm run optimize          └─ 헬스 체크 + 롤백

package.json의 scripts = 자동화된 작업 목록
deploy.sh = 서버에서 자동화된 작업 목록
```

**배포 흐름:**
```
1. SSH로 배포 서버 접속 (06-B-06)
2. deploy.sh 실행
3. 스크립트가 Docker 이미지 업데이트
4. 헬스 체크로 정상 작동 확인
5. 실패 시 자동 롤백
6. 배포 완료 로그 기록
```

## 실습 1단계: 기본 deploy.sh 스크립트 작성

### 목표
배포 서버에서 새 이미지를 pull하고 컨테이너를 재시작하는 기본 스크립트 작성

### 완료 기준
- `/opt/my-app/deploy.sh` 파일 생성
- 스크립트 실행 권한 설정 (755)
- 로컬에서 수동 테스트 성공

### 단계별 작업

**1-1) deploy.sh 파일 생성**

배포 서버에서 실행:

```bash
# 1. 앱 디렉토리 이동
cd /opt/my-app

# 2. deploy.sh 파일 생성
cat > deploy.sh << 'EOF'
#!/bin/bash

# 배포 스크립트: Docker 이미지 업데이트 및 컨테이너 재시작
# 사용 방법: ./deploy.sh

set -e  # 오류 발생 시 스크립트 중단

# ============================================
# 설정
# ============================================

APP_NAME="my-app"
IMAGE_NAME="ghcr.io/yourname/my-app"
IMAGE_TAG="main"
DOCKER_COMPOSE_FILE="/opt/my-app/docker-compose.yml"
HEALTH_CHECK_URL="http://localhost:3000/health"
HEALTH_CHECK_TIMEOUT=30  # 초
BACKUP_DIR="/opt/my-app/backups/$(date +%Y%m%d_%H%M%S)"

# 색상 정의
RED='\033[0;31m'
GREEN='\033[0;32m'
YELLOW='\033[1;33m'
NC='\033[0m'  # No Color

# ============================================
# 함수 정의
# ============================================

log_info() {
  echo -e "${GREEN}[INFO]${NC} $1"
}

log_warn() {
  echo -e "${YELLOW}[WARN]${NC} $1"
}

log_error() {
  echo -e "${RED}[ERROR]${NC} $1"
}

health_check() {
  local counter=0
  log_info "헬스 체크 시작 (최대 ${HEALTH_CHECK_TIMEOUT}초 대기)..."

  while [ $counter -lt $HEALTH_CHECK_TIMEOUT ]; do
    if curl -sf "$HEALTH_CHECK_URL" > /dev/null 2>&1; then
      log_info "✓ 헬스 체크 성공"
      return 0
    fi
    counter=$((counter + 1))
    echo -n "."
    sleep 1
  done

  log_error "✗ 헬스 체크 실패 (${HEALTH_CHECK_TIMEOUT}초 타임아웃)"
  return 1
}

rollback() {
  log_warn "롤백 시작..."

  # 이전 컨테이너 시작
  docker-compose -f "$DOCKER_COMPOSE_FILE" up -d

  # 롤백 후 헬스 체크
  if health_check; then
    log_info "✓ 롤백 성공"
    return 0
  else
    log_error "✗ 롤백 실패 - 수동 개입 필요"
    return 1
  fi
}

save_state() {
  log_info "현재 상태 저장 (롤백용)..."

  mkdir -p "$BACKUP_DIR"

  # 현재 이미지 ID 저장
  CURRENT_IMAGE=$(docker images "$IMAGE_NAME" --format "{{.ID}}" | head -1)
  echo "$CURRENT_IMAGE" > "$BACKUP_DIR/image.id"

  # 현재 컨테이너 상태 저장
  docker ps -a --filter="name=$APP_NAME" > "$BACKUP_DIR/containers.log"

  # docker-compose 설정 백업
  cp "$DOCKER_COMPOSE_FILE" "$BACKUP_DIR/docker-compose.yml.bak"

  log_info "백업 저장 완료: $BACKUP_DIR"
}

# ============================================
# 배포 프로세스
# ============================================

main() {
  echo "========================================="
  echo "배포 시작: $(date '+%Y-%m-%d %H:%M:%S')"
  echo "========================================="

  # 1. 현재 상태 저장
  save_state

  # 2. 최신 이미지 pull
  log_info "Docker 이미지 pull 시작: $IMAGE_NAME:$IMAGE_TAG"
  if ! docker pull "$IMAGE_NAME:$IMAGE_TAG"; then
    log_error "이미지 pull 실패"
    exit 1
  fi
  log_info "✓ 이미지 pull 완료"

  # 3. 기존 컨테이너 중지 및 삭제
  log_info "기존 컨테이너 중지..."
  docker-compose -f "$DOCKER_COMPOSE_FILE" down --remove-orphans || true
  log_info "✓ 컨테이너 중지 완료"

  # 4. 새 컨테이너 시작
  log_info "새 컨테이너 시작..."
  if ! docker-compose -f "$DOCKER_COMPOSE_FILE" up -d; then
    log_error "컨테이너 시작 실패"
    rollback
    exit 1
  fi
  log_info "✓ 컨테이너 시작 완료"

  # 5. 헬스 체크
  if ! health_check; then
    log_error "헬스 체크 실패 - 롤백 시작"
    rollback
    exit 1
  fi

  # 6. 배포 완료
  log_info "✓ 배포 완료"

  echo ""
  echo "현재 실행 중인 컨테이너:"
  docker ps --filter="name=$APP_NAME"

  echo ""
  echo "========================================="
  echo "배포 완료: $(date '+%Y-%m-%d %H:%M:%S')"
  echo "========================================="
}

# 스크립트 실행
main
EOF

# 3. 실행 권한 설정
chmod +x deploy.sh

# 4. 파일 생성 확인
ls -la deploy.sh
# -rwxr-xr-x 1 root root 3456 Apr  1 12:00 deploy.sh
```

**1-2) 스크립트 주요 함수 설명**

| 함수 | 목적 |
|------|------|
| `log_info()` | 정보 메시지 출력 (초록색) |
| `log_warn()` | 경고 메시지 출력 (노란색) |
| `log_error()` | 오류 메시지 출력 (빨간색) |
| `health_check()` | HTTP 요청으로 앱 정상 작동 확인 |
| `rollback()` | 이전 컨테이너 상태로 복구 |
| `save_state()` | 현재 상태 백업 (롤백용) |
| `main()` | 배포 프로세스 실행 |

**1-3) 스크립트 실행 테스트 (로컬)**

```bash
# 1. 배포 스크립트 실행
./deploy.sh

# 예상 출력:

# =========================================
# [INFO] 현재 상태 저장 (롤백용)...
# [INFO] 백업 저장 완료: /opt/my-app/backups/20260401_123045
# [INFO] Docker 이미지 pull 시작: ghcr.io/yourname/my-app:main
# [INFO] ✓ 이미지 pull 완료
# [INFO] 기존 컨테이너 중지...
# [INFO] ✓ 컨테이너 중지 완료
# [INFO] 새 컨테이너 시작...
# [INFO] ✓ 컨테이너 시작 완료
# [INFO] 헬스 체크 시작...
# [INFO] ✓ 헬스 체크 성공
# [INFO] ✓ 배포 완료
#
# 현재 실행 중인 컨테이너:

#

# =========================================
```

## 실습 2단계: GitHub Actions에서 deploy.sh 호출

### 목표
SSH 배포 워크플로우에서 deploy.sh를 자동으로 실행

### 완료 기준
- GitHub Actions Deploy workflow에서 deploy.sh 호출
- 배포 자동화 완전 구현

### 단계별 작업

**2-1) SSH Action에서 deploy.sh 호출**

`.github/workflows/deploy.yml` 수정:

```yaml
name: Deploy to Server

on:
  push:
    branches:
      - main
  workflow_dispatch:

jobs:
  deploy:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout code
        uses: actions/checkout@v4

      # deploy.sh 호출
      - name: Deploy via SSH
        uses: appleboy/ssh-action@master
        with:
          host: ${{ secrets.SERVER_HOST }}
          username: ${{ secrets.SERVER_USER }}
          key: ${{ secrets.SSH_PRIVATE_KEY }}
          port: ${{ secrets.SERVER_PORT }}

          # deploy.sh 실행
          script: |
            cd /opt/my-app
            bash ./deploy.sh
```

**2-2) 선택사항: SSH 연결 후 로그 저장**

더 상세한 배포 로그를 원한다면:

```yaml
script: |
  cd /opt/my-app
  bash ./deploy.sh 2>&1 | tee deploy-$(date +%Y%m%d-%H%M%S).log

  # 로그 파일 확인
  tail -20 deploy-*.log
```

### 단계별 작업

**3-1) deploy.sh에 로깅 기능 추가**

```bash
# ============================================
# 배포 이력 로깅
# ============================================

DEPLOY_LOG="/opt/my-app/deploy-history.log"
DEPLOY_START_TIME=$(date '+%Y-%m-%d %H:%M:%S')

log_deploy() {
  local status=$1
  local message=$2

  echo "[${DEPLOY_START_TIME}] Status: $status | Message: $message | Image: $IMAGE_NAME:$IMAGE_TAG" >> "$DEPLOY_LOG"
}

# main 함수 마지막에 추가:
# 배포 성공 로그
log_deploy "SUCCESS" "배포 완료"

# 또는 실패 시:
# log_deploy "FAILURE" "배포 실패 - 롤백됨"
```

**3-2) 배포 이력 조회**

배포 서버에서:

```bash

# 예상 출력:
```

```bash
run_migrations() {
```

## 실습 5단계: 배포 스크립트 디버깅

### 목표
배포 실패 시 원인을 파악하고 해결하는 방법 배우기

### 완료 기준
- 배포 실패 원인 파악 능력
- 수동 복구 방법 이해

### 단계별 작업

**5-1) 배포 실패 로그 확인**

```bash
# 1. 배포 스크립트 상세 출력으로 실행 (디버깅 모드)
bash -x /opt/my-app/deploy.sh

# -x 플래그: 각 명령어 실행 전 출력 (디버깅)

# 2. 마지막 배포 로그 확인
tail -50 /opt/my-app/deploy-history.log

# 3. Docker 컨테이너 로그 확인
docker logs my-app | tail -100

# 4. 특정 에러 메시지 검색
grep -i "error" /opt/my-app/deploy-*.log
```

**5-2) 일반적인 배포 실패 원인**

| 오류 | 원인 | 해결 방법 |
|------|------|---------|
| `docker pull` 실패 | 이미지가 GHCR에 없음 | GitHub Actions 빌드 상태 확인 |
| `health_check` 실패 | 앱이 3000 포트 응답 안 함 | `docker logs my-app` 확인 |
| `docker-compose` 명령 오류 | docker-compose 문법 오류 | `docker-compose config` 확인 |
| `port already in use` | 포트 충돌 | `docker ps` 확인, 기존 컨테이너 제거 |
| 권한 오류 (`permission denied`) | 스크립트 실행 권한 없음 | `chmod +x deploy.sh` 실행 |

**5-3) 수동 복구 프로세스**

배포가 완전히 실패했을 때:

```bash
# 1. 현재 상태 확인
docker ps -a
docker images

# 2. 문제 컨테이너 중지 및 삭제
docker stop my-app
docker rm my-app

# 3. 이전 이미지 확인
docker images | grep ghcr.io

# 4. 이전 이미지로 컨테이너 시작
docker run -d -p 3000:3000 --name my-app \
  ghcr.io/yourname/my-app:main

# 또는 docker-compose 사용
docker-compose -f /opt/my-app/docker-compose.yml up -d

# 5. 헬스 체크
curl http://localhost:3000/health

# 6. 문제 파악 후 배포 재시도
cd /opt/my-app && bash ./deploy.sh
```

**5-4) 배포 스크립트 수동 테스트**

배포 전에 로컬에서 테스트:

```bash
# 1. deploy.sh 문법 검증
bash -n /opt/my-app/deploy.sh

# 2. 각 명령어 수동 실행
docker pull ghcr.io/yourname/my-app:main
docker-compose -f /opt/my-app/docker-compose.yml config
curl http://localhost:3000/health

# 3. 전체 배포 시뮬레이션 (드라이런)
bash -x /opt/my-app/deploy.sh 2>&1 | head -50
```

## 핵심 요약

| 개념 | 설명 | 파일 위치 |
|------|------|---------|
| **deploy.sh** | 배포 실행 쉘 스크립트 | `/opt/my-app/deploy.sh` |
| **헬스 체크** | 배포 후 앱 정상 작동 확인 | deploy.sh 내 `health_check()` 함수 |
| **롤백** | 배포 실패 시 이전 상태 복구 | deploy.sh 내 `rollback()` 함수 |
| **배포 이력** | 각 배포 기록 저장 | `/opt/my-app/deploy-history.log` |
| **백업** | 롤백용 현재 상태 저장 | `/opt/my-app/backups/*/` |
