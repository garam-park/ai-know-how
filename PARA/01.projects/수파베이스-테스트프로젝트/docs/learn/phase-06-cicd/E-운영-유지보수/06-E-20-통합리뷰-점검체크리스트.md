```
[ ] main 브랜치에 푸시 시 자동 배포 실행되나?
[ ] 빌드 실패 시 Slack 알림 오나?
[ ] PR 머지 전 자동 테스트 실행되나?
[ ] 배포 완료 후 롤백 가능한가?

최근 30일 배포 현황:
- 총 배포 수: __ 회
- 성공률: __ %
- 평균 배포 시간: __ 분
```

### 자동 배포 프로세스 확인

```bash
# 1. 최근 배포 로그 확인
cd /opt/supabase
git log --oneline -10 | grep "Automated deployment by GitHub Actions"

# 2. GitHub Actions 워크플로 상태
# GitHub 접속 → Actions 탭 → 최근 실행 기록 확인

# 3. 배포 환경 변수 확인
env | grep -i deploy | head -10

# 4. 서비스 정상 작동
docker-compose ps
# 모든 컨테이너 "Up" 상태 확인
```

**문제 발생 시:**


## Phase 5: 배포 & 환경 설정 점검

### 역 프록시 (Caddy) 상태

```
[ ] HTTPS 인증서 유효한가?
[ ] 도메인 리다이렉트 정상인가?
[ ] API 요청이 올바른 포트로 라우팅되나?
[ ] 캐시 설정이 최적화되었나?

점검 명령어:
```

```bash
# 1. Caddy 상태
docker-compose ps | grep caddy
# Status: Up 확인

# 2. HTTPS 인증서 확인
docker exec caddy caddy list-certs

# 3. API 응답 확인
curl -i https://api.your-domain.com/health
# HTTP/1.1 200 OK

# 4. 리다이렉트 확인
curl -L http://your-domain.com/ \
  -o /dev/null -w "Final URL: %{url_effective}\n"
# https://your-domain.com/ ← HTTPS로 리다이렉트 확인
```

### 환경 변수 점검

```bash
# 1. 환경 변수 파일 존재 확인
ls -la /opt/supabase/.env.prod

# 2. 필수 변수 확인
grep -E "DATABASE_URL|JWT_SECRET|API_KEY" /opt/supabase/.env.prod

# 3. 민감 정보 노출 확인 (Git에 커밋되지 않았는지)
git log --all -S 'password' --oneline | head -5
# 결과가 없으면 정상

# 4. 권한 확인 (600 권장)
ls -la /opt/supabase/.env.prod
# 비트: -rw------- (600)
```

### 백업 설정 점검

```bash
# 1. 백업 자동화 확인
crontab -l | grep backup

# 2. 백업 파일 존재 확인
ls -lh /backup/supabase-*.sql.gz | tail -7
# 최근 7일간 백업 있는지 확인

# 3. 백업 크기 이상 확인
du -sh /backup/ | awk '{print $1}'
# 너무 크거나 너무 작으면 이상 신호

# 4. 백업 무결성 테스트 (필수!)
file /backup/supabase-$(date +%Y%m%d)*.sql.gz
# 실패하면: "cannot open"
```

## Phase 6: 모니터링 & 운영 점검

### 모니터링 시스템 상태

```
[ ] Uptime Kuma 대시보드에 모든 서비스 표시되나?
[ ] Grafana 메트릭 수집 중인가?
[ ] 경보 규칙이 활성화되었나?
[ ] 최근 24시간 이상 서비스 다운이 없었나?

점검 항목:
```

```bash
# 1. Uptime Kuma 접속
# http://모니터링-서버:3001

# 2. Grafana 접속
# http://모니터링-서버:3000

# 3. 서비스 헬스 체크
curl -s http://localhost:3001/api/health | jq .

# 출력 예시 (정상):
# {
#   "status": "ok",
#   "database": "ok",
#   "storage": "ok",
#   "realtime": "ok",
#   "uptime_hours": 45,
#   "memory_usage_percent": 65
# }
```

### 경보 규칙 점검

```bash
# 1. 설정된 경보 규칙 확인
cat /opt/supabase/monitoring/alerts.yml | grep -E "alert|condition"

# 2. 최근 경보 히스토리
docker logs uptime-kuma --tail=50 | grep -i "alert\|warning"

# 3. 임계값 재검토
# - CPU: 80% 이상
# - 메모리: 85% 이상
# - 디스크: 90% 이상
# - DB 응답시간: 500ms 이상

# 필요시 조정:
vi /opt/supabase/monitoring/alerts.yml
docker-compose restart monitoring
```

### 로그 수집 점검

```bash
# 1. 각 서비스 로그 크기 확인
du -sh /var/lib/docker/containers/*/

# 너무 크면 (> 5GB):
# - 로그 자동 로테이션 설정 확인
# - 불필요한 로그 정리

# 2. 로그 레벨 확인
grep "LOG_LEVEL" /opt/supabase/.env.prod
# INFO 권장 (DEBUG는 용량 소비)

# 3. 중요 에러 로그 확인
for service in postgres api realtime storage; do
  echo "=== $service ==="
  docker logs supabase-$service 2>&1 | grep -i error | tail -3
done
```
