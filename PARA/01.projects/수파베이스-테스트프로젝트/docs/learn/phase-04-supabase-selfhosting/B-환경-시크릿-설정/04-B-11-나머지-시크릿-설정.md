# 04-B-11 나머지 시크릿 설정

> 유형: 실습 | 예상 소요 시간: 15~20분
> 이전: ANON key / SERVICE key 생성 | 다음: 서버에 Docker & Compose 설치 (그룹 C)

---

## 이 파일에서 하는 것

Postgres 슈퍼유저 비밀번호, Studio 관리자 자격증명, SMTP 이메일 설정, 외부 URL 등 `.env` 파일의 나머지 중요 시크릿들을 설정합니다. 이를 통해 데이터베이스 접근, 관리 페이지 로그인, 이메일 발송 기능을 안전하게 구성할 수 있습니다.

---

## 사전 조건

- 04-B-08(필수 변수 정리) 완료
- 04-B-09 또는 04-B-10에서 JWT_SECRET, ANON_KEY, SERVICE_ROLE_KEY 생성 완료
- `.env` 파일이 준비되어 있음 (아직 생성하지 않았다면 이 파일에서 함께 생성)

---

## 실습

### Step 1. Postgres 슈퍼유저 비밀번호 설정

```bash
# 강력한 비밀번호 생성 (최소 12자, 대문자, 숫자, 특수문자 포함)
openssl rand -base64 32
# 출력 예: aB3+xK9mL2pQwRs4tUvWxYz1cDeFgHiJ==
```

`.env` 파일에 추가:
```bash
POSTGRES_PASSWORD=aB3+xK9mL2pQwRs4tUvWxYz1cDeFgHiJ==
POSTGRES_HOST=db
POSTGRES_DB=postgres
POSTGRES_PORT=5432
```

> **POSTGRES_PASSWORD 설명**: 
> - PostgreSQL 데이터베이스 관리자(슈퍼유저)의 암호입니다.
> - 이 비밀번호는 Docker 컨테이너가 처음 시작할 때만 사용됩니다.
> - 나중에 변경하려면 데이터베이스에 직접 접속해서 SQL로 변경해야 합니다.
> - **절대 Git에 커밋하면 안 됩니다.**

---

### Step 2. Supabase Studio 관리자 자격증명 설정

```bash
# Studio 로그인용 사용자명과 비밀번호 설정
# (기본값을 사용하거나 커스텀 값 설정 가능)
```

`.env` 파일에 추가:
```bash
DASHBOARD_USERNAME=admin
DASHBOARD_PASSWORD=aSecure_Dashboard_Pass123!
```

> **설명**:
> - `DASHBOARD_USERNAME`: Supabase Studio(관리 페이지)에 로그인할 때 사용하는 사용자명
> - `DASHBOARD_PASSWORD`: Studio 로그인 비밀번호
> - 처음 배포 후, Studio에 접속해서 이 자격증명으로 로그인합니다.
> - **권장**: 기본값(admin) 대신 고유한 사용자명으로 변경하세요.

---

### Step 3. SMTP 이메일 서버 설정

이메일 발송 기능이 필요한 경우 설정합니다. 개발 환경과 프로덕션 환경을 나누어 설명합니다.

#### 3-1. 개발 환경: Mailhog 사용 (로컬 테스트)

```bash
# Mailhog는 docker-compose에 이미 포함됨 (개발용)
GOTRUE_SMTP_HOST=mailhog
GOTRUE_SMTP_PORT=1025
GOTRUE_SMTP_USER=
GOTRUE_SMTP_PASS=
GOTRUE_SMTP_ADMIN_EMAIL=admin@example.com
GOTRUE_SMTP_FROM_NAME=MyApp
GOTRUE_SMTP_FROM_EMAIL=noreply@example.com
```

> **개발 환경에서 Mailhog 사용:**
> - Mailhog는 실제로 메일을 보내지 않고, 로컬에서 받은 메일을 웹 UI에 표시합니다.
> - Mailhog SMTP는 `localhost:1025`, Web UI는 `http://localhost:8025` 에서 접근 가능
> - 실제 메일은 보내지 않지만, 회원가입 확인 메일, 비밀번호 리셋 메일 등을 시각적으로 확인할 수 있습니다.

#### 3-2. 프로덕션 환경: Gmail SMTP 사용

```bash
# Gmail 앱 비밀번호 방식 (권장)
GOTRUE_SMTP_HOST=smtp.gmail.com
GOTRUE_SMTP_PORT=587
GOTRUE_SMTP_USER=your-email@gmail.com
GOTRUE_SMTP_PASS=abcd wxyz 1234 5678
GOTRUE_SMTP_ADMIN_EMAIL=admin@example.com
GOTRUE_SMTP_FROM_NAME=MyApp
GOTRUE_SMTP_FROM_EMAIL=noreply@example.com
```

> **Gmail 앱 비밀번호 획득 방법:**
> 1. Google 계정 보안 설정 (https://myaccount.google.com/security) 접속
> 2. "2단계 인증" 활성화 필수
> 3. "앱 비밀번호" 섹션에서 "Mail" > "Windows 컴퓨터" 선택
> 4. 생성된 16자 비밀번호를 `GOTRUE_SMTP_PASS`에 입력
> 5. **절대 실제 Google 계정 비밀번호를 사용하면 안 됩니다.**

#### 3-3. 다른 SMTP 제공자 (예: SendGrid, AWS SES)

```bash
# SendGrid 예시
GOTRUE_SMTP_HOST=smtp.sendgrid.net
GOTRUE_SMTP_PORT=587
GOTRUE_SMTP_USER=apikey
GOTRUE_SMTP_PASS=SG.your-sendgrid-api-key-here
GOTRUE_SMTP_ADMIN_EMAIL=admin@example.com
GOTRUE_SMTP_FROM_NAME=MyApp
GOTRUE_SMTP_FROM_EMAIL=noreply@example.com
```

### Step 4. 외부 URL 설정

클라이언트가 접근할 Supabase API 주소와 웹 애플리케이션 주소를 설정합니다.

```bash
# 로컬 개발 환경
API_EXTERNAL_URL=http://localhost:8000
SUPABASE_PUBLIC_URL=http://localhost:8000
SITE_URL=http://localhost:3000
STUDIO_URL=http://localhost:3001
```

```bash
# 또는 실제 도메인 (프로덕션)
API_EXTERNAL_URL=https://api.example.com
SUPABASE_PUBLIC_URL=https://api.example.com
SITE_URL=https://example.com
STUDIO_URL=https://studio.example.com
```

> **각 URL의 역할:**
> - `API_EXTERNAL_URL`: 클라이언트가 Supabase API를 호출할 주소 (중요!)
> - `SUPABASE_PUBLIC_URL`: `API_EXTERNAL_URL`과 동일하게 설정
> - `SITE_URL`: 이메일 링크, OAuth 리다이렉트 등에서 사용되는 서비스 도메인
> - `STUDIO_URL`: Supabase Studio 관리 페이지 접근 주소
> - **슬래시(/) 없이** 마지막에 포트 번호까지만 작성합니다.

---

### Step 5. 선택적 보안 설정

필요한 경우 추가로 설정:

```bash
# 회원가입 자동 승인 (개발 환경에서만 true)
GOTRUE_DISABLE_SIGNUP=false

# 이메일 확인 필수 여부 (프로덕션은 true 권장)
GOTRUE_EMAIL_AUTOCONFIRM=false

# CORS 허용 도메인 (쉼표로 구분)
GOTRUE_SITE_URL=http://localhost:3000
API_CORS_ALLOW_ORIGINS=http://localhost:3000,https://example.com
```

### Step 6. .env 파일 전체 검증 체크리스트

지금까지 추가한 모든 값을 확인하세요:

```bash
# 파일 전체 내용 확인
cat .env
```

아래 체크리스트를 따라 검증:

| 항목 | 확인 사항 | 상태 |
|------|---------|------|
| **POSTGRES_PASSWORD** | 최소 12자, 특수문자 포함, 무작위 생성됨 | ☐ |
| **DASHBOARD_USERNAME** | admin이 아닌 고유한 값으로 변경 (권장) | ☐ |
| **DASHBOARD_PASSWORD** | 최소 8자 이상, 특수문자 포함 | ☐ |
| **SMTP 설정** | 개발/프로덕션 환경에 맞춰 설정됨 | ☐ |
| **API_EXTERNAL_URL** | 슬래시(/)로 끝나지 않음, 올바른 도메인/포트 | ☐ |
| **SITE_URL** | 클라이언트 앱이 실행될 주소 | ☐ |
| **STUDIO_URL** | Supabase Studio 접근 주소 | ☐ |
| **JWT_SECRET** | 04-B-09/10에서 생성한 값 포함 | ☐ |
| **ANON_KEY** | JWT 형식, 유효한 값 | ☐ |
| **SERVICE_ROLE_KEY** | JWT 형식, ANON_KEY와 다른 값 | ☐ |
| **파일 권한** | Git에 `.env` 미포함 (.gitignore 확인) | ☐ |

## 확인 방법

### 1. .env 파일 문법 확인

```bash
# .env 파일이 올바르게 작성되었는지 확인
grep -E "^[A-Z_]+=" .env | wc -l
# 최소 20개 이상의 변수가 설정되어야 합니다.
```

### 2. 필수 변수 모두 설정되었는지 확인

```bash
# 반드시 필요한 변수들이 모두 있는지 확인
for var in POSTGRES_PASSWORD JWT_SECRET ANON_KEY SERVICE_ROLE_KEY API_EXTERNAL_URL DASHBOARD_USERNAME DASHBOARD_PASSWORD; do
  if grep -q "^$var=" .env; then
    echo "✓ $var 설정됨"
  else
    echo "✗ $var 미설정"
  fi
done
```

### 3. 특수문자 이스케이프 확인

```bash
# 비밀번호에 특수문자가 있으면 따옴표로 감싸야 합니다.
# 확인: POSTGRES_PASSWORD에 $ 또는 \ 같은 특수문자가 있으면 "값" 형식으로 변경
grep "POSTGRES_PASSWORD" .env
# 예시: POSTGRES_PASSWORD="aB3+xK9mL2pQwRs4tUvWxYz1cDeFgHiJ==" (O)
#      또는 POSTGRES_PASSWORD=aB3+xK9mL2pQwRs4tUvWxYz1cDeFgHiJ== (O)
```

### 4. URL 형식 확인

```bash
# URL이 슬래시로 끝나지 않는지 확인
grep -E "URL=" .env | grep -v "/$"
# 결과: API_EXTERNAL_URL=http://localhost:8000 (O)
#      SITE_URL=http://localhost:3000 (O)
```

## 자주 겪는 오류

| 오류 메시지 | 원인 | 해결 |
|---|---|---|
| `docker-compose: command not found` | docker-compose가 설치되지 않음 | 다음 파일(그룹 C)에서 설치 예정 |
| `POSTGRES_PASSWORD: permission denied` | 비밀번호에 $를 포함할 때 shell 변수로 인식됨 | `POSTGRES_PASSWORD="비밀번호"` (따옴표 추가) |
| `SMTP 연결 실패` | SMTP_HOST, PORT, 자격증명 오류 | 설정값 재확인, 개발시 Mailhog 사용 권장 |
| `Studio 로그인 실패` | DASHBOARD_USERNAME/PASSWORD 오류 | `.env` 파일 재확인 후 컨테이너 재시작 필요 |
| `API_EXTERNAL_URL 오류` | 끝에 슬래시(/) 포함 | `https://api.example.com/` → `https://api.example.com` 변경 |
| `이메일 주소 형식 오류` | GOTRUE_SMTP_FROM_EMAIL이 유효하지 않은 형식 | `noreply@example.com` 형식 확인 |
| `ANON_KEY/SERVICE_ROLE_KEY 불일치` | JWT 토큰 검증 실패 | 04-B-09/10에서 올바르게 생성되었는지 재확인 |
