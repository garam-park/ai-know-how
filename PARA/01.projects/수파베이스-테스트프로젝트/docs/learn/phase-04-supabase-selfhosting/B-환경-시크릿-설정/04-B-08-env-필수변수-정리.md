# 04-B-08 .env 파일 필수 변수 목록 정리

> 유형: 개념 | 예상 소요 시간: 10분
> 이전: 공식 docker-compose.yml 전체 구조 분석 | 다음: JWT Secret 생성 및 설정

---

## 이 파일에서 배우는 것

Supabase 셀프호스팅에서 docker-compose.yml이 참조하는 .env 파일의 모든 필수 환경변수를 이해합니다. 어떤 변수는 반드시 변경해야 보안상 안전하고, 어떤 변수는 개발 단계에서 기본값으로 둬도 되는지 구분할 수 있게 됩니다.

---

## 본문

### .env 파일의 역할

FE 개발할 때 `.env.local` 파일에서 API 엔드포인트를 설정하는 것처럼, Supabase 셀프호스팅도 `.env` 파일에서 모든 서비스의 설정을 한곳에서 관리합니다. 다만 차이점은:

- **FE**: `NEXT_PUBLIC_` 변수는 클라이언트에 노출되어도 되고, 그 외 변수는 서버에서만 사용
- **Supabase**: `.env`의 모든 변수는 서버(docker-compose) 내부에서만 사용되며, 절대 외부에 노출되면 안 됨

### 필수 변수 카테고리별 정리

#### 1. **기본 설정 (Core Configuration)**

| 변수명 | 설명 | 보안 | 수정 필요 |
|--------|------|------|----------|
| `POSTGRES_PASSWORD` | PostgreSQL 데이터베이스 관리자 암호 | 🔴 필수 변경 | ✅ 반드시 변경 |
| `POSTGRES_HOST` | PostgreSQL 데이터베이스 호스트 | 🟡 낮음 | ❌ 기본값 사용 |
| `POSTGRES_DB` | 생성될 데이터베이스 이름 | 🟡 낮음 | ❌ 기본값 사용 |
| `POSTGRES_PORT` | PostgreSQL 포트 | 🟡 낮음 | ❌ 기본값 사용 |

**예시:**
```bash
POSTGRES_PASSWORD=your-super-secure-password-here-min-12-chars
POSTGRES_HOST=db
POSTGRES_DB=postgres
POSTGRES_PORT=5432
```

#### 2. **JWT 인증 (JSON Web Token)**

| 변수명 | 설명 | 보안 | 수정 필요 |
|--------|------|------|----------|
| `JWT_SECRET` | JWT 토큰 서명용 시크릿 키 | 🔴 필수 변경 | ✅ 반드시 변경 |
| `JWT_EXPIRY` | JWT 토큰 만료 시간 (초) | 🟡 낮음 | ❌ 기본값 사용 |
| `REFRESH_TOKEN_ROTATION_ENABLED` | 리프레시 토큰 자동 갱신 | 🟡 낮음 | ❌ 기본값 사용 |

**설명:**
- `JWT_SECRET`: 사용자 로그인 시 발급되는 토큰을 암호화하는 열쇠. 이 값이 유출되면 누구나 가짜 토큰을 만들 수 있음
- 최소 32자 이상의 랜덤 문자열이어야 함

**예시:**
```bash
JWT_SECRET=your-random-32-char-secret-key-change-this-value
JWT_EXPIRY=3600
REFRESH_TOKEN_ROTATION_ENABLED=true
```

#### 3. **API 키 (Anon & Service Role)**

| 변수명 | 설명 | 보안 | 수정 필요 |
|--------|------|------|----------|
| `ANON_KEY` | 익명(비로그인) 클라이언트 API 키 | 🟡 중간 | ✅ 권장 변경 |
| `SERVICE_ROLE_KEY` | 관리자 권한 API 키 | 🔴 필수 변경 | ✅ 반드시 변경 |

**설명:**
- `ANON_KEY`: FE에서 로그인 없이 사용하는 키. 노출되어도 제한된 권한만 가짐
- `SERVICE_ROLE_KEY`: 백엔드에서만 사용하는 관리자 키. 절대 FE에 노출되면 안 됨

**예시:**
```bash
ANON_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
SERVICE_ROLE_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
```

#### 4. **이메일 설정 (SMTP)**

| 변수명 | 설명 | 보안 | 수정 필요 |
|--------|------|------|----------|
| `SMTP_HOST` | SMTP 서버 주소 | 🟡 낮음 | ✅ 필요시 변경 |
| `SMTP_PORT` | SMTP 포트 | 🟡 낮음 | ❌ 기본값 사용 |
| `SMTP_USER` | SMTP 사용자 (발신 이메일) | 🟡 중간 | ✅ 필요시 변경 |
| `SMTP_PASS` | SMTP 비밀번호 | 🔴 필수 변경 | ✅ 필요시 변경 |
| `SMTP_ADMIN_EMAIL` | 관리자 이메일 | 🟡 낮음 | ✅ 필요시 변경 |
| `SMTP_FROM_NAME` | 발신 이름 | 🟡 낮음 | ❌ 기본값 사용 |

**설명:**
- 회원가입 확인 메일, 비밀번호 리셋 메일 등을 보내기 위한 설정
- 개발 단계에서는 테스트 SMTP(예: Mailhog) 사용 가능

**예시:**
```bash
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_USER=your-email@gmail.com
SMTP_PASS=your-app-password-not-real-password
SMTP_ADMIN_EMAIL=admin@example.com
SMTP_FROM_NAME=MyApp
```

#### 5. **API 엔드포인트 (URLs)**

| 변수명 | 설명 | 보안 | 수정 필요 |
|--------|------|------|----------|
| `API_EXTERNAL_URL` | 클라이언트가 접근할 Supabase 주소 | 🟡 낮음 | ✅ 반드시 변경 |
| `SITE_URL` | 웹 애플리케이션 주소 | 🟡 낮음 | ✅ 반드시 변경 |
| `STUDIO_URL` | Supabase Studio 관리 페이지 주소 | 🟡 낮음 | ✅ 반드시 변경 |

**설명:**
- `API_EXTERNAL_URL`: 클라이언트가 FE에서 `supabaseClient.from(...)`할 때 요청보낼 주소
- `SITE_URL`: 이메일 링크 등에서 사용되는 서비스 도메인
- `STUDIO_URL`: 데이터베이스 관리 UI에 접근할 주소

**예시:**
```bash
API_EXTERNAL_URL=https://api.example.com
SITE_URL=https://example.com
STUDIO_URL=https://studio.example.com
```

#### 6. **Redis 설정**

| 변수명 | 설명 | 보안 | 수정 필요 |
|--------|------|------|----------|
| `REDIS_URL` | Redis 연결 문자열 | 🟡 중간 | ❌ 기본값 사용 |

**설명:**
- 캐시, 세션 관리, 실시간 기능에 사용
- 로컬 개발 환경에서는 기본값(redis:6379)으로 충분

**예시:**
```bash
REDIS_URL=redis://redis:6379
```

#### 7. **로깅 및 모니터링**

| 변수명 | 설명 | 보안 | 수정 필요 |
|--------|------|------|----------|
| `LOG_LEVEL` | 로그 레벨 (debug, info, warn, error) | 🟡 낮음 | ❌ 기본값 사용 |

**설명:**
- 개발 중: `debug` 사용하여 상세 로그 확인
- 프로덕션: `info` 또는 `warn`으로 설정하여 로그 양 제어

**예시:**
```bash
LOG_LEVEL=info
```

### 보안 수준별 변경 필수도

```
🔴 필수 변경 (프로덕션 배포 전 반드시)
  ├─ POSTGRES_PASSWORD
  ├─ JWT_SECRET
  ├─ SERVICE_ROLE_KEY
  ├─ API_EXTERNAL_URL
  ├─ SITE_URL
  └─ STUDIO_URL

🟡 권장 변경 (환경에 맞게)
  ├─ ANON_KEY
  ├─ SMTP_HOST
  ├─ SMTP_USER
  ├─ SMTP_PASS
  ├─ SMTP_ADMIN_EMAIL
  └─ REDIS_URL

❌ 그대로 둬도 됨 (기본값 OK)
  ├─ POSTGRES_HOST
  ├─ POSTGRES_DB
  ├─ POSTGRES_PORT
  ├─ JWT_EXPIRY
  ├─ LOG_LEVEL
  └─ 기타 옵션 변수들
```

### FE 개발자 관점에서의 비유

```
FE .env.local 관리          vs    Supabase .env 관리
─────────────────────────        ───────────────────
NEXT_PUBLIC_API_KEY (노출)        ANON_KEY (클라이언트용)
  ↓                                ↓
API_SECRET_KEY (숨김)              SERVICE_ROLE_KEY (백엔드용)
  ↓                                ↓
.env.local에 저장 (로컬)            .env에 저장 (서버 내부)
  ↓                                ↓
Git에 커밋 X                       Git에 커밋 X (반드시!)
```

## 핵심 정리

- **.env 파일은 Supabase 서버의 모든 설정을 담는 중앙 집중식 구성 파일**로, docker-compose.yml에서 참조함

- **보안 관련 변수(POSTGRES_PASSWORD, JWT_SECRET, SERVICE_ROLE_KEY, SMTP_PASS)는 반드시 강력한 임의의 값으로 변경**하고, Git에 절대 커밋하면 안 됨

- **URL 관련 변수(API_EXTERNAL_URL, SITE_URL, STUDIO_URL)는 배포 환경에 맞춰 변경**해야 클라이언트가 올바른 주소로 접근 가능

- **ANON_KEY와 SERVICE_ROLE_KEY는 다르게 생성되는데**, 전자는 FE에서 사용 가능하고 후자는 백엔드 전용이므로 혼동 금지
