# 핸드오프 문서 — Supabase 셀프호스팅 학습 MD 제작

> 이 문서는 학습 MD 파일을 제작할 에이전트에게 전달하는 컨텍스트 문서입니다.  
> 아래 내용을 모두 숙지한 뒤 작업을 시작해주세요.

---

## 1. 작업 배경

- 사용자는 **FE만 경험한 프론트엔드 개발자**입니다.
- 목표는 Supabase를 클라우드(Supabase Cloud)가 아닌 **직접 서버에 셀프호스팅**하고 운영하는 DevOps 역량을 갖추는 것입니다.
- 학습 방식은 **10분 단위로 읽고 실습할 수 있는 MD 파일** 형태입니다.

---

## 2. 학습 범위

전체 로드맵 중 아래 3개 Phase만 다룹니다. Phase 1(Linux), Phase 2(Docker), Phase 3(서버 준비)는 **이미 사전 학습 완료**로 가정합니다.

| Phase | 주제 |
|---|---|
| Phase 4 | Supabase 셀프호스팅 배포 |
| Phase 5 | PostgreSQL 운영 기초 |
| Phase 6 | CI/CD & 운영 자동화 |

## 4. MD 파일 작성 규칙

각 파일은 아래 구조를 반드시 따릅니다.

### 개념 파일 템플릿

```markdown
# [파일 번호] 제목

> 유형: 개념 | 예상 소요 시간: 10분  
> 이전: [이전 파일 제목] | 다음: [다음 파일 제목]

---

## 이 파일에서 배우는 것

(2~3줄 이내로 핵심만)

---

## 본문

(설명, 다이어그램 대용 텍스트, 예시 등)

---

## 핵심 정리

- 포인트 1
- 포인트 2
- 포인트 3

---

## 다음 파일 예고

(다음에 뭘 하는지 한 줄)
```

### 실습 파일 템플릿

```markdown
# [파일 번호] 제목

> 유형: 실습 | 예상 소요 시간: 15~20분  
> 이전: [이전 파일 제목] | 다음: [다음 파일 제목]

---

## 이 파일에서 하는 것

(2~3줄 이내로 핵심만)

---

## 사전 조건

- 필요한 환경 또는 앞서 완료해야 할 것

---

## 실습

### Step 1. 제목
\```bash
# 명령어
\```
> 설명

### Step 2. 제목
\```bash
# 명령어
\```
> 설명

---

## 확인 방법

(실습이 성공했는지 검증하는 방법)

---

## 자주 겪는 오류

| 오류 메시지 | 원인 | 해결 |
|---|---|---|
| ... | ... | ... |

---

(다음에 뭘 하는지 한 줄)
```

(트러블슈팅 목록, 체크리스트 등)
```

## 5. 작성 시 주의사항

- 독자는 **JS/TS는 능숙하지만 서버/DB/DevOps는 처음**인 FE 개발자입니다.
- CLI 명령어는 **복사해서 바로 쓸 수 있도록** 코드 블록으로 작성합니다.
- 개념 설명 시 **FE 개발 경험에 비유**하면 이해가 빠릅니다.  
  예) "Kong은 Express의 미들웨어 라우터와 비슷한 역할입니다."
- 각 파일은 **독립적으로 읽을 수 있어야** 합니다. 앞 파일을 안 봐도 최소한 맥락을 알 수 있도록 작성합니다.
- 파일 상단의 `이전 / 다음` 링크는 **같은 Phase 내** 흐름만 연결합니다.
- 불필요한 이론 깊이는 피하고 **"이걸 알아야 실습이 된다"** 수준까지만 설명합니다.

## 7. 작업 시작 방법 (에이전트에게)

1. 이 문서를 모두 읽고 전체 구조를 파악합니다.
2. 사용자에게 **어느 Phase, 어느 그룹부터** 시작할지 확인합니다.
3. 그룹 단위로 묶어서 작업합니다 (한 번에 한 그룹).
