# 위험 관리 대장

> 제주 스마트도시 데이터허브 시범솔루션 발굴사업
> 컨소시엄 협업 관련 리스크 및 대응 방안
> 작성일: 2026-04-05

## 위험 관리 매트릭스

### R001 — 디토닉 인력 최소화 및 범위 과다 위임

| 항목               | 내용                                                                                                                                                             |
| ------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **위험 설명**      | 디토닉이 자체 인력을 최소화하고 파트너사(특히 이노팸)에 더 많은 구현 범위를 위임할 가능성                                                                        |
| **영향도**         | 상 (일정 지연, 품질 저하, 과다 업무)                                                                                                                             |
| **발생 확률**      | 중~상                                                                                                                                                            |
| **담당**           | 이노팸                                                                                                                                                           |
| **대응 방안**      | (1) 초기에 모듈별 책임 범위를 산출물 단위로 문서화 및 합의, (2) 디토닉 리딩 하에 공동 수행이라는 원칙을 보고서에 명시, (3) 추가 범위 발생 시 변경 관리 절차 적용 |
| **조기 경보 지표** | 회의에서 디토닉 인력 참석률 저하, 이노팸 담당 이슈 급증                                                                                                          |

| 항목               | 내용                                                                                                                                                                                                                                            |
| ------------------ | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **위험 설명**      | "LLM 일부"의 경계가 모호하여 추후 분쟁 소지                                                                                                                                                                                                     |
| **영향도**         | 상 (범위 크리프, 일정 영향)                                                                                                                                                                                                                     |
| **발생 확률**      | 중                                                                                                                                                                                                                                              |
| **담당**           | 이노팸                                                                                                                                                                                                                                          |
| **대응 방안**      | (1) LLM 세부 모듈(요약/분류/검색/질의응답/클러스터링)별 담당사를 표로 명시, (2) 이노팸 담당: 시민용 요약, 부서별 요약, LLM 분류, LLM 검색, AI Assistant 프론트엔드, (3) 디토닉 담당: 프롬프트 템플릿, 리라이팅, 벡터 인덱스, 리랭킹, 클러스터링 |
| **조기 경보 지표** | 모듈 간 인터페이스 정의 지연, 담당 불명확 이슈 발생                                                                                                                                                                                             |
