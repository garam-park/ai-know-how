## 1. 에이전트 개념 이해

OpenCode는 **목적별로 분리된 에이전트** 시스템을 씁니다.

### 내장 에이전트 3종

| 에이전트   | 전환 방법 | 파일 수정 | 주 사용 상황               |
| ---------- | --------- | --------- | -------------------------- |
| **build**  | `Tab`     | ✅ 가능   | 실제 코드 작성 / 버그 수정 |
| **plan**   | `Tab`     | ❌ 불가   | 기능 설계 / 리스크 분석    |
| **review** | 수동 정의 | ❌ 불가   | 코드 리뷰 / 품질 검토      |

> 💡 **핵심 원칙**: AI가 코드를 바로 건드리게 하지 말고, 먼저 **plan** 으로 계획을 검토한 뒤 **build** 로 실행하세요.

## 2. Plan → Build 기본 워크플로우

### Step 1: Plan 에이전트로 설계 요청

`Tab` 키로 **plan 모드**로 전환 후:

````
사용자가 노트를 삭제하면 실제로 지우지 않고 DB에 deleted 플래그를 세우는 기능을 추가하고 싶어.
"최근 삭제된 항목" 화면도 필요하고, 거기서 복원하거나 영구 삭제할 수 있어야 해.
기존 settings 페이지와 동일한 디자인 패턴을 써줘.

어떻게 구현할지 계획을 세워줘.
```

AI가 변경 없이 아래 내용을 분석해 줍니다:
- 영향받는 파일 목록
- 구현 순서
- 잠재적 리스크
- 예상 작업량

### Step 2: 계획 검토 & 피드백

계획을 읽고 문제가 있으면 수정 요청합니다.

```
users 테이블 대신 audit_log 테이블을 따로 만드는 건 어때?
그리고 복원 기능에 권한 체크도 추가해줘.
```

### Step 3: Build 에이전트로 실행

`Tab` 키로 **build 모드**로 전환 후:

```
방금 계획한 대로 구현해줘.
```

## 3. 실전 워크플로우 예시

### 예시 A: 새 API 엔드포인트 추가

```bash
# 1. Plan 모드에서
"GET /api/users/:id/activity 엔드포인트를 추가하려고 해.
 인증 미들웨어를 거치고, 최근 30일 활동 로그를 페이지네이션으로 반환해야 해.
 어떤 파일을 수정해야 하고 어떤 순서로 작업해야 해?"

# 2. 검토 후 Build 모드에서
"계획대로 구현해줘. 테스트 파일도 같이 만들어줘."
```

### 예시 B: 버그 원인 분석 후 수정

```bash
# 1. Plan 모드에서
"@src/payment/checkout.ts 이 파일에서 할인율 계산이 가끔 음수가 나오는 버그가 있어.
 원인이 뭔지 분석하고 어떻게 고칠지 설명해줘."

# 2. Build 모드에서
"분석한 대로 버그를 수정하고, 해당 케이스를 커버하는 테스트도 추가해줘."
```

### 예시 C: 리팩토링 계획

```bash
# 1. Plan 모드에서
"@src/services/ 이 폴더 전체를 보고,
 중복 코드나 개선할 수 있는 부분을 찾아줘.
 어떤 순서로 리팩토링하면 좋을지 우선순위를 매겨줘."
```

## 4. 커스텀 에이전트 만들기

### 마크다운 파일로 에이전트 정의

프로젝트 루트에 `.opencode/agents/` 폴더를 만들고 `.md` 파일로 에이전트를 정의합니다.

#### 예시: 코드 리뷰 전용 에이전트

```markdown
<!-- .opencode/agents/review.md -->
---
description: PR 코드 리뷰 전용 에이전트. 변경 없이 피드백만 제공.
model: claude-sonnet-4-20250514
temperature: 0.3
permissions:
  - read
---

당신은 시니어 코드 리뷰어입니다.

## 리뷰 기준
- 버그 가능성
- 보안 취약점
- 성능 이슈
- 코드 가독성
- 테스트 커버리지

## 규칙
- 직접 코드를 수정하지 마세요
- 구체적인 개선 방법을 제안하세요
- 잘 된 부분도 언급하세요
```

#### 예시: 문서화 에이전트

```markdown
<!-- .opencode/agents/docs.md -->
---
description: 코드를 읽고 JSDoc / 주석을 자동으로 작성하는 에이전트
model: claude-sonnet-4-20250514
permissions:
  - read
  - write
---

당신은 기술 문서 작성 전문가입니다.
함수, 클래스, 모듈에 명확하고 유용한 JSDoc 주석을 작성하세요.
한국어와 영어를 혼용하지 말고 일관된 언어를 사용하세요.
```

## 5. 에이전트 모드(mode) 설정

```markdown
---
mode: primary    # 사용자가 직접 선택 가능
mode: subagent   # 다른 에이전트가 호출할 때만 사용
mode: all        # 둘 다 가능 (기본값)
hidden: true     # @자동완성 메뉴에서 숨김
---
```

## 6. 실습 과제

**과제 1**: 현재 작업 중인 프로젝트에서 추가하고 싶은 기능 하나를 골라, Plan → Build 순서로 작업해보세요.

**과제 2**: `.opencode/agents/review.md` 파일을 만들고, 본인 프로젝트 스타일에 맞는 코드 리뷰 에이전트를 정의하세요.

**과제 3**: Plan 에이전트한테 현재 코드베이스에서 가장 개선이 필요한 부분 Top 3를 물어보세요.

```
이 프로젝트에서 기술 부채가 가장 심한 부분 3가지를 찾아서
우선순위와 이유를 함께 설명해줘.
```

## ✅ 2단계 완료 기준

- Plan 에이전트로 구현 계획을 받고, 검토 후 Build로 전환하는 흐름이 자연스럽다
- 커스텀 에이전트 `.md` 파일을 직접 만들 수 있다
- AI가 제안한 계획을 무비판적으로 실행하지 않고, 피드백을 주고받을 수 있다
````
