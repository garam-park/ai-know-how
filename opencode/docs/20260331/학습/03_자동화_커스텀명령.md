## 1. 비대화형 모드 (Non-Interactive Mode)

TUI를 열지 않고 **프롬프트를 바로 전달하고 결과를 받는** 모드입니다.  
스크립팅, 자동화, 빠른 1회성 질문에 적합합니다.

### 기본 사용법

```bash
# 기본 — 결과를 터미널에 출력
opencode -p "이 프로젝트에서 사용 중인 외부 라이브러리 목록을 뽑아줘"

# JSON 포맷으로 출력 (파싱에 유리)
opencode -p "package.json의 dependencies를 정리해줘" -f json

# 스피너(로딩 애니메이션) 없이 실행 (스크립트용)
opencode -p "README.md 초안을 작성해줘" -q

# 특정 디렉터리를 작업 경로로 지정
opencode -c /path/to/project -p "테스트 파일이 없는 함수를 찾아줘"
```

> ⚠️ 비대화형 모드에서는 **모든 권한이 자동 승인**됩니다. 파일 수정 작업은 주의해서 사용하세요.

## 2. 출력 포맷 활용

### 텍스트 출력 (기본)

```bash
opencode -p "CHANGELOG.md를 최신 커밋 기반으로 업데이트해줘" > CHANGELOG.md
```

### JSON 출력

```bash
# JSON 결과를 jq로 파싱
opencode -p "버그가 있을 가능성이 높은 함수 5개를 JSON 배열로 반환해줘" -f json | jq '.[]'
```

## 3. 커스텀 명령 (Custom Commands)

반복적으로 사용하는 프롬프트를 **마크다운 파일**로 저장해 명령어로 등록합니다.

### 저장 위치

```
~/.config/opencode/commands/          # 전역 (모든 프로젝트에서 사용)
.opencode/commands/                   # 프로젝트별 (해당 프로젝트에서만 사용)
```

### 커스텀 명령 만들기 예시

#### 예시 1: 컨텍스트 초기화 명령

```markdown
<!-- ~/.config/opencode/commands/init-context.md -->
이 프로젝트의 다음 내용을 분석하고 요약해줘:

1. 주요 기술 스택과 의존성
2. 핵심 디렉터리 구조 설명
3. 진입점(entry point) 파일 위치
4. 주요 비즈니스 로직이 어디 있는지

이후 작업을 위해 컨텍스트를 파악하는 중이야.
```

TUI에서 `/user:init-context` 로 실행합니다.

#### 예시 2: PR 설명 자동 생성

```markdown
<!-- .opencode/commands/pr-description.md -->
최근 변경된 파일들을 확인하고, 아래 형식으로 PR 설명을 작성해줘:

## 변경 요약
(한 줄 요약)

## 변경 내용
- (주요 변경사항 목록)

## 테스트 방법
(검증 방법)

## 체크리스트
- [ ] 테스트 추가/수정
- [ ] 문서 업데이트
- [ ] Breaking change 없음
```

#### 예시 3: 인자(argument) 사용하기

```markdown
<!-- .opencode/commands/explain-func.md -->
$FUNC_NAME 함수에 대해 다음을 설명해줘:
1. 이 함수가 하는 일
2. 파라미터 설명
3. 반환값
4. 사용 예시
5. 주의사항이나 엣지 케이스
```

```bash
# TUI에서 사용
/user:explain-func FUNC_NAME=calculateDiscount
```

## 4. 셸 스크립트 자동화 레시피

### 레시피 A: 커밋 전 코드 리뷰 자동화

```bash
#!/bin/bash
# pre-commit-review.sh

CHANGED_FILES=$(git diff --cached --name-only --diff-filter=ACM | grep -E '\.(ts|js|py)$')

if [ -z "$CHANGED_FILES" ]; then
  echo "변경된 파일 없음. 리뷰 건너뜀."
  exit 0
fi

echo "=== AI 코드 리뷰 시작 ==="
echo "$CHANGED_FILES" | while read file; do
  echo "--- 리뷰 중: $file ---"
  opencode -q -p "$(cat $file) 이 코드에서 버그, 보안 취약점, 개선점을 간단히 지적해줘."
done
echo "=== 리뷰 완료 ==="
```

```bash
chmod +x pre-commit-review.sh
# .git/hooks/pre-commit 에 연결하거나 수동 실행
```

### 레시피 B: 테스트 없는 파일 탐지 & 테스트 생성

```bash
#!/bin/bash
# generate-missing-tests.sh

# 테스트 파일이 없는 소스 파일 목록 추출
UNTESTED=$(find src -name "*.ts" ! -name "*.test.ts" ! -name "*.spec.ts")

for file in $UNTESTED; do
  test_file="${file/.ts/.test.ts}"
  if [ ! -f "$test_file" ]; then
    echo "테스트 생성 중: $file"
    opencode -q -p "@$file 이 파일의 주요 함수에 대한 단위 테스트를 Jest로 작성하고 $test_file 에 저장해줘."
  fi
done
```

### 레시피 C: 일일 코드 품질 리포트

```bash
#!/bin/bash
# daily-quality-report.sh

DATE=$(date +%Y-%m-%d)
REPORT_FILE="reports/quality-$DATE.md"

mkdir -p reports
opencode -q -p "
이 프로젝트 전체를 스캔하고 다음을 포함한 일일 코드 품질 리포트를 작성해줘:

1. 복잡도가 높은 함수 Top 5
2. TODO/FIXME 주석 목록
3. 중복 코드 패턴
4. 타입 안정성 이슈 (TypeScript 기준)
5. 권장 리팩토링 우선순위

마크다운 형식으로 작성해줘.
" > "$REPORT_FILE"

echo "리포트 생성 완료: $REPORT_FILE"
```

## 5. GitHub Actions 통합

### PR에 자동 코드 리뷰 달기

```yaml
# .github/workflows/ai-review.yml
name: AI Code Review

on:
  pull_request:
    types: [opened, synchronize]

jobs:
  review:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0

      - name: Install OpenCode
        run: curl -fsSL https://opencode.ai/install | bash

      - name: Run AI Review
        env:
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
        run: |
          DIFF=$(git diff origin/main...HEAD -- '*.ts' '*.js' '*.py')
          REVIEW=$(opencode -q -p "다음 diff를 리뷰해줘. 버그, 보안, 성능 이슈를 중심으로 간결하게 작성해줘:

          $DIFF")
          echo "## 🤖 AI 코드 리뷰" > review.md
          echo "$REVIEW" >> review.md

      - name: Comment PR
        uses: actions/github-script@v7
        with:
          script: |
            const fs = require('fs');
            const review = fs.readFileSync('review.md', 'utf8');
            github.rest.issues.createComment({
              issue_number: context.issue.number,
              owner: context.repo.owner,
              repo: context.repo.repo,
              body: review
            });
```
