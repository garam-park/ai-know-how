````text
## 2. 6가지 핵심 아키텍처 원칙

### 원칙 1: 이벤트 드리븐 설계 (Event-Driven Design)

모든 컴포넌트는 **타입이 지정된 이벤트 버스**를 통해 통신합니다.
직접 함수 호출 대신 이벤트를 발행(publish)하고 구독(subscribe)합니다.

```go
// OpenCode 소스코드 패턴 (Go)
type Event struct {
    Type    EventType
    Payload interface{}
}

// 이벤트 발행
bus.Publish(Event{Type: FileChanged, Payload: filePath})

// 이벤트 구독
bus.Subscribe(FileChanged, func(e Event) {
    // 파일 변경에 반응
})
```text

**왜 이렇게?** 컴포넌트 간 결합도를 낮춰, 하나를 바꿔도 다른 것이 깨지지 않습니다.

---

### 원칙 2: 구조화된 메모리 (Structured Memory)

단순히 대화를 텍스트로 저장하는 것이 아니라, **SQLite에 구조화된 형태**로 저장합니다.

```sql
-- 세션 테이블
CREATE TABLE sessions (
  id TEXT PRIMARY KEY,
  project_path TEXT,
  agent_id TEXT,
  created_at DATETIME,
  context_summary TEXT  -- 컨텍스트 압축 요약
);

-- 메시지 테이블
CREATE TABLE messages (
  id TEXT PRIMARY KEY,
  session_id TEXT,
  role TEXT,         -- user / assistant / tool
  content TEXT,
  tool_calls JSON,   -- 사용된 도구 정보
  created_at DATETIME
);
```text

**왜 이렇게?** 단순 텍스트 저장보다 검색, 필터링, 압축이 효율적이고, 컨텍스트 윈도우 관리가 쉽습니다.

---

### 원칙 3: 권한 시스템 (Permission System)

에이전트가 할 수 있는 행동을 **화이트리스트 방식으로 명시적으로 제한**합니다.

```markdown
<!-- 에이전트 권한 설정 예시 -->
---
permissions:
  - read                    # 파일 읽기
  - write: "src/**/*.ts"    # src 폴더 ts 파일만 쓰기
  - bash: "npm test"        # npm test 명령만 허용
  - bash: "git *"           # git 명령 전체 허용
  - deny: "rm -rf"          # 이 명령 절대 금지
---
```text

**왜 이렇게?** 자율성과 안전성을 동시에 확보합니다. AI가 실수로 중요한 파일을 삭제하는 사고를 방지합니다.

---

### 원칙 4: 스냅샷 시스템 (Snapshot System)

AI가 파일을 수정하기 전에 **자동으로 스냅샷을 생성**합니다.

```text
작업 전: 스냅샷 저장
    ↓
AI가 파일 수정
    ↓
결과 검토
    ↓
마음에 안 들면 → 스냅샷으로 롤백
마음에 들면  → 스냅샷 삭제
```text

**왜 이렇게?** AI 에이전트의 자율적 행동에 대한 안전망(undo)을 제공합니다.

---

### 원칙 5: 프로바이더 추상화 (Provider Abstraction)

모든 LLM 제공자를 **동일한 인터페이스로 래핑**합니다.

```go
// 추상화 인터페이스
type Provider interface {
    Complete(ctx context.Context, req CompletionRequest) (*CompletionResponse, error)
    Stream(ctx context.Context, req CompletionRequest) (<-chan StreamChunk, error)
}

// Claude, GPT, Gemini 모두 같은 인터페이스로 교체 가능
var p Provider = NewAnthropicProvider(apiKey)
// var p Provider = NewOpenAIProvider(apiKey)
// var p Provider = NewGeminiProvider(apiKey)
```text

**왜 이렇게?** 특정 AI 회사에 종속되지 않아, 모델이 바뀌어도 코드 변경 최소화.

---

### 원칙 6: 자동 컨텍스트 압축 (Auto Compact)

대화가 길어져 컨텍스트 윈도우 한계에 도달하면 **자동으로 요약·압축**합니다.

```text
대화 길이 → 80% 초과 감지
    ↓
숨겨진 Summarizer 에이전트 작동
    ↓
이전 대화를 핵심 요약으로 압축
    ↓
새 컨텍스트로 대화 계속
```text

## 4. 나만의 미니 에이전트 만들기

OpenCode의 원칙을 적용해 **간단한 Python 코드 리뷰 에이전트**를 만들어봅니다.

### 프로젝트 구조

```text
my-agent/
├── main.py
├── providers/
│   ├── base.py         # 프로바이더 추상화
│   └── anthropic.py    # Claude 구현
├── tools/
│   ├── file_reader.py  # 파일 읽기 도구
│   └── shell.py        # 셸 실행 도구
├── memory/
│   └── session.py      # SQLite 세션 저장
└── agent/
    └── loop.py         # 에이전트 루프
```text

### 핵심 코드 패턴

```python
# agent/loop.py — 에이전트 루프
class AgentLoop:
    def __init__(self, provider, tools, memory):
        self.provider = provider
        self.tools = {t.name: t for t in tools}
        self.memory = memory

    async def run(self, user_message: str):
        # 1. 메모리에서 이전 컨텍스트 로드
        history = self.memory.load_history()
        history.append({"role": "user", "content": user_message})

        while True:
            # 2. LLM 호출
            response = await self.provider.complete(
                messages=history,
                tools=list(self.tools.values())
            )

            # 3. 도구 호출이 있으면 실행
            if response.tool_calls:
                for tool_call in response.tool_calls:
                    # 권한 체크
                    if not self.check_permission(tool_call):
                        raise PermissionError(f"도구 {tool_call.name} 사용 권한 없음")

                    # 스냅샷 생성 (파일 수정 전)
                    if tool_call.name in ["write_file", "edit_file"]:
                        self.create_snapshot(tool_call.args["path"])

                    # 도구 실행
                    result = await self.tools[tool_call.name].execute(tool_call.args)
                    history.append({"role": "tool", "content": result})
            else:
                # 4. 도구 호출 없으면 최종 응답
                self.memory.save_history(history)
                return response.content
```text
````
