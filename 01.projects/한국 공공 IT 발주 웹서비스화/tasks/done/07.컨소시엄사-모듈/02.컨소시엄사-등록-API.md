# 컨소시엄사 등록 API

## 예상 소요시간
| 구분 | 시간 |
|------|------|
| 사람 (숙련 개발자) | ~30분 |
| AI 에이전트 | ~10분 |

## What (무엇을)
POST `/api/v1/projects/:projectId/companies` 엔드포인트를 구현합니다. 프로젝트에 참여사, 하청사 등 컨소시엘사를 추가합니다. 역할(role)과 상위사(parentId)를 지정할 수 있으며, 트리 구조를 유지합니다.

## Why (왜)
프로젝트의 협력사 구조를 정의하는 핵심 기능입니다. 주관사 아래에 참여사, 참여사 아래에 하청사 등 계층 구조를 만들어 조직도를 형성합니다. 멤버 배정 시 이 구조에 기반합니다.

### Step 1: CreateProjectCompanyDto 검증 규칙 추가

- 작업 내용:

  ```typescript
  import { IsString, IsOptional, IsUUID } from 'class-validator';

  export class CreateProjectCompanyDto {
    @IsUUID()
    companyId: string;

    @IsString()
    role: string; // 'PRIME', 'PARTNER', 'SUB' 등

    @IsOptional()
    @IsUUID()
    parentId?: string;
  }
  ```

### Step 2: ProjectCompanyService에 create 메서드 구현
- 대상 파일: `apps/backend/src/modules/project-company/project-company.service.ts`
- 작업 내용:
  ```typescript
  async create(
    projectId: string,
    dto: CreateProjectCompanyDto,
  ): Promise<ProjectCompanyDto> {
    // 프로젝트 존재 확인
    const project = await this.prisma.project.findFirst({
      where: { id: projectId, deletedAt: null },
    });

    if (!project) {
      throw new NotFoundException({
        code: 404000,
        message: '프로젝트를 찾을 수 없습니다.',
      });
    }

    // 회사 존재 확인
    const company = await this.prisma.company.findFirst({
      where: { id: dto.companyId, deletedAt: null },
    });

    if (!company) {
      throw new NotFoundException({
        code: 404000,
        message: '회사를 찾을 수 없습니다.',
      });
    }

    // parentId 유효성 확인 (같은 프로젝트에 속하는지)
    if (dto.parentId) {
      const parent = await this.prisma.projectCompany.findFirst({
        where: { id: dto.parentId, projectId, deletedAt: null },
      });

      if (!parent) {
        throw new BadRequestException({
          code: 400000,
          message: '상위 회사가 같은 프로젝트에 속하지 않습니다.',
        });
      }
    }

    // role 검증: PRIME은 중복 불가
    if (dto.role === 'PRIME' && !dto.parentId) {
      const existing = await this.prisma.projectCompany.findFirst({
        where: {
          projectId,
          role: 'PRIME',
          parentId: null,
          deletedAt: null,
        },
      });

      if (existing) {
        throw new ConflictException({
          code: 409001,
          message: '주관사(PRIME)는 중복으로 등록할 수 없습니다.',
        });
      }
    }

    const projectCompany = await this.prisma.projectCompany.create({
      data: {
        projectId,
        companyId: dto.companyId,
        role: dto.role,
        parentId: dto.parentId,
      },
      include: { company: true },
    });

    return {
      id: projectCompany.id,
      projectId: projectCompany.projectId,
      companyId: projectCompany.companyId,
      role: projectCompany.role,
      parentId: projectCompany.parentId,
      createdAt: projectCompany.createdAt,
      updatedAt: projectCompany.updatedAt,
    };
  }
  ```

### Step 3: ProjectCompanyController에 POST 엔드포인트 추가
- 대상 파일: `apps/backend/src/modules/project-company/project-company.controller.ts`
- 작업 내용:
  ```typescript
  import { Post, Body, Param, UseGuards } from '@nestjs/common';
  import { JwtAuthGuard } from '@/auth/guards/jwt-auth.guard';
  import { PermissionGuard } from '@/auth/guards/permission.guard';
  import { RequirePermission } from '@/common/decorators/require-permission.decorator';
  import { CreateProjectCompanyDto } from './dto/project-company.dto';

  @Post()
  @UseGuards(JwtAuthGuard, PermissionGuard)
  @RequirePermission('COMPANY', 'MANAGE')
  async create(
    @Param('projectId') projectId: string,
    @Body() dto: CreateProjectCompanyDto,
  ): Promise<{
    code: number;
    result: { projectCompany: ProjectCompanyDto };
  }> {
    const projectCompany = await this.projectCompanyService.create(
      projectId,
      dto,
    );
    return {
      code: 201000,
      result: { projectCompany },
    };
  }
  ```

## 참고
- CompanyRole enum: PRIME(주관사), PARTNER(참여사), SUB(하청사) 등
- parentId null: 최상위 노드 (주관사는 반드시 parentId가 null)
- PRIME 중복 불가: 주관사는 프로젝트당 하나만 가능
- 트리 구조: parentId를 통해 상위-하위 관계 정의
