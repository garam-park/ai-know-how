# RequirePermission 데코레이터

## 예상 소요시간
| 구분 | 시간 |
|------|------|
| 사람 (숙련 개발자) | ~20분 |
| AI 에이전트 | ~8분 |

## What (무엇을)
@RequirePermission(resource, action) 커스텀 데코레이터를 생성하여 컨트롤러 메서드에 필요한 권한을 선언적으로 명시합니다.

## Why (왜)
컨트롤러 메서드마다 필요한 권한을 명확하게 명시하면, 권한 체크 로직의 의도가 명확해지고 관리가 쉬워집니다. 이후 Guard에서 메타데이터를 읽어 권한을 검증합니다.

## 수행 가이드

### Step 1: RequirePermission 데코레이터 생성
- 대상 파일: `apps/backend/src/common/decorators/require-permission.decorator.ts`
- 작업 내용: 권한 요구 메타데이터 데코레이터 구현
  ```typescript
  import { SetMetadata } from '@nestjs/common';
  import { Resource, Action } from '@shared/enums';

  export interface RequirePermissionMetadata {
    resource: Resource;
    action: Action;
  }

  export const RequirePermission = (resource: Resource, action: Action) =>
    SetMetadata('permission', {
      resource,
      action,
    } as RequirePermissionMetadata);
  ```

### Step 2: 데코레이터 내보내기
- 대상 파일: `apps/backend/src/common/decorators/index.ts`
- 작업 내용: 모든 데코레이터를 한 파일에서 내보내기 (선택사항이지만 권장)
  ```typescript
  export { CurrentUser } from './current-user.decorator';
  export { RequirePermission, type RequirePermissionMetadata } from './require-permission.decorator';
  ```

### Step 3: 컨트롤러에서 사용 예시
- 작업 내용: 데코레이터 사용 방법 문서화
  ```typescript
  import { Controller, Get, UseGuards } from '@nestjs/common';
  import { JwtAuthGuard } from '../../common/guards/jwt-auth.guard';
  import { RequirePermission } from '../../common/decorators/require-permission.decorator';
  import { Resource, Action } from '@shared/enums';

  @Controller('api/v1/wbs')
  export class WbsController {
    // WBS 조회 권한 필요
    @UseGuards(JwtAuthGuard)
    @RequirePermission(Resource.WBS, Action.VIEW)
    @Get()
    async getWbs() {
      return {
        code: 200000,
        result: {
          wbs: [],
        },
      };
    }

    // WBS 편집 권한 필요
    @UseGuards(JwtAuthGuard)
    @RequirePermission(Resource.WBS, Action.EDIT)
    @Post()
    async createWbs() {
      return {
        code: 201000,
        result: {
          wbs: {},
        },
      };
    }

    // MEMBER 권한 관리 권한 필요
    @UseGuards(JwtAuthGuard)
    @RequirePermission(Resource.MEMBER, Action.PERMISSION)
    @Patch(':id/permission')
    async updateMemberPermission() {
      return {
        code: 200000,
        result: {},
      };
    }
  }
  ```

### Step 4: Reflector를 이용한 메타데이터 확인
- 작업 내용: Reflector 사용 방법 예시 (PermissionGuard에서 사용)
  ```typescript
  import { Injectable } from '@nestjs/common';
  import { Reflector } from '@nestjs/core';
  import { RequirePermissionMetadata } from './require-permission.decorator';

  @Injectable()
  export class SomeService {
    constructor(private reflector: Reflector) {}

    getPermissionMetadata(target: any): RequirePermissionMetadata | null {
      return this.reflector.get<RequirePermissionMetadata>(
        'permission',
        target,
      );
    }
  }
  ```

### Step 5: 메타데이터 우선순위 설정 (선택사항)
- 작업 내용: 메서드에 선언된 권한이 클래스의 권한을 덮어쓰도록 설정 가능
  ```typescript
  import { Controller, UseGuards } from '@nestjs/common';
  import { RequirePermission } from '../../common/decorators/require-permission.decorator';
  import { Resource, Action } from '@shared/enums';

  // 클래스 레벨: 모든 메서드에 기본 권한 설정
  @Controller('api/v1/projects')
  @RequirePermission(Resource.PROJECT, Action.VIEW)
  export class ProjectController {
    // 메서드 레벨: 더 높은 권한이 필요한 경우 덮어쓰기
    @RequirePermission(Resource.PROJECT, Action.EDIT)
    @Patch(':id')
    async updateProject() {
      return {};
    }
  }
  ```

  위의 경우 PermissionGuard에서는 메서드 메타데이터를 우선으로 조회해야 합니다:
  ```typescript
  const permission =
    this.reflector.get<RequirePermissionMetadata>('permission', handler) ||
    this.reflector.get<RequirePermissionMetadata>('permission', target);
  ```

## 참고
- SetMetadata는 NestJS에서 제공하는 메타데이터 저장 함수입니다.
- 메타데이터는 Guard에서 Reflector를 사용하여 조회됩니다.
- 선언적 접근으로 권한 체크 의도가 명확하고 문서화하기 쉬워집니다.
- 추후 자동 문서 생성 (Swagger) 시에도 이 메타데이터를 활용할 수 있습니다.
