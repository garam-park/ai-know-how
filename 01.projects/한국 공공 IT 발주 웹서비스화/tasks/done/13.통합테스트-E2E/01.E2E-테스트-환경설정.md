# E2E 테스트 환경설정

## 예상 소요시간
| 구분 | 시간 |
|------|------|
| 사람 (숙련 개발자) | ~30분 |
| AI 에이전트 | ~10분 |

## What (무엇을)
Playwright 또는 Cypress를 사용한 E2E 테스트 환경을 구성합니다. 테스트 데이터베이스 분리, seed 데이터 작성, 테스트 계정 설정을 포함합니다.

## Why (왜)
E2E 테스트는 사용자 관점에서 전체 기능을 검증하는 가장 신뢰성 높은 테스트 방법입니다.

### Step 4: 테스트 헬퍼 함수
`e2e/fixtures/auth.ts` 파일을 생성합니다:

```typescript
import { test as base } from '@playwright/test';
import { APIRequestContext } from '@playwright/test';

type AuthenticatedUser = {
  email: string;
  password: string;
  name: string;
  accessToken: string;
  refreshToken: string;
};

export const test = base.extend<{ authenticatedRequest: APIRequestContext }>({
  authenticatedRequest: async ({ request }, use) => {
    const testUser: AuthenticatedUser = {
      email: 'test@example.com',
      password: 'Test1234!',
      name: 'Test User',
      accessToken: '',
      refreshToken: '',
    };

    // 테스트 계정 생성/로그인
    const loginResponse = await request.post(
      `${process.env.API_BASE_URL || 'http://localhost:3001/api/v1'}/auth/login`,
      {
        data: {
          email: testUser.email,
          password: testUser.password,
        },
      }
    );

    if (loginResponse.ok()) {
      const data = await loginResponse.json();
      testUser.accessToken = data.data.accessToken;
      testUser.refreshToken = data.data.refreshToken;
    }

    // 인증 헤더 추가
    const authenticatedRequest = request;
    const originalFetch = authenticatedRequest.fetch;

    await use(authenticatedRequest);
  },
});

export { expect } from '@playwright/test';
```

### Step 5: Seed 데이터 스크립트
`scripts/seed-test-data.ts` 파일을 생성합니다:

```typescript
import { PrismaClient } from '@prisma/client';
import bcrypt from 'bcrypt';

const prisma = new PrismaClient();

async function seedTestData() {
  console.log('테스트 데이터 Seed 시작...');

  // 기존 데이터 삭제
  await prisma.member.deleteMany();
  await prisma.wBS.deleteMany();
  await prisma.company.deleteMany();
  await prisma.project.deleteMany();
  await prisma.user.deleteMany();

  // 테스트 사용자 생성
  const hashedPassword = await bcrypt.hash('Test1234!', 10);

  const adminUser = await prisma.user.create({
    data: {
      email: 'admin@example.com',
      password: hashedPassword,
      name: '관리자',
      role: 'ADMIN',
    },
  });

  const pmUser = await prisma.user.create({
    data: {
      email: 'pm@example.com',
      password: hashedPassword,
      name: 'PM',
      role: 'PM',
    },
  });

  const staffUser = await prisma.user.create({
    data: {
      email: 'staff@example.com',
      password: hashedPassword,
      name: '실무자',
      role: 'STAFF',
    },
  });

  // 테스트 프로젝트 생성
  const project = await prisma.project.create({
    data: {
      name: 'E2E 테스트 프로젝트',
      code: 'E2E-2024',
      startDate: new Date('2024-01-01'),
      endDate: new Date('2024-12-31'),
      description: 'E2E 테스트용 프로젝트',
      progress: 50,
      primaryCompanyId: '',
    },
  });

  // 회사 생성
  const company1 = await prisma.company.create({
    data: {
      projectId: project.id,
      name: '주관사',
      code: 'MAIN-001',
      depth: 0,
      sequence: 1,
    },
  });

  const company2 = await prisma.company.create({
    data: {
      projectId: project.id,
      name: '참여사',
      code: 'SUB-001',
      parentId: company1.id,
      depth: 1,
      sequence: 1,
    },
  });

  // 멤버 추가
  await prisma.member.create({
    data: {
      projectId: project.id,
      userId: pmUser.id,
      companyId: company1.id,
      role: 'PM',
      allocationRate: 100,
      joinedDate: new Date(),
    },
  });

  await prisma.member.create({
    data: {
      projectId: project.id,
      userId: staffUser.id,
      companyId: company2.id,
      role: 'STAFF',
      allocationRate: 80,
      joinedDate: new Date(),
    },
  });

  // WBS 생성
  const wbs1 = await prisma.wBS.create({
    data: {
      projectId: project.id,
      code: 'WBS-001',
      title: '요구사항 분석',
      type: 'CATEGORY',
      depth: 0,
      sequence: 1,
      progress: 100,
      startDate: new Date('2024-01-01'),
      endDate: new Date('2024-01-31'),
    },
  });

  const wbs2 = await prisma.wBS.create({
    data: {
      projectId: project.id,
      parentId: wbs1.id,
      code: 'WBS-001-001',
      title: '기본 요구사항 수집',
      type: 'TASK',
      depth: 1,
      sequence: 1,
      progress: 100,
      startDate: new Date('2024-01-01'),
      endDate: new Date('2024-01-15'),
      responsibleCompanyId: company1.id,
    },
  });

  const wbs3 = await prisma.wBS.create({
    data: {
      projectId: project.id,
      code: 'WBS-002',
      title: '설계',
      type: 'CATEGORY',
      depth: 0,
      sequence: 2,
      progress: 50,
      startDate: new Date('2024-02-01'),
      endDate: new Date('2024-03-31'),
    },
  });

  console.log('테스트 데이터 Seed 완료!');
  console.log('생성된 데이터:');
  console.log('- 관리자 계정:', adminUser.email);
  console.log('- PM 계정:', pmUser.email);
  console.log('- 실무자 계정:', staffUser.email);
  console.log('- 프로젝트:', project.name);

  await prisma.$disconnect();
}

seedTestData().catch((e) => {
  console.error('Seed 실패:', e);
  process.exit(1);
});
```

### Step 7: 테스트 유틸리티
`e2e/utils/helpers.ts` 파일을 생성합니다:

```typescript
import { Page, expect } from '@playwright/test';

export async function login(page: Page, email: string, password: string) {
  await page.goto('/login');
  await page.fill('input[type="email"]', email);
  await page.fill('input[type="password"]', password);
  await page.click('button:has-text("로그인")');
  await page.waitForURL('/projects');
}

export async function logout(page: Page) {
  await page.click('button:has-text("로그아웃")');
  await page.waitForURL('/login');
}

export async function navigateToProject(page: Page, projectName: string) {
  await page.goto('/projects');
  await page.click(`text=${projectName}`);
  await page.waitForURL(/\/projects\/\w+$/);
}

export async function expectToast(page: Page, message: string) {
  await expect(page.locator(`.toast:has-text("${message}")`)).toBeVisible();
}

export async function waitForLoading(page: Page) {
  await page.waitForSelector('.loading', { timeout: 1000 }).catch(() => {});
  await page.waitForSelector(':not(.loading)', { timeout: 5000 });
}
```

### Step 8: CI/CD 환경 설정
`.github/workflows/e2e-tests.yml` 파일을 생성합니다:

```yaml
name: E2E Tests

on:
  push:
    branches: [main, develop]
  pull_request:
    branches: [main, develop]

jobs:
  test:
    timeout-minutes: 60
    runs-on: ubuntu-latest

    services:
      postgres:
        image: postgres:15
        env:
          POSTGRES_USER: test_user
          POSTGRES_PASSWORD: test_password
          POSTGRES_DB: test_db
        options: >-
          --health-cmd pg_isready
          --health-interval 10s
          --health-timeout 5s
          --health-retries 5
        ports:
          - 5432:5432

    steps:
      - uses: actions/checkout@v3

      - name: Setup Node
        uses: actions/setup-node@v3
        with:
          node-version: '18'

      - name: Install dependencies
        run: npm ci

      - name: Setup test database
        run: |
          npx prisma migrate deploy
          npm run seed:test
        env:
          DATABASE_URL: postgresql://test_user:test_password@localhost:5432/test_db

      - name: Start backend
        run: npm run dev &
        env:
          DATABASE_URL: postgresql://test_user:test_password@localhost:5432/test_db

      - name: Wait for backend
        run: npx wait-on http://localhost:3001

      - name: Run E2E tests
        run: npm run test:e2e
        env:
          NODE_ENV: test
          DATABASE_URL: postgresql://test_user:test_password@localhost:5432/test_db
          API_BASE_URL: http://localhost:3001/api/v1
          FRONTEND_URL: http://localhost:3000

      - name: Upload report
        if: always()
        uses: actions/upload-artifact@v3
        with:
          name: playwright-report
          path: playwright-report/
          retention-days: 30
```
