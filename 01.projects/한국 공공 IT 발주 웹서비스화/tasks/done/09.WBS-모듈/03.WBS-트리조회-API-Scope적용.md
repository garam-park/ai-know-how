# WBS 트리 조회 API - Scope 적용

## 예상 소요시간
| 구분 | 시간 |
|------|------|
| 사람 (숙련 개발자) | ~40분 |
| AI 에이전트 | ~15분 |

## What (무엇을)
GET /api/v1/projects/:id/wbs-nodes 엔드포인트를 구현하여 권한 Scope에 따라 필터링된 WBS 트리를 반환합니다.

## Why (왜)
권한 시스템의 세 가지 Scope (ALL, OWN_COMPANY, OWN_TASK)에 따라 사용자가 볼 수 있는 WBS 노드가 달라집니다. 주관사PM은 전체 WBS를 보고, 참여사PM은 자신이 소속된 회사의 노드만, 실무자는 자신에게 할당된 작업만 봐야 합니다.

## 수행 가이드

### Step 1: buildWbsTree 유틸리티 함수 생성
`apps/backend/src/modules/wbs/utils/build-wbs-tree.ts`:
```typescript
interface WbsNode {
  id: number;
  parentId: number | null;
  children?: WbsNode[];
}

export function buildWbsTree(nodes: WbsNode[]): WbsNode[] {
  const nodeMap = new Map<number, WbsNode>();
  const roots: WbsNode[] = [];

  // 노드 맵 생성
  nodes.forEach(node => {
    nodeMap.set(node.id, { ...node, children: [] });
  });

  // 트리 구조 구성
  nodeMap.forEach(node => {
    if (node.parentId === null) {
      roots.push(node);
    } else {
      const parent = nodeMap.get(node.parentId);
      if (parent) {
        parent.children!.push(node);
      }
    }
  });

  return roots;
}
```

### Step 2: WbsService에 조회 로직 추가
`apps/backend/src/modules/wbs/wbs.service.ts`:
```typescript
async getWbsTree(projectId: number, scope: string, userId: number) {
  let nodes: any[] = [];

  if (scope === 'ALL') {
    // 전체 WBS 조회
    nodes = await this.prisma.wbsNode.findMany({
      where: {
        projectId,
        deletedAt: null,
      },
      include: {
        assignee: true,
        projectCompany: true,
      },
      orderBy: [
        { parentId: 'asc' },
        { order: 'asc' },
      ],
    });
  } else if (scope === 'OWN_COMPANY') {
    // 사용자 소속 회사의 노드 + 경로
    const user = await this.prisma.user.findUnique({
      where: { id: userId },
      include: { company: true },
    });

    const companyId = user.company.id;

    // 1. 해당 회사 담당 노드 조회
    const companyNodes = await this.prisma.wbsNode.findMany({
      where: {
        projectId,
        projectCompanyId: companyId,
        deletedAt: null,
      },
      include: {
        assignee: true,
        projectCompany: true,
      },
    });

    // 2. 각 노드의 root까지의 경로 조회
    const pathNodeIds = new Set<number>();
    companyNodes.forEach(node => pathNodeIds.add(node.id));

    for (const node of companyNodes) {
      let currentNode = node;
      while (currentNode.parentId !== null) {
        const parent = await this.prisma.wbsNode.findUnique({
          where: { id: currentNode.parentId },
          include: {
            assignee: true,
            projectCompany: true,
          },
        });
        if (parent) {
          pathNodeIds.add(parent.id);
          currentNode = parent;
        } else {
          break;
        }
      }
    }

    nodes = await this.prisma.wbsNode.findMany({
      where: {
        projectId,
        id: { in: Array.from(pathNodeIds) },
        deletedAt: null,
      },
      include: {
        assignee: true,
        projectCompany: true,
      },
      orderBy: [
        { parentId: 'asc' },
        { order: 'asc' },
      ],
    });
  } else if (scope === 'OWN_TASK') {
    // 사용자에게 할당된 노드 + 경로
    const assignedNodes = await this.prisma.wbsNode.findMany({
      where: {
        projectId,
        assigneeId: userId,
        deletedAt: null,
      },
      include: {
        assignee: true,
        projectCompany: true,
      },
    });

    // 각 노드의 root까지의 경로 조회
    const pathNodeIds = new Set<number>();
    assignedNodes.forEach(node => pathNodeIds.add(node.id));

    for (const node of assignedNodes) {
      let currentNode = node;
      while (currentNode.parentId !== null) {
        const parent = await this.prisma.wbsNode.findUnique({
          where: { id: currentNode.parentId },
          include: {
            assignee: true,
            projectCompany: true,
          },
        });
        if (parent) {
          pathNodeIds.add(parent.id);
          currentNode = parent;
        } else {
          break;
        }
      }
    }

    nodes = await this.prisma.wbsNode.findMany({
      where: {
        projectId,
        id: { in: Array.from(pathNodeIds) },
        deletedAt: null,
      },
      include: {
        assignee: true,
        projectCompany: true,
      },
      orderBy: [
        { parentId: 'asc' },
        { order: 'asc' },
      ],
    });
  }

  const tree = buildWbsTree(nodes);
  return {
    nodes: tree,
    totalCount: nodes.length,
  };
}
```

### Step 3: WbsController에 조회 엔드포인트 추가
`apps/backend/src/modules/wbs/wbs.controller.ts`:
```typescript
@Get()
@RequirePermission(Resource.WBS, Action.VIEW)
async getWbsTree(
  @Param('id') projectId: string,
  @Request() req,
) {
  const scope = req.scope; // PermissionGuard에서 설정
  const userId = req.user.id;

  const result = await this.wbsService.getWbsTree(
    parseInt(projectId),
    scope,
    userId,
  );

  return {
    code: 200000,
    result,
  };
}
```

## 완료 기준
- [ ] buildWbsTree 함수 구현 완료
- [ ] Scope = ALL: 전체 WBS 트리 반환
- [ ] Scope = OWN_COMPANY: 자기 회사 노드 + 경로 표시
- [ ] Scope = OWN_TASK: 자기 할당 노드 + 경로 표시
- [ ] 트리 구조 계층적으로 정확히 구성됨
- [ ] GET 요청 시 200000 응답 반환
- [ ] 삭제된 노드(deletedAt != null) 제외

## 참고
- Scope는 PermissionGuard에서 request.scope로 전달
- OWN_COMPANY/OWN_TASK는 해당 노드만이 아니라 root까지의 경로도 포함해야 트리 표시 가능
- children 배열은 클라이언트에서 재귀적으로 처리
