# 프론트엔드 React 초기화

## 예상 소요시간
| 구분 | 시간 |
|------|------|
| 사람 (숙련 개발자) | ~20분 |
| AI 에이전트 | ~8분 |

## What (무엇을)
`apps/frontend` 디렉토리에 Vite를 기반으로 React + TypeScript 프로젝트를 생성하고, 공유 패키지(`@shared/types`)를 의존성으로 추가하며, 기본 라우팅 구조를 설정합니다.

## Why (왜)
Vite는 매우 빠른 개발 서버와 빌드 성능을 제공하며, React와 TypeScript를 함께 사용하면 타입 안정성이 있는 사용자 인터페이스를 개발할 수 있습니다. 공유 패키지와의 연결로 백엔드와 일관된 타입을 사용할 수 있습니다.

## 선행 작업
- `01.루트-모노레포-초기화.md` 완료

## 수행 가이드

### Step 1: Vite + React + TypeScript 프로젝트 생성
```bash
cd project-root/apps
pnpm create vite frontend --template react-ts
```

프롬프트에서 기본값 선택.

### Step 2: 프론트엔드 package.json 구성
- 대상 파일: `project-root/apps/frontend/package.json`
- 수정 항목:

```json
{
  "name": "frontend",
  "private": true,
  "version": "1.0.0",
  "type": "module",
  "scripts": {
    "dev": "vite",
    "build": "tsc && vite build",
    "preview": "vite preview",
    "lint": "eslint src/**/*.{ts,tsx}",
    "format": "prettier --write src/**/*.{ts,tsx}"
  },
  "dependencies": {
    "react": "^18.2.0",
    "react-dom": "^18.2.0",
    "react-router-dom": "^6.20.0",
    "@shared/types": "workspace:*"
  },
  "devDependencies": {
    "@types/react": "^18.2.0",
    "@types/react-dom": "^18.2.0",
    "@vitejs/plugin-react": "^4.0.0",
    "typescript": "^5.0.0",
    "vite": "^5.0.0"
  }
}
```

### Step 3: TypeScript 설정 (tsconfig.json)
- 대상 파일: `project-root/apps/frontend/tsconfig.json`
- Vite가 생성한 기본 설정을 다음과 같이 수정:

```json
{
  "compilerOptions": {
    "target": "ES2020",
    "useDefineForClassFields": true,
    "lib": ["ES2020", "DOM", "DOM.Iterable"],
    "module": "ESNext",
    "skipLibCheck": true,
    "esModuleInterop": true,
    "strict": true,
    "noUnusedLocals": true,
    "noUnusedParameters": true,
    "noFallthroughCasesInSwitch": true,
    "resolveJsonModule": true,
    "moduleResolution": "bundler",
    "allowImportingTsExtensions": true,
    "allowJs": true,
    "baseUrl": ".",
    "paths": {
      "@shared/*": ["../../packages/shared/src/*"],
      "src/*": ["src/*"]
    },
    "jsx": "react-jsx"
  },
  "include": ["src"],
  "references": [{ "path": "./tsconfig.node.json" }]
}
```

### Step 4: Vite 설정 (vite.config.ts)
- 대상 파일: `project-root/apps/frontend/vite.config.ts`

```typescript
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'
import path from 'path'

export default defineConfig({
  plugins: [react()],
  resolve: {
    alias: {
      '@shared': path.resolve(__dirname, '../../packages/shared/src'),
      'src': path.resolve(__dirname, 'src'),
    },
  },
  server: {
    port: 5173,
    strictPort: false,
  },
})
```

### Step 5: 기본 App 컴포넌트 생성
- 대상 파일: `project-root/apps/frontend/src/App.tsx`

```typescript
import { BrowserRouter, Routes, Route } from 'react-router-dom';
import './App.css';
import { Resource } from '@shared/types';

export function App() {
  return (
    <BrowserRouter>
      <div className="app">
        <header>
          <h1>한국 공공 IT 발주 웹서비스</h1>
          <nav>
            <ul>
              <li><a href="/">홈</a></li>
              <li><a href="/projects">프로젝트</a></li>
              <li><a href="/companies">회사</a></li>
            </ul>
          </nav>
        </header>
        
        <main>
          <Routes>
            <Route path="/" element={<HomePage />} />
            <Route path="/projects" element={<ProjectsPage />} />
            <Route path="/companies" element={<CompaniesPage />} />
          </Routes>
        </main>
      </div>
    </BrowserRouter>
  );
}

function HomePage() {
  return <div><h2>홈페이지</h2><p>공공 IT 프로젝트 관리 시스템</p></div>;
}

function ProjectsPage() {
  return <div><h2>프로젝트 목록</h2></div>;
}

function CompaniesPage() {
  return <div><h2>회사 목록</h2></div>;
}

export default App;
```

### Step 6: 공유 패키지 의존성 추가
```bash
cd project-root/apps/frontend
pnpm add @shared/types@workspace:*
pnpm add react-router-dom
```

### Step 7: React Router 설치 확인
프론트엔드 package.json에서 react-router-dom이 설치되어 있는지 확인하고, 필요하면:

```bash
cd project-root/apps/frontend
pnpm add react-router-dom@^6.20.0
```

### Step 8: 개발 서버 실행 테스트
```bash
cd project-root/apps/frontend
pnpm run dev
```

브라우저에서 `http://localhost:5173`에 접속하여 기본 페이지가 표시되는지 확인.

### Step 9: TypeScript 검사 실행
```bash
cd project-root/apps/frontend
pnpm run build
```

TypeScript 컴파일 에러가 없는지 확인.

## 완료 기준
- [ ] `apps/frontend/package.json`이 존재하고 `@shared/types` 및 `react-router-dom` 의존성이 포함됨
- [ ] `apps/frontend/tsconfig.json`에 경로 별칭이 설정됨 (`@shared/*`)
- [ ] `apps/frontend/src/` 디렉토리에 App.tsx, main.tsx 파일 존재
- [ ] `pnpm run dev` 명령으로 개발 서버 시작 성공
- [ ] `http://localhost:5173`에서 애플리케이션 로드됨
- [ ] 공유 패키지로부터 타입 import 가능 (예: `import { Resource } from '@shared/types'`)

## 참고
- Vite 개발 서버는 기본적으로 localhost:5173에서 실행됨
- `pnpm run build` 명령은 `dist/` 디렉토리에 프로덕션 빌드를 생성
- React Router는 SPA 라우팅을 위해 필요
- 향후 페이지 컴포넌트는 `src/pages/` 디렉토리에, 재사용 가능한 컴포넌트는 `src/components/` 디렉토리에 생성
