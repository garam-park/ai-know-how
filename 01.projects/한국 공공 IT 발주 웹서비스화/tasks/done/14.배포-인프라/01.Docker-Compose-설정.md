# Docker Compose 설정

## 예상 소요시간
| 구분 | 시간 |
|------|------|
| 사람 (숙련 개발자) | ~30분 |
| AI 에이전트 | ~10분 |

## What (무엇을)
Docker Compose로 PostgreSQL, Backend, Frontend를 한 번에 실행할 수 있도록 설정합니다. 개발 환경 표준화와 배포 준비의 기반을 마련합니다.

## Why (왜)
모든 개발자/CI-CD가 동일한 환경에서 작업할 수 있게 합니다. 로컬 개발 환경 설정 시간을 획기적으로 단축합니다.

## 수행 가이드

### Step 1: Backend Dockerfile 생성
`apps/backend/Dockerfile` 파일을 생성합니다:

```dockerfile
# 빌드 스테이지
FROM node:18-alpine AS builder

WORKDIR /app

# 의존성 설치
COPY package*.json ./
RUN npm ci

# 타입스크립트 컴파일 및 빌드
COPY . .
RUN npm run build

# 프로덕션 스테이지
FROM node:18-alpine

WORKDIR /app

# 타임존 설정
RUN apk add --no-cache tzdata
ENV TZ=Asia/Seoul

# 프로덕션 의존성만 설치
COPY package*.json ./
RUN npm ci --only=production

# 빌드된 코드 복사
COPY --from=builder /app/dist ./dist

# Prisma migration 및 생성 파일 복사
COPY --from=builder /app/prisma ./prisma

EXPOSE 3001

# 헬스 체크
HEALTHCHECK --interval=30s --timeout=3s --start-period=5s --retries=3 \
  CMD node -e "require('http').get('http://localhost:3001/health', (r) => {if (r.statusCode !== 200) throw new Error(r.statusCode)})"

CMD ["sh", "-c", "npx prisma migrate deploy && node dist/main.js"]
```

### Step 2: Frontend Dockerfile 생성
`apps/frontend/Dockerfile` 파일을 생성합니다:

```dockerfile
# 빌드 스테이지
FROM node:18-alpine AS builder

WORKDIR /app

COPY package*.json ./
RUN npm ci

COPY . .
RUN npm run build

# Nginx 스테이지
FROM nginx:alpine

# 타임존 설정
RUN apk add --no-cache tzdata
ENV TZ=Asia/Seoul

# Nginx 설정
RUN rm -rf /etc/nginx/conf.d/*
COPY nginx.conf /etc/nginx/conf.d/default.conf

# 빌드된 정적 파일 복사
COPY --from=builder /app/dist /usr/share/nginx/html

EXPOSE 3000

HEALTHCHECK --interval=30s --timeout=3s --start-period=5s --retries=3 \
  CMD wget --no-verbose --tries=1 --spider http://localhost:3000/ || exit 1

CMD ["nginx", "-g", "daemon off;"]
```

### Step 3: Nginx 설정
`apps/frontend/nginx.conf` 파일을 생성합니다:

```nginx
server {
  listen 3000;
  server_name localhost;

  # 클라이언트 최대 업로드 크기
  client_max_body_size 100M;

  # 정적 파일 캐싱
  location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
    expires 30d;
    add_header Cache-Control "public, immutable";
    add_header X-Content-Type-Options "nosniff" always;
  }

  # API 프록시
  location /api/ {
    proxy_pass http://backend:3001;
    proxy_http_version 1.1;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection 'upgrade';
    proxy_set_header Host $host;
    proxy_cache_bypass $http_upgrade;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
  }

  # SPA 라우팅: 모든 요청을 index.html로
  location / {
    root /usr/share/nginx/html;
    try_files $uri $uri/ /index.html;
    add_header Cache-Control "no-cache, no-store, must-revalidate";
  }

  # 보안 헤더
  add_header X-Frame-Options "SAMEORIGIN" always;
  add_header X-Content-Type-Options "nosniff" always;
  add_header X-XSS-Protection "1; mode=block" always;
  add_header Referrer-Policy "no-referrer-when-downgrade" always;
}
```

### Step 4: Docker Compose 설정
`docker-compose.yml` 파일을 생성합니다:

```yaml
version: '3.8'

services:
  # PostgreSQL 데이터베이스
  postgres:
    image: postgres:15-alpine
    container_name: pg-db
    environment:
      POSTGRES_USER: ${DB_USER:-app_user}
      POSTGRES_PASSWORD: ${DB_PASSWORD:-app_password}
      POSTGRES_DB: ${DB_NAME:-app_db}
      TZ: Asia/Seoul
    volumes:
      - postgres_data:/var/lib/postgresql/data
      - ./scripts/init-db.sql:/docker-entrypoint-initdb.d/init.sql
    ports:
      - "${DB_PORT:-5432}:5432"
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U ${DB_USER:-app_user}"]
      interval: 10s
      timeout: 5s
      retries: 5
    networks:
      - app_network

  # Backend API 서버
  backend:
    build:
      context: ./apps/backend
      dockerfile: Dockerfile
    container_name: backend-api
    environment:
      NODE_ENV: ${NODE_ENV:-development}
      DATABASE_URL: postgresql://${DB_USER:-app_user}:${DB_PASSWORD:-app_password}@postgres:5432/${DB_NAME:-app_db}
      JWT_SECRET: ${JWT_SECRET:-dev_secret_key_change_in_production}
      JWT_EXPIRY: ${JWT_EXPIRY:-3600}
      REFRESH_TOKEN_EXPIRY: ${REFRESH_TOKEN_EXPIRY:-604800}
      CORS_ORIGIN: http://localhost:3000
      PORT: 3001
      TZ: Asia/Seoul
    ports:
      - "${BACKEND_PORT:-3001}:3001"
    depends_on:
      postgres:
        condition: service_healthy
    volumes:
      - ./apps/backend:/app
      - /app/node_modules
      - /app/dist
    restart: unless-stopped
    networks:
      - app_network
    healthcheck:
      test: ["CMD", "wget", "--no-verbose", "--tries=1", "--spider", "http://localhost:3001/health"]
      interval: 30s
      timeout: 5s
      retries: 3

  # Frontend 웹 서버
  frontend:
    build:
      context: ./apps/frontend
      dockerfile: Dockerfile
    container_name: frontend-web
    environment:
      REACT_APP_API_BASE_URL: http://localhost:3001/api/v1
      TZ: Asia/Seoul
    ports:
      - "${FRONTEND_PORT:-3000}:3000"
    depends_on:
      - backend
    restart: unless-stopped
    networks:
      - app_network
    healthcheck:
      test: ["CMD", "wget", "--no-verbose", "--tries=1", "--spider", "http://localhost:3000"]
      interval: 30s
      timeout: 5s
      retries: 3

volumes:
  postgres_data:
    driver: local

networks:
  app_network:
    driver: bridge
```

### Step 5: 환경 변수 설정
`.env.docker` 파일을 생성합니다:

```ini

NODE_ENV=development

# 데이터베이스
DB_USER=app_user
DB_PASSWORD=app_password
DB_NAME=app_db
DB_PORT=5432

# JWT
JWT_SECRET=dev_secret_key_change_in_production
JWT_EXPIRY=3600
REFRESH_TOKEN_EXPIRY=604800

# 포트
BACKEND_PORT=3001
FRONTEND_PORT=3000

# CORS
CORS_ORIGIN=http://localhost:3000
```

### Step 7: Package.json 스크립트 추가
`package.json`의 scripts 섹션에 추가:

```json
{
  "scripts": {
    "docker:build": "docker-compose build",
    "docker:up": "docker-compose up -d",
    "docker:down": "docker-compose down",
    "docker:logs": "docker-compose logs -f",
    "docker:logs:backend": "docker-compose logs -f backend",
    "docker:logs:frontend": "docker-compose logs -f frontend",
    "docker:logs:postgres": "docker-compose logs -f postgres",
    "docker:restart": "docker-compose restart",
    "docker:clean": "docker-compose down -v",
    "docker:reset": "docker-compose down -v && docker-compose up -d"
  }
}
```

### Step 9: 실행 가이드
```bash
# 첫 번째 실행
docker-compose up -d

# 또는 환경 파일 지정
docker-compose --env-file .env.docker up -d

# 로그 확인
docker-compose logs -f

# 상태 확인
docker-compose ps

# 중지
docker-compose down

# 전체 리셋 (데이터 삭제 포함)
docker-compose down -v
```
