# 수동 검증: React Query 적용 및 UI 상태 체크

## 개요
기존의 `useState` + `useEffect` 패턴을 완전히 제거하고 도입한 `TanStack Query (React Query)` 환경에서 로딩 상태나 캐싱 시스템이 문제없이 구동되는지, 그리고 멤버 투입률 경고 로직이 UI에 나타나는지 점검합니다.

## 검증 대상
- 프론트엔드 라우팅 및 전역 상태(App.tsx `QueryClientProvider` 연동)
- MembersTab (투입률 100% 초과 경고 UI)

## 검증 시나리오
1. **React Query 로딩 상태 검증**
   - 네트워크 연결 속도를 브라우저 개발자 도구의 Network 탭에서 `Slow 3G`로 강제 조작합니다.
   - 프로젝트 목록이나 각 상세 탭들을 전환할 때 부드럽게 **"로딩 중..."** 텍스트 렌더가 나타나고 자연스럽게 화면으로 전환되는지 확인합니다.
   - 탭을 전환했다가 기존에 갔던 같은 탭으로 다시 돌아왔을 때, 이전처럼 "로딩 중"이 강제로 노출되지 않고 **캐싱(Cache)된 화면이 즉시 표시**되는지 확인합니다.

2. **멤버 투입률 UI 검증**
   - `MembersTab`에 진입 후, 백엔드 데이터에 강제로 투입률(`inputRate`)이 `101%` 이상인 멤버가 세팅되었을 때 (DB 수정이 필요할 수 있음)
   - 해당 사용자의 레코드 줄에서 **투입률 글씨가 빨갛게 표시되고 셀 바탕색이 연한 붉은색**이 적용되었는지 확인합니다.
   - 투입률 옆에 노란색 경고(⚠️) 아이콘이 생기고, 마우스를 올렸을 때 툴팁 힌트("다른 프로젝트 포함 총 투입률 과다")가 나오는지 체크합니다.

3. **콘솔 에러 방지**
   - 개발자 환경(크롬 브라우저 콘솔 창)을 띄우고, 페이지들을 돌아다닐 때 콘솔에 빨간색 `Unhandled Promise Rejection` 혹은 React 렌더링 에러가 발생하지 않는지 체크합니다.
