| **요구사항 명칭**     | 데이터 연계/수집/정제 |

## 정의

데이터 연계/수집/정제 요구사항

## 세부내용

- 데이터 수집․제공 기능 공통
  - 데이터 유형 및 포맷 등에 맞는 수집/제공 기술, 수집/제공 주기 적용
  - 법령상 해당되는 개인정보 필터링 후 개인정보 제외 또는 비식별화 처리
  - 향후 장애 점검 등을 위하여 수집/제공 로그 기록
  - 시스템 연계를 통한 자동 또는 반자동 수집기능 제공
  - 외부(공공, 민간) 수집 방법 구현
- 내/외부 다양한 원천 데이터를 수집/연계 시스템 구축
  - 이 기종 DB 연결을 지원하고, SQL를 이용한 데이터 추출 기능
  - 부서에서 오프라인으로 관리하고 있는 통계, 도정 주요지표(통계사이트 제공 API 연계 등)에 대해 발주기관 표준포맷으로 DB 스키마 생성 후 도 및 시군에서 파일데이터(엑셀, CSV 등) 업로드 기능 구현
  - 업로드를 위한 표준 레이아웃 샘플 제공 및 컬럼, 정의된 데이터형식 적합 유무 체크, 목록 검색기능
- 수집 시 데이터 원천 및 접속 정보를 관리할 수 있는 기능 제공
  - 다양한 유형의 데이터 소스 연결 설정 제공
- 다양한 데이터 원천과 연결을 등록･수정･삭제기능 제공
  - 다양한 수집 프로토콜 지원 (JDBC, SSH, SFTP, FTP, Rsync, TCP, UDP, SNMP 등) 시스템 연계 수집/제공 기능 구현
  - 시스템 연계 수집/제공 시 타 시스템 성능에 영향이 없도록 설계
  - 다양한 연계 방식에 대한 직관적인 GUI를 통해 이를 관리 및 모니터링 할 수 있도록 구축 연계 및 수집 데이터의 정제 기능 구현
  - 데이터 분석을 위하여 불필요한 항목 제거
  - 데이터 품질을 향상시킬 수 있도록 전후처리 수행
  - 공간분석, 시계열분석, 통계분석 등 다양한 분석 형태에 따라 적절한 데이터 처리
