# SFR-001 데이터 수집·연계 체계 구축

| 항목                  | 내용                       |
| --------------------- | -------------------------- |
| **요구사항 분류**     | 기능 요구사항              |
| **요구사항 고유번호** | SFR-001                    |
| **요구사항 명칭**     | 데이터 수집·연계 체계 구축 |

## 정의

데이터 수집·연계 및 운영 관리 요건

## 세부내용

- 기 구축된 제주 스마트시티 데이터허브 시스템의 구조, 데이터 수집체계, 데이터 표준, 연계 인터페이스(API/DB 등)를 분석하고, 본 사업을 통해 수집·가공되는 민원·행정 데이터 및 영상분석·음원분석·센서 기반 AI 분석 데이터를 데이터허브와 상호 연계·활용할 수 있는 구조 및 연계 전략을 구체적으로 제시하여야 함
- 제안사는 시범솔루션 구현을 위해 필요한 데이터에 대해 데이터 항목별 수집방안을 반드시 명시하여야 하며, 각 데이터별로 데이터 출처, 수집 방식, 수집 주기 및 범위, 데이터 생성·정제·저장 절차를 포함한 구체적인 데이터 수집체계를 제시하여야 함
- 제주시·서귀포시 민원포털 및 제주도청 120만덕콜센터의 주차 및 안전 민원데이터를 AI RPA를 이용하여 자동으로 수집할 수 있어야 함
- 공영주차장 및 불법 주정차 관련 지자체의 정책, 사업계획, 시행내용 및 결과 등 행정 문서 데이터를 수집·연계할 수 있어야 함
- 음성으로 녹취된 민원 정보를 텍스트로 변환하고 개인정보를 비식별 처리할 수 있어야 함
- 수집된 모든 데이터는 NGSI-LD 기반 표준 스키마(Entity/Attribute)로 구조화하여 저장할 수 있어야 함
- 공영주차장 화재 및 안전 데이터 3종을 실시간으로 수집·저장할 수 있어야 함
- 공영주차장 내 화재·이상 상황 감지 및 수집 연계할 수 있는 기술적 방안(IoT 센서, 영상 기반 감지, 이상음원 기반 감지 등)을 사업자가 제안할 수 있어야 함
- 초기 구축 단계에서 확보 가능한 데이터가 제한적일 수 있음을 고려하여, 필요 시 추가 센서(화재 감지 센서, 환경 센서, 음원 센서 등) 설치, 카메라 보강, 수집 채널 확대 등을 통해 데이터 수집 범위를 확장할 수 있도록 데이터 확보 전략을 수립하고, 단계별 센서·장비 확충 계획을 포함한 데이터 확보 방안을 제시하여야 함
- 제주도 내 전기차 충전기 운영 관련 데이터를 수집·연계할 수 있어야 함
- 기상청, 112/119 등 공공데이터 API와 실시간 연계를 통해 안전·민원 대응을 지원할 수 있어야 함
