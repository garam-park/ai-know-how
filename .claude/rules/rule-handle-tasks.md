# 작업 가이드 생성 규칙

## 디렉토리 구조

- `tasks/`: 작업 목록, 작업 상태, 작업 담당자 등
  - `done/`: 완료된 작업
  - `plan/`: 작업 계획 중인 항목 — AI가 수행 가이드를 생성하여 저장하는 공간
  - `templates/`: 가이드 작성에 참고할 템플릿

## 작업 가이드 생성 워크플로

사용자가 "X 기능 구현 가이드 작성해줘" 형태로 요청하면 다음 순서로 진행한다.

<!-- 1. **탐색**: `apps/` 디렉토리에서 관련 소스코드를 파악하고, `docs/` 디렉토리에서 관련 문서를 파악한다.
2. **분석**: 기능 구현에 필요한 맥락(의존성, 관련 모듈, 도메인 규칙 등)을 정리한다. -->
1. **가이드 생성**: 분석이 완료되면 `tasks/plan/{YYYYMMDD}/{title}.md` 경로에 수행 가이드를 생성한다.
   - `{YYYYMMDD}`: 오늘 날짜 (예: `20260326`)
   - `{title}`: 기능을 간결하게 표현한 한글 (예: `비행계획_API`)
2. **가이드 품질**: AI나 사람이 가이드만 보고 그대로 따라할 수 있을 정도의 구체성을 갖춰야 한다.

## 가이드 작성 원칙

- 각 단계는 명확한 **목표**와 **완료 기준**을 포함한다.
- 코드 변경이 필요한 경우, 대상 파일 경로와 변경 내용을 구체적으로 명시한다.
- 판단이 필요한 부분은 "판단 기준"으로 별도 표시한다.
- 작업 완료 후 `tasks/done/`으로 이동한다.
