# 문서 관리 규칙

## 프로젝트 구현 및 관련 문서

- `docs/`: 프로젝트 관련 문서
  - `develop/`: 개발 관련 문서
  - `deliverables/`: 최종 산출물, 감리에 필요한 문서
  - `plans/`: 프로젝트 계획서, 일정, 작업 분배 등
  - `reports/`: 프로젝트 진행 보고서, 회의록 등

## docs 에서 일반 규칙

- `**/inbox/` : 위치가 정해지지 않은 문서, 작업, 아이디어 등을 임시로 저장하는 공간

## 문서 수정 작업

1. 수정할 문서를 문서가 있는 디렉토리에 `wip/` 디렉토리에 복사, 디렉토리가 없으면 생성 후 복사
   1.1. 복사 시에 원래 문서의 버전의 patch 버전을 증가시켜 이동
   1.2. 날짜가 포함된 문서의 경우, 날짜를 현재 날짜로 변경하여 이동
   1.3. 예: `docs/UAM/UAM_엔터티정의_확장_v1.0.0_20260102.md` -> `docs/UAM/wip/UAM_엔터티정의_확장_v1.0.1_20260325.md`
2. 문서 수정 작업을 완료한 후, `wip/` 디렉토리에서 원래 위치로 이동
   2.1 예: `docs/UAM/wip/UAM_엔터티정의_확장_v1.0.1_20260325.md` -> `docs/UAM/UAM_엔터티정의_확장_v1.0.1_20260325.md`
3. 원본 문서는 문서가 있는 곳에 `histories/{YYYYMMDD}/` 디렉토리를 만들어 이동
   3.1 예: `docs/UAM/UAM_엔터티정의_확장_v1.0.0_20260102.md` -> `docs/UAM/histories/20260325/UAM_엔터티정의_확장_v1.0.0_20260102.md`
