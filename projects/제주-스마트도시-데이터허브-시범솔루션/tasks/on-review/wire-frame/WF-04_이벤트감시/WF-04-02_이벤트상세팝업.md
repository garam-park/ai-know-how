## 화면 구성

### 팝업 기본 구조
- **형식**: 모달 또는 오른쪽 사이드패널
- **크기**: 모달의 경우 600px x 700px 정도, 또는 전체 높이 사이드패널
- **닫기**: X 버튼 (우상단)

### 상단 기본정보 영역
- **좌측 아이콘**: 이벤트 유형 아이콘 (32x32px)
  - 화재, 음향, 비정상, 민원, 기타 구분
- **중앙 정보**:
  - 이벤트명 (큰 폰트, 굵음)
    - 예: "화재경보 신호 수신"
  - 발생시각 (작은 폰트, 회색)
    - 예: "2024-04-01 14:25:30"
  - 주차장명 (중간 폰트)
    - 예: "신도심 A 주차장"
- **우측 심각도 뱃지**:
  - 큰 뱃지, 색상 + 텍스트
  - 예: 빨강 배경에 "긴급"

### 위치 정보
- 상세 주소
- 위도/경도 (작은 텍스트)
- "지도에서 보기" 링크 (클릭 시 지도 오픈)

### 중앙 미디어 영역
- **2개 섹션** (탭 또는 분할):

#### 1. CCTV 스냅샷
- 큰 이미지 영역 (400x300px)
- 플레이스홀더 이미지 또는 실제 CCTV 프레임
- 이미지 하단: 촬영시각, 카메라ID
- 버튼: "영상 재생" (동영상 플레이어로 이동)

#### 2. 음향 파형
- 파형 시각화 (Waveform 차트)
- 가로축: 시간 (초)
- 세로축: 음량 레벨
- 컬러: 음량이 높은 구간을 빨강으로 표시
- 플레이버튼: 음향 재생 가능
- 녹음기기ID + 녹음시각

### 하단 AI 분석 결과 영역
- **제목**: "AI 분석 결과"
- **분석 항목**:
  - 탐지 신뢰도: 95% (프로그레스 바 또는 수치)
  - 분류 근거:
    - "음향 특성 분석: 4-8kHz 범위에서 알람음 패턴 감지"
    - "MFCC 특성: 표준 화재경보음과 98% 일치"
    - "시각적 분석: 밝기 이상 감지 안 됨 (오탐 가능성 낮음)"

### 대응 버튼 영역
- **버튼 레이아웃**: 가로 배치 (최대 4개)
- **버튼 목록**:
  - "확인": 이벤트 확인 (상태 변경: 미확인 → 확인)
  - "대응시작": 대응 프로세스 시작 (WF-04-05로 이동 또는 체크리스트 오픈)
  - "오탐보고": 거짓 알람 보고 (확인 팝업 후 상태 변경)
  - "에스컬레이션": 상위기관 보고 또는 긴급 호출

- **버튼 색상**:
  - "대응시작": 파란색 (주요 액션)
  - "확인": 회색
  - "오탐보고": 주황색
  - "에스컬레이션": 빨강색
