## 화면 구성
### 상단 버튼
- "규칙 추가" 버튼 → 설정 모달

### 알림 규칙 목록 테이블
| 항목 | 설명 |
|------|------|
| 규칙명 * | 규칙의 이름 (클릭 시 편집 모달) |
| 조건 | 트리거 조건 (예: "데이터 수집 실패") |
| 채널 | 알림 방식 (SMS/이메일/앱푸시/웹훅 태그) |
| 수신그룹 | 대상 그룹명 (예: "기술팀", "관리자") |
| 활성 | Toggle switch (On/Off) |
| 액션 | 편집, 삭제 버튼 |

- 테이블 정렬 가능 (규칙명, 조건)
- 필터링: 활성 여부
- 페이지네이션 또는 무한 스크롤

### 규칙 설정 모달 (신규 추가 또는 편집)
**섹션 1: 조건 설정**
- 이벤트 유형 * (Dropdown)
  - 데이터 수집 실패
  - OCR 처리 실패율 높음
  - STT 처리 실패율 높음
  - 데이터 품질 저하
  - 시스템 오류
  - 기타

- 심각도 * (Dropdown)
  - 높음 (빨강)
  - 중간 (주황)
  - 낮음 (초록)

- 임계값 설정 (Number)
  - 예: "실패율이 10% 이상", "에러 발생 5회 이상"

**섹션 2: 알림 채널 선택 (다중선택)**
- [ ] SMS
- [ ] 이메일
- [ ] 앱푸시
- [ ] 웹훅

**섹션 3: 수신자 설정**
- 수신 그룹 (다중선택, Dropdown)
  - 관리자
  - 기술팀
  - 운영팀
  - 사용자정의 그룹

**섹션 4: 반복 알림 설정**
- 반복 알림 활성 (Toggle)
  - 활성화 시 간격 입력 필드 표시
  - 간격 (분): 숫자 입력 (기본값: 60분)

**섹션 5: 규칙명 및 설명**
- 규칙명 * (Text)
- 설명 (Textarea, 선택사항)

- "저장" 및 "취소" 버튼

### 하단 알림 발송 이력
알림 발송 이력 테이블 (최근 20건):
| 시각 | 규칙명 | 이벤트 | 채널 | 수신자 | 상태 |
|------|--------|--------|------|--------|------|
| 2026-03-25 14:32 | 데이터 수집 실패 | 민원포털 수집 실패 | SMS, 이메일 | 관리자 | 발송완료 |
| 2026-03-25 13:15 | 데이터 품질 저하 | 완전성 85% | 이메일 | 기술팀 | 발송완료 |

- 테이블 정렬 가능
- 상태: 발송완료(초록), 발송중(파란), 발송실패(빨강) 뱃지
