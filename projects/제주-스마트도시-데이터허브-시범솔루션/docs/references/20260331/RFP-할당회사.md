# 제주 스마트도시 데이터허브 시범솔루션 RFP 할당 회사 정리(안)

> 작성일: 2026-04-02  
> 기준 문서: RFPs 하위 COR/DAR/ECR/PMR/PER/PSR/QUR/SFR/SIR/TER

## 1. 할당 원칙

1. 데이터허브/표준/수집 파이프라인은 플랫폼 보유사(디토닉) 주관
2. 대시보드/사용자 인터페이스는 이노팸 주관
3. 현장 장비/센서/CCTV 연동은 이노뎁 주관
4. EV 충전 데이터 연계는 진우이노베이션 주관
5. PM·품질·보안·테스트는 디토닉 총괄 + 각사 모듈별 책임 이행

## 2. 회사별 담당 업무 요약

| 회사 | 주관 업무 | 협업 업무 |
|---|---|---|
| 🧠 디토닉 | 데이터 수집·연계 체계, 데이터 표준, 전처리 백엔드, LLM 핵심 모듈, 데이터 카탈로그, PM 총괄 | 대시보드 연계 API, AI Assistant 백엔드, 현장 이벤트 통합 |
| 🎨 이노팸 | 대시보드/시각화, UI/UX 표준, 사용자 포털/리포트, AI Assistant 프론트 | LLM 요약/분류/검색 일부, 통합 모니터링 화면 |
| 📡 이노뎁 | 공영주차장 현장 장비 설치, 센서·CCTV 연계, 이벤트 감시 현장 연동 | 이벤트 AI 분석 파이프라인 연계, 통합관제 연계 |
| 🔌 진우이노베이션 | EV 충전 인프라 데이터 수집·연계 인터페이스 | EV 이상행동 데이터 제공/연계 |

## 5. 협의 필요 사항

1. SFR-005(스마트 안전 AI) 주관을 디토닉 단독으로 할지, 디토닉·이노뎁 공동으로 명문화할지 확정 필요
2. LLM 세부 모듈(요약/분류/검색/질의응답)별 이노팸 담당 범위를 산출물 단위로 확정 필요
3. 데이터허브 API(스키마/주기/SLA)와 대시보드 화면 개발 일정의 인터페이스 기준일 확정 필요
