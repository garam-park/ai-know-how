# 컨소시엘사 트리 조회 API

## 예상 소요시간
| 구분 | 시간 |
|------|------|
| 사람 (숙련 개발자) | ~40분 |
| AI 에이전트 | ~15분 |

## What (무엇을)
GET `/api/v1/projects/:projectId/companies` 엔드포인트를 구현합니다. 프로젝트의 모든 컨소시엘사를 트리 구조로 반환합니다. 프론트에서 조직도를 시각화할 수 있는 형태입니다.

## Why (왜)
프로젝트의 협력사 구조를 계층적으로 조회해야 합니다. 사용자가 조직도 형태로 전체 컨소시엘 구조를 파악하고, 멤버 배정 시 올바른 회사를 선택할 수 있어야 합니다.

## 선행 작업

## 수행 가이드

### Step 1: ProjectCompanyService에 buildTree 헬퍼 메서드 구현
- 대상 파일: `apps/backend/src/modules/project-company/project-company.service.ts`
- 작업 내용:
  ```typescript
  private buildTree(
    items: any[],
  ): ProjectCompanyTreeDto[] {
    // Map을 활용한 효율적인 트리 구성
    const map = new Map<string, ProjectCompanyTreeDto>();

    // 모든 노드를 맵에 저장
    items.forEach((item) => {
      map.set(item.id, {
        id: item.id,
        company: {
          id: item.company.id,
          name: item.company.name,
          bizNo: item.company.bizNo,
          address: item.company.address,
          tel: item.company.tel,
          ownerId: item.company.ownerId,
          createdAt: item.company.createdAt,
          updatedAt: item.company.updatedAt,
        },
        role: item.role,
        children: [],
      });
    });

    const roots: ProjectCompanyTreeDto[] = [];

    // 부모-자식 관계 설정
    items.forEach((item) => {
      const node = map.get(item.id);
      if (item.parentId) {
        const parent = map.get(item.parentId);
        if (parent) {
          parent.children.push(node);
        }
      } else {
        roots.push(node);
      }
    });

    return roots;
  }
  ```

### Step 2: ProjectCompanyService에 findTree 메서드 구현
- 대상 파일: `apps/backend/src/modules/project-company/project-company.service.ts`
- 작업 내용:
  ```typescript
  async findTree(projectId: string): Promise<{
    companies: ProjectCompanyTreeDto[];
    totalCount: number;
  }> {
    // 프로젝트 존재 확인
    const project = await this.prisma.project.findFirst({
      where: { id: projectId, deletedAt: null },
    });

    if (!project) {
      throw new NotFoundException({
        code: 404000,
        message: '프로젝트를 찾을 수 없습니다.',
      });
    }

    // 모든 ProjectCompany를 평탄하게 조회
    const items = await this.prisma.projectCompany.findMany({
      where: { projectId, deletedAt: null },
      include: { company: true },
      orderBy: [{ parentId: 'asc' }, { role: 'asc' }],
    });

    const totalCount = items.length;

    // 트리 구조로 변환
    const companies = this.buildTree(items);

    return { companies, totalCount };
  }
  ```

### Step 3: ProjectCompanyController에 GET 엔드포인트 추가
- 대상 파일: `apps/backend/src/modules/project-company/project-company.controller.ts`
- 작업 내용:
  ```typescript
  import { Get, Param, UseGuards } from '@nestjs/common';
  import { JwtAuthGuard } from '@/auth/guards/jwt-auth.guard';
  import { PermissionGuard } from '@/auth/guards/permission.guard';
  import { RequirePermission } from '@/common/decorators/require-permission.decorator';

  @Get()
  @UseGuards(JwtAuthGuard, PermissionGuard)
  @RequirePermission('COMPANY', 'VIEW')
  async findTree(@Param('projectId') projectId: string): Promise<{
    code: number;
    result: {
      companies: ProjectCompanyTreeDto[];
      totalCount: number;
    };
  }> {
    const result = await this.projectCompanyService.findTree(projectId);
    return {
      code: 200000,
      result,
    };
  }
  ```

## 완료 기준
- [ ] GET `/api/v1/projects/:projectId/companies` 엔드포인트 동작 확인
- [ ] 프로젝트의 모든 컨소시엘사를 트리 구조로 반환 (parentId 기반)
- [ ] 최상위 노드(parentId=null)가 루트 배열로 반환됨
- [ ] 각 노드는 children 배열로 자식 노드 포함
- [ ] soft delete 된 ProjectCompany 제외 (deletedAt IS NULL)
- [ ] 응답 형식: `{code:200000, result:{companies: ProjectCompanyTreeDto[], totalCount}}`
- [ ] 빈 프로젝트 시 빈 배열 반환 (totalCount: 0)
- [ ] 순환 참조 없이 정상적인 트리 구조 생성

## 참고
- buildTree: Map을 활용한 O(n) 시간 복잡도의 트리 구성
- 레이아웃: 먼저 모든 노드를 맵에 저장하고, 부모-자식 관계를 설정
- ProjectCompanyTreeDto: children 배열이 포함된 재귀 구조
- 프론트에서 쉽게 트리 시각화 가능 (D3, react-tree 등)
