# ProjectCompany 모듈 구조생성

## 예상 소요시간
| 구분 | 시간 |
|------|------|
| 사람 (숙련 개발자) | ~15분 |
| AI 에이전트 | ~5분 |

## What (무엇을)
`apps/backend/src/modules/project-company/` 디렉토리를 생성하고 NestJS 모듈 구조를 세팅합니다. ProjectCompany 모듈의 기본 프레임워크(module, controller, service, DTO)를 완성하고 AppModule에 등록합니다.

## Why (왜)
프로젝트의 컨소시엘 구조(주관사, 참여사, 하청사 등)를 관리하기 위한 기반입니다. 프로젝트마다 협력사의 트리 구조를 관리하고, 멤버 배정 시 해당 ProjectCompany를 선택하게 됩니다.

## 선행 작업
- `06.프로젝트-모듈/02.프로젝트-생성-API.md`

## 수행 가이드

### Step 1: ProjectCompany 모듈 디렉토리 및 파일 생성
- 대상 경로: `apps/backend/src/modules/project-company/`
- 작업 내용:
  ```bash
  mkdir -p apps/backend/src/modules/project-company/dto
  touch apps/backend/src/modules/project-company/project-company.module.ts
  touch apps/backend/src/modules/project-company/project-company.controller.ts
  touch apps/backend/src/modules/project-company/project-company.service.ts
  ```

### Step 2: project-company.module.ts 작성
- 대상 파일: `apps/backend/src/modules/project-company/project-company.module.ts`
- 작업 내용:
  ```typescript
  import { Module } from '@nestjs/common';
  import { PrismaService } from '@/prisma/prisma.service';
  import { ProjectCompanyController } from './project-company.controller';
  import { ProjectCompanyService } from './project-company.service';

  @Module({
    controllers: [ProjectCompanyController],
    providers: [ProjectCompanyService, PrismaService],
    exports: [ProjectCompanyService],
  })
  export class ProjectCompanyModule {}
  ```

### Step 5: dto/ 폴더 및 기본 DTO 작성
- 대상 파일: `apps/backend/src/modules/project-company/dto/project-company.dto.ts`
- 작업 내용:
  ```typescript
  import { CompanyDto } from '@/modules/company/dto/company.dto';

  export class ProjectCompanyDto {
    id: string;
    projectId: string;
    companyId: string;
    role: string; // CompanyRole enum: PRIME, PARTNER, SUB
    parentId?: string;
    createdAt: Date;
    updatedAt: Date;
  }

  export class CreateProjectCompanyDto {
    companyId: string;
    role: string;
    parentId?: string;
  }

  export class UpdateProjectCompanyDto {
    role?: string;
    parentId?: string;
  }

  export class ProjectCompanyTreeDto {
    id: string;
    company: CompanyDto;
    role: string;
    children: ProjectCompanyTreeDto[];
  }
  ```

### Step 6: AppModule에 ProjectCompanyModule 등록
- 대상 파일: `apps/backend/src/app.module.ts`
- 작업 내용:
  - imports 배열에 `ProjectCompanyModule` 추가
  - 예시:
    ```typescript
    import { ProjectCompanyModule } from '@/modules/project-company/project-company.module';

    @Module({
      imports: [
        // ... 기타 모듈들
        ProjectCompanyModule,
      ],
    })
    export class AppModule {}
    ```

## 참고
- CompanyRole enum: PRIME(주관사), PARTNER(참여사), SUB(하청사) 등으로 정의됨
- parentId: self-reference로 상위 회사를 지정 (null이면 최상위/주관사)
- ProjectCompanyTreeDto: 트리 구조로 반환할 때 사용 (children 배열 포함)
