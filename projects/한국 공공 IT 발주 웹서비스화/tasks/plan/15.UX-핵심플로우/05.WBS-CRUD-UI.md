# WBS 노드 생성/수정/삭제/진척률 입력 UI

## 예상 소요시간
| 구분 | 시간 |
|------|------|
| 사람 (숙련 개발자) | ~60분 |
| AI 에이전트 | ~25분 |

## What (무엇을)
`WbsTab`에 WBS 노드 추가(카테고리/태스크), 수정(제목, 가중치), 삭제, 진척률 입력 기능을 추가합니다. 카테고리 노드 아래에 하위 항목을 추가하고, 태스크 노드에 직접 진척률을 입력하면 상위 카테고리의 진척률이 자동 계산됩니다.

## Why (왜)
WBS(작업분류체계)는 프로젝트 진행 관리의 핵심입니다. 현재 WBS 탭은 트리를 읽기 전용으로만 표시하며, 노드를 추가하거나 진척률을 입력할 수 없습니다. WBS 관리 없이는 대시보드의 진척률 표시도 항상 0%입니다.

## 선행 작업
- `15.UX-핵심플로우/02.프로젝트-생성-회사선택.md` (프로젝트가 완전히 구성되어야 WBS 편집 가능)

## 수행 가이드

### Step 1: WBS 노드 추가 인라인 폼
- 대상 파일: `apps/frontend/src/pages/WbsTab.tsx`
- 작업 내용:
  - 각 카테고리 노드 옆에 "+" 버튼 추가 (하위 노드 생성)
  - 트리 상단에 "루트 카테고리 추가" 버튼 추가
  - 클릭 시 인라인 입력 폼 표시: 타입(CATEGORY/TASK), 제목, 가중치
  ```typescript
  interface AddNodeForm {
    parentId: number | null;
    type: 'CATEGORY' | 'TASK';
    title: string;
    weight: number;
  }

  const [addForm, setAddForm] = useState<AddNodeForm | null>(null);

  const handleAddNode = async () => {
    if (!addForm) return;
    try {
      await api.post(`/projects/${projectId}/wbs-nodes`, {
        parentId: addForm.parentId,
        type: addForm.type,
        title: addForm.title,
        weight: addForm.weight,
      });
      setAddForm(null);
      fetchNodes();
    } catch (err: any) {
      alert(err?.message || 'WBS 추가에 실패했습니다.');
    }
  };
  ```

### Step 2: 진척률 인라인 편집 (TASK 노드만)
- 대상 파일: `apps/frontend/src/pages/WbsTab.tsx`
- 작업 내용:
  - TASK 타입 노드의 진척률을 클릭하면 `<input type="number">` 로 변경
  - 0~100 범위 제한
  - onBlur 또는 Enter 시 PATCH API 호출
  ```typescript
  const [editingProgress, setEditingProgress] = useState<number | null>(null);

  const handleProgressUpdate = async (nodeId: number, progress: number) => {
    try {
      await api.patch(`/projects/${projectId}/wbs-nodes/${nodeId}`, { progress });
      fetchNodes(); // 상위 카테고리 자동 계산 반영
    } catch (err: any) {
      alert(err?.message || '진척률 수정에 실패했습니다.');
    }
    setEditingProgress(null);
  };
  ```

### Step 3: 노드 제목/가중치 수정
- 대상 파일: `apps/frontend/src/pages/WbsTab.tsx`
- 작업 내용:
  - 노드 제목 더블클릭 시 인라인 편집 모드
  - PATCH `/projects/:projectId/wbs-nodes/:nodeId` 호출
  ```typescript
  const [editingNode, setEditingNode] = useState<{ id: number; title: string; weight: number } | null>(null);

  const handleUpdateNode = async () => {
    if (!editingNode) return;
    try {
      await api.patch(`/projects/${projectId}/wbs-nodes/${editingNode.id}`, {
        title: editingNode.title,
        weight: editingNode.weight,
      });
      setEditingNode(null);
      fetchNodes();
    } catch (err: any) {
      alert(err?.message || '수정에 실패했습니다.');
    }
  };
  ```

### Step 4: 노드 삭제
- 대상 파일: `apps/frontend/src/pages/WbsTab.tsx`
- 작업 내용:
  - 각 노드에 삭제 버튼(x) 추가
  - 카테고리 삭제 시 하위 노드도 함께 삭제됨을 안내
  ```typescript
  const handleDeleteNode = async (nodeId: number, hasChildren: boolean) => {
    const msg = hasChildren
      ? '하위 항목도 모두 삭제됩니다. 계속하시겠습니까?'
      : '이 항목을 삭제하시겠습니까?';
    if (!confirm(msg)) return;
    try {
      await api.delete(`/projects/${projectId}/wbs-nodes/${nodeId}`);
      fetchNodes();
    } catch (err: any) {
      alert(err?.message || '삭제에 실패했습니다.');
    }
  };
  ```

### Step 5: fetchNodes 함수 분리 및 상태 관리
- 대상 파일: `apps/frontend/src/pages/WbsTab.tsx`
- 작업 내용:
  - 기존 useEffect 내부의 API 호출을 `fetchNodes` 함수로 분리
  - 추가/수정/삭제 후 재호출하여 트리 갱신
  ```typescript
  const fetchNodes = () => {
    if (!projectId) return;
    api.get<{ nodes: WbsNode[] }>(`/projects/${projectId}/wbs-nodes`)
      .then((res) => setNodes(res.result.nodes))
      .catch(() => {});
  };

  useEffect(() => { fetchNodes(); }, [projectId]);
  ```
