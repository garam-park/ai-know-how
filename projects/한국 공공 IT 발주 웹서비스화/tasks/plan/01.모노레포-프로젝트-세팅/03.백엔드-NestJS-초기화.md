# 백엔드 NestJS 초기화

## 예상 소요시간
| 구분 | 시간 |
|------|------|
| 사람 (숙련 개발자) | ~20분 |
| AI 에이전트 | ~8분 |

## What (무엇을)
`apps/backend` 디렉토리에 NestJS 프로젝트를 생성하고, TypeScript 설정을 구성하며, 공유 패키지(`@shared/types`)를 의존성으로 추가합니다. 또한 기본적인 애플리케이션 구조(modules, common 디렉토리)를 설정합니다.

## Why (왜)
NestJS는 엔터프라이즈급 백엔드 애플리케이션 개발을 위한 프레임워크입니다. 올바른 초기화를 통해 TypeScript 지원, 의존성 주입, 모듈 시스템 등의 기능을 활용할 수 있으며, 공유 패키지와의 연결로 프론트엔드와 일관된 타입을 사용할 수 있습니다.

## 선행 작업
- `01.루트-모노레포-초기화.md` 완료

## 수행 가이드

### Step 1: NestJS 프로젝트 생성
```bash
cd project-root
pnpm dlx @nestjs/cli@latest new backend --directory apps/backend --package-manager pnpm --skip-git
```

또는 수동으로 진행하는 경우:

```bash
cd project-root/apps
mkdir backend
cd backend
pnpm init
```

### Step 2: NestJS 필수 패키지 설치
백엔드 디렉토리에서 다음을 실행:

```bash
cd project-root/apps/backend
pnpm add @nestjs/core @nestjs/common @nestjs/platform-express reflect-metadata rxjs
pnpm add -D @types/express typescript @types/node ts-node @nestjs/cli
```

### Step 5: 백엔드 디렉토리 구조 생성
```bash
cd project-root/apps/backend
mkdir -p src/{modules,common,config}
```

### Step 6: 메인 진입점 파일 생성
- 대상 파일: `project-root/apps/backend/src/main.ts`

```typescript
import { NestFactory } from '@nestjs/core';
import { AppModule } from './app.module';

async function bootstrap() {
  const app = await NestFactory.create(AppModule);
  const port = process.env.PORT || 3000;
  await app.listen(port);
  console.log(`Backend server listening on http://localhost:${port}`);
}

bootstrap();
```

### Step 7: 기본 App Module 생성
- 대상 파일: `project-root/apps/backend/src/app.module.ts`

```typescript
import { Module } from '@nestjs/common';
import { ConfigModule } from '@nestjs/config';
import { AppController } from './app.controller';
import { AppService } from './app.service';

@Module({
  imports: [
    ConfigModule.forRoot({
      isGlobal: true,
      envFilePath: '../../.env',
    }),
  ],
  controllers: [AppController],
  providers: [AppService],
})
export class AppModule {}
```

### Step 8: App Controller 생성
- 대상 파일: `project-root/apps/backend/src/app.controller.ts`

```typescript
import { Controller, Get } from '@nestjs/common';
import { AppService } from './app.service';

@Controller('api')
export class AppController {
  constructor(private readonly appService: AppService) {}

  @Get('health')
  getHealth() {
    return this.appService.getHealth();
  }
}
```

### Step 9: App Service 생성
- 대상 파일: `project-root/apps/backend/src/app.service.ts`

```typescript
import { Injectable } from '@nestjs/common';

@Injectable()
export class AppService {
  getHealth() {
    return {
      status: 'ok',
      timestamp: new Date().toISOString(),
    };
  }
}
```

### Step 10: 공유 패키지 의존성 추가
```bash
cd project-root/apps/backend
pnpm add @shared/types@workspace:*
```

### Step 11: 개발 서버 실행 테스트
```bash
cd project-root/apps/backend
pnpm run dev
```

브라우저에서 `http://localhost:3000/api/health`에 접속하여 다음과 같은 응답이 나오는지 확인:

```json
{
  "status": "ok",
  "timestamp": "2026-04-01T..."
}
```

## 완료 기준

- [ ] `apps/backend/src/` 디렉토리에 main.ts, app.module.ts, app.controller.ts, app.service.ts 파일 존재
- [ ] `pnpm run dev` 명령으로 개발 서버 시작 성공
- [ ] `http://localhost:3000/api/health` 엔드포인트 응답 확인 (상태 200 OK)

## 참고
- NestJS CLI를 사용하면 자동으로 데코레이터와 기본 구조가 생성됨
- `ConfigModule.forRoot()`는 루트의 `.env` 파일을 읽도록 설정
- 향후 새로운 모듈은 `src/modules/` 디렉토리에 생성
- 공유 유틸리티나 인터셉터, 가드는 `src/common/` 디렉토리에 배치
