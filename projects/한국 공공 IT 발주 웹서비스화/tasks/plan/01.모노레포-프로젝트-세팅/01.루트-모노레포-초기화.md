# 루트 모노레포 초기화

## 예상 소요시간
| 구분 | 시간 |
|------|------|
| 사람 (숙련 개발자) | ~15분 |
| AI 에이전트 | ~5분 |

## What (무엇을)
pnpm workspace 루트를 초기화하고 monorepo 구조의 기초를 설정합니다. 이는 `pnpm-workspace.yaml` 파일을 생성하고, 루트 `package.json`을 구성하여 모든 하위 패키지와 앱들이 하나의 workspace로 통합되도록 하는 작업입니다.

## Why (왜)
모든 하위 패키지(`packages/shared`, `apps/backend`, `apps/frontend`)가 하나의 workspace로 연결되어야 `pnpm` 패키지 매니저가 의존성을 올바르게 관리할 수 있습니다. 루트 설정 없이는 하위 패키지들이 서로를 인식할 수 없으므로 workspace 프로토콜을 사용한 내부 의존성 참조도 불가능합니다.

## 선행 작업
- 없음 (최초 작업)

## 수행 가이드

### Step 1: 프로젝트 루트 디렉토리 생성 및 진입
```bash
mkdir -p project-root
cd project-root
```

### Step 2: pnpm으로 루트 package.json 초기화
```bash
pnpm init
```

생성된 `package.json`을 다음과 같이 수정합니다:
- 대상 파일: `project-root/package.json`
- 작업 내용:

```json
{
  "name": "korean-public-it-monorepo",
  "version": "1.0.0",
  "description": "한국 공공 IT 발주 웹서비스화 프로젝트",
  "private": true,
  "scripts": {
    "dev": "pnpm -r --parallel run dev",
    "build": "pnpm -r run build",
    "lint": "pnpm -r run lint",
    "format": "pnpm -r run format",
    "test": "pnpm -r run test"
  },
  "devDependencies": {}
}
```

**수정 설명:**
- `"private": true` — monorepo 루트는 npm에 배포되지 않음
- `-r` 플래그는 모든 workspace 패키지에 대해 명령 실행
- `--parallel` — 여러 패키지 명령을 병렬 실행 (dev 서버 동시 시작용)

### Step 3: pnpm-workspace.yaml 파일 생성
- 대상 파일: `project-root/pnpm-workspace.yaml`
- 작업 내용:

```yaml
packages:
  - 'packages/*'
  - 'apps/*'
```

**설명:**
- `packages/*` — `packages/shared`, `packages/...` 등 모든 공유 패키지 포함
- `apps/*` — `apps/backend`, `apps/frontend` 등 모든 앱 포함
- pnpm은 이 설정을 읽어 workspace 내 모든 패키지를 자동 감지

### Step 4: .gitignore 파일 생성
- 대상 파일: `project-root/.gitignore`
- 작업 내용:

```
node_modules
.pnpm-debug.log*
*.tsbuildinfo
dist
build
.env
.env.local
.DS_Store
.vscode
.idea
```

### Step 5: 초기 pnpm install 실행
```bash
cd project-root
pnpm install
```

이 명령은 루트의 `node_modules`를 설정하고 workspace 연결을 초기화합니다.

## 완료 기준
- [ ] `project-root/pnpm-workspace.yaml` 파일이 존재하고 올바른 구조
- [ ] `project-root/package.json`에 `"private": true` 설정되어 있음
- [ ] `project-root/package.json`에 dev, build, lint 스크립트 정의됨
- [ ] `pnpm install` 명령 실행 성공 (에러 없음)
- [ ] `node_modules` 디렉토리가 생성됨

## 참고
- pnpm 설치 필요: `npm install -g pnpm` (아직 설치되지 않은 경우)
- 이후 모든 패키지 설치는 `pnpm add` 또는 `pnpm --filter <패키지명> add`로 수행
- workspace 내에서 다른 패키지를 의존성으로 추가할 때는 `workspace:*` 프로토콜 사용
  예: `pnpm --filter backend add @shared/types@workspace:*`
