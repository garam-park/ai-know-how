# ESLint & Prettier 공통 설정

## 예상 소요시간
| 구분 | 시간 |
|------|------|
| 사람 (숙련 개발자) | ~20분 |
| AI 에이전트 | ~8분 |

## What (무엇을)
프로젝트 루트에 ESLint와 Prettier를 설치하고 공통 설정을 생성하여, 백엔드와 프론트엔드 모든 코드의 스타일과 포맷을 통일합니다. 각 앱에서 공통 설정을 상속받도록 구성합니다.

## Why (왜)
일관된 코드 스타일은 팀 협업과 코드 리뷰를 효율적으로 만들며, 특히 AI 에이전트가 생성하는 코드의 품질을 보장합니다. ESLint는 잠재적 버그를 감지하고, Prettier는 자동 포맷팅으로 스타일 논쟁을 제거합니다.

## 선행 작업
- `03.백엔드-NestJS-초기화.md` 완료
- `04.프론트엔드-React-초기화.md` 완료

## 수행 가이드

### Step 1: ESLint 및 Prettier 패키지 설치 (루트)
```bash
cd project-root
pnpm add -Dw eslint prettier @typescript-eslint/parser @typescript-eslint/eslint-plugin eslint-config-prettier
```

**패키지 설명:**
- `eslint` — 코드 분석 및 오류 감지
- `prettier` — 코드 자동 포맷팅
- `@typescript-eslint/parser` — TypeScript 파싱
- `@typescript-eslint/eslint-plugin` — TypeScript 전용 ESLint 규칙
- `eslint-config-prettier` — Prettier와 ESLint 충돌 해결

### Step 2: ESLint 설정 파일 생성
- 대상 파일: `project-root/.eslintrc.json`
- 작업 내용:

```json
{
  "parser": "@typescript-eslint/parser",
  "parserOptions": {
    "ecmaVersion": 2020,
    "sourceType": "module"
  },
  "extends": [
    "eslint:recommended",
    "plugin:@typescript-eslint/recommended",
    "prettier"
  ],
  "plugins": [
    "@typescript-eslint"
  ],
  "rules": {
    "@typescript-eslint/explicit-function-return-types": "warn",
    "@typescript-eslint/no-unused-vars": [
      "warn",
      {
        "argsIgnorePattern": "^_"
      }
    ],
    "@typescript-eslint/no-explicit-any": "warn",
    "no-console": [
      "warn",
      {
        "allow": [
          "warn",
          "error"
        ]
      }
    ]
  },
  "overrides": [
    {
      "files": [
        "*.tsx"
      ],
      "parserOptions": {
        "ecmaFeatures": {
          "jsx": true
        }
      }
    }
  ]
}
```

**규칙 설명:**
- `@typescript-eslint/explicit-function-return-types` — 함수의 반환 타입 명시 권장
- `@typescript-eslint/no-unused-vars` — 사용하지 않는 변수 경고 (`_`로 시작하면 무시)
- `no-console` — console 사용 제한 (warn, error는 허용)

### Step 3: Prettier 설정 파일 생성
- 대상 파일: `project-root/.prettierrc`
- 작업 내용:

```json
{
  "semi": true,
  "singleQuote": true,
  "trailingComma": "es5",
  "printWidth": 100,
  "tabWidth": 2,
  "useTabs": false,
  "arrowParens": "always",
  "endOfLine": "lf"
}
```

**옵션 설명:**
- `semi: true` — 문장 끝에 세미콜론 추가
- `singleQuote: true` — 싱글 쿼트 사용
- `trailingComma: "es5"` — 배열/객체 마지막에 쉼표 (ES5 호환)
- `printWidth: 100` — 한 줄 최대 길이 100자
- `tabWidth: 2` — 들여쓰기 2칸
- `endOfLine: "lf"` — Unix 줄 끝 (cross-platform 호환)

### Step 4: Prettier 무시 파일 생성 (선택)
- 대상 파일: `project-root/.prettierignore`
- 작업 내용:

```
node_modules
dist
build
.next
coverage
.env
.env.local
*.lock
*.tsbuildinfo
```

### Step 5: ESLint 무시 파일 생성 (선택)
- 대상 파일: `project-root/.eslintignore`
- 작업 내용:

```
node_modules
dist
build
.next
coverage
*.min.js
```

### Step 6: 루트 package.json에 lint/format 스크립트 추가
- 대상 파일: `project-root/package.json` (기존 스크립트 확장)
- 수정 항목:

```json
{
  "name": "korean-public-it-monorepo",
  "version": "1.0.0",
  "description": "한국 공공 IT 발주 웹서비스화 프로젝트",
  "private": true,
  "scripts": {
    "dev": "pnpm -r --parallel run dev",
    "build": "pnpm -r run build",
    "lint": "eslint . --ext .ts,.tsx",
    "lint:fix": "eslint . --ext .ts,.tsx --fix",
    "format": "prettier --write \"**/*.{ts,tsx,json,md}\"",
    "format:check": "prettier --check \"**/*.{ts,tsx,json,md}\"",
    "test": "pnpm -r run test"
  },
  "devDependencies": {
    "eslint": "^8.0.0",
    "prettier": "^3.0.0",
    "@typescript-eslint/parser": "^6.0.0",
    "@typescript-eslint/eslint-plugin": "^6.0.0",
    "eslint-config-prettier": "^9.0.0"
  }
}
```

**스크립트 설명:**
- `pnpm lint` — 모든 코드 스타일 검사
- `pnpm lint:fix` — ESLint 오류 자동 수정
- `pnpm format` — 코드 포맷팅
- `pnpm format:check` — 포맷팅 검사 (수정 X)

### Step 7: 백엔드 개별 ESLint 설정 (선택)
각 앱이 공통 설정을 상속받으려면, 앱 디렉토리에 `.eslintrc.json` 생성:

- 대상 파일: `project-root/apps/backend/.eslintrc.json`

```json
{
  "extends": ["../../.eslintrc.json"]
}
```

- 대상 파일: `project-root/apps/frontend/.eslintrc.json`

```json
{
  "extends": ["../../.eslintrc.json"],
  "parserOptions": {
    "ecmaFeatures": {
      "jsx": true
    }
  }
}
```

### Step 8: 각 앱 package.json에 lint/format 스크립트 추가 (선택)

**백엔드:**
- 대상 파일: `project-root/apps/backend/package.json`

```json
{
  "scripts": {
    "lint": "eslint src/**/*.ts",
    "lint:fix": "eslint src/**/*.ts --fix",
    "format": "prettier --write src/**/*.ts"
  }
}
```

**프론트엔드:**
- 대상 파일: `project-root/apps/frontend/package.json`

```json
{
  "scripts": {
    "lint": "eslint src/**/*.{ts,tsx}",
    "lint:fix": "eslint src/**/*.{ts,tsx} --fix",
    "format": "prettier --write src/**/*.{ts,tsx}"
  }
}
```

### Step 9: Lint 및 Format 실행
모든 코드 검사:

```bash
cd project-root
pnpm lint
```

자동 수정:

```bash
cd project-root
pnpm lint:fix
pnpm format
```

## 참고
- IDE 확장 프로그램 설치 권장:
  - VS Code: ESLint, Prettier - Code formatter
  - 설정: "Format On Save" 체크 (자동 포맷팅)
- Pre-commit 훅 설정 가능 (husky + lint-staged)로 커밋 시 자동 검사
- CI/CD 파이프라인에 `pnpm lint` 및 `pnpm format:check` 포함 권장
- 규칙 충돌 시 팀과 합의하여 `.eslintrc.json` 또는 `.prettierrc` 수정
