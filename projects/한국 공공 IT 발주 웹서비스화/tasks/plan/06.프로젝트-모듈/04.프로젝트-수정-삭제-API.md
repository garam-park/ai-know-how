# 프로젝트 수정 및 삭제 API

## 예상 소요시간
| 구분 | 시간 |
|------|------|
| 사람 (숙련 개발자) | ~20분 |
| AI 에이전트 | ~8분 |

## What (무엇을)
PATCH `/api/v1/projects/:id` (수정)과 DELETE `/api/v1/projects/:id` (삭제) 엔드포인트를 구현합니다. 프로젝트 정보를 변경하고 소프트 삭제합니다. 수정/삭제는 권한 검사를 거칩니다.

## Why (왜)
프로젝트의 예산, 기간, WBS 깊이 등을 변경해야 할 때 수정 API가 필요합니다. 완료되거나 취소된 프로젝트는 삭제하되 데이터 추적을 위해 소프트 삭제를 합니다.

## 수행 가이드

### Step 1: ProjectService에 update 메서드 구현
- 대상 파일: `apps/backend/src/modules/project/project.service.ts`
- 작업 내용:
  ```typescript
  async update(
    id: string,
    dto: UpdateProjectDto,
    userId: string,
  ): Promise<ProjectDto> {
    // 프로젝트 존재 확인
    const project = await this.prisma.project.findFirst({
      where: { id, deletedAt: null },
    });

    if (!project) {
      throw new NotFoundException({
        code: 404000,
        message: '프로젝트를 찾을 수 없습니다.',
      });
    }

    // 권한 검사: @RequirePermission(Resource.PROJECT, Action.EDIT)
    // 권한 검사는 컨트롤러의 가드에서 수행

    const updated = await this.prisma.project.update({
      where: { id },
      data: {
        ...(dto.name && { name: dto.name }),
        ...(dto.code !== undefined && { code: dto.code }),
        ...(dto.description !== undefined && { description: dto.description }),
        ...(dto.startDate && { startDate: dto.startDate }),
        ...(dto.endDate && { endDate: dto.endDate }),
        ...(dto.budget !== undefined && { budget: dto.budget }),
        ...(dto.maxWbsDepth && { maxWbsDepth: dto.maxWbsDepth }),
      },
    });

    return {
      id: updated.id,
      name: updated.name,
      code: updated.code,
      description: updated.description,
      startDate: updated.startDate,
      endDate: updated.endDate,
      budget: updated.budget,
      maxWbsDepth: updated.maxWbsDepth,
      createdAt: updated.createdAt,
      updatedAt: updated.updatedAt,
    };
  }
  ```

### Step 2: ProjectService에 delete (soft delete) 메서드 구현
- 대상 파일: `apps/backend/src/modules/project/project.service.ts`
- 작업 내용:
  ```typescript
  async delete(id: string, userId: string): Promise<void> {
    // 프로젝트 존재 확인
    const project = await this.prisma.project.findFirst({
      where: { id, deletedAt: null },
    });

    if (!project) {
      throw new NotFoundException({
        code: 404000,
        message: '프로젝트를 찾을 수 없습니다.',
      });
    }

    // 권한 검사: @RequirePermission(Resource.PROJECT, Action.SETTING)
    // 권한 검사는 컨트롤러의 가드에서 수행

    // Soft delete
    await this.prisma.project.update({
      where: { id },
      data: { deletedAt: new Date() },
    });
  }
  ```

### Step 3: ProjectController에 PATCH/:id 엔드포인트 추가
- 대상 파일: `apps/backend/src/modules/project/project.controller.ts`
- 작업 내용:
  ```typescript
  import { Patch, Delete, Body, Param, UseGuards } from '@nestjs/common';
  import { PermissionGuard } from '@/auth/guards/permission.guard';
  import { RequirePermission } from '@/common/decorators/require-permission.decorator';

  @Patch(':id')
  @UseGuards(JwtAuthGuard, PermissionGuard)
  @RequirePermission('PROJECT', 'EDIT')
  async update(
    @Param('id') id: string,
    @Body() dto: UpdateProjectDto,
    @CurrentUser() user: any,
  ): Promise<{
    code: number;
    result: { project: ProjectDto };
  }> {
    const project = await this.projectService.update(id, dto, user.id);
    return {
      code: 200000,
      result: { project },
    };
  }
  ```

### Step 4: ProjectController에 DELETE/:id 엔드포인트 추가
- 대상 파일: `apps/backend/src/modules/project/project.controller.ts`
- 작업 내용:
  ```typescript
  @Delete(':id')
  @UseGuards(JwtAuthGuard, PermissionGuard)
  @RequirePermission('PROJECT', 'SETTING')
  async delete(
    @Param('id') id: string,
    @CurrentUser() user: any,
  ): Promise<{
    code: number;
    message: string;
  }> {
    await this.projectService.delete(id, user.id);
    return {
      code: 200000,
      message: '프로젝트가 삭제되었습니다.',
    };
  }
  ```

## 참고
- UpdateProjectDto: 모든 필드 선택 사항 (partial)
- Soft delete: deleted_at 필드에 삭제 시간 기록하여 복구 가능하게 함
- PermissionGuard와 @RequirePermission: 권한 검사 자동 수행
- Resource.PROJECT, Action.EDIT/SETTING: 권한 상수 (별도 정의 필요)
