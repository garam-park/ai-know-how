# Toast 알림 시스템

## 예상 소요시간
| 구분 | 시간 |
|------|------|
| 사람 (숙련 개발자) | ~30분 |
| AI 에이전트 | ~12분 |

## What (무엇을)
전역 Toast 알림 시스템을 구현합니다. 성공/에러/경고 메시지를 화면 우측 상단에 자동 소멸되는 토스트로 표시합니다. React Context 기반으로 앱 어디서든 `useToast()` 훅으로 호출할 수 있습니다.

## Why (왜)
현재 성공/에러 피드백은 `alert()`이나 인라인 에러 텍스트뿐입니다. "등록 성공", "삭제 완료" 같은 피드백이 없으면 사용자가 작업 성공 여부를 파악하기 어렵습니다. Toast는 UX의 기본이며, 전 페이지에서 일관된 피드백을 제공합니다.

## 선행 작업

## 수행 가이드

### Step 1: Toast Context 및 Provider 생성
- 대상 파일: `apps/frontend/src/contexts/ToastContext.tsx` (신규)
- 작업 내용:
  ```typescript
  import { createContext, useContext, useState, useCallback, ReactNode } from 'react';

  type ToastType = 'success' | 'error' | 'warning' | 'info';

  interface Toast {
    id: number;
    type: ToastType;
    message: string;
  }

  interface ToastContextType {
    toasts: Toast[];
    addToast: (type: ToastType, message: string) => void;
    removeToast: (id: number) => void;
  }

  const ToastContext = createContext<ToastContextType | null>(null);
  let toastId = 0;

  export function ToastProvider({ children }: { children: ReactNode }) {
    const [toasts, setToasts] = useState<Toast[]>([]);

    const addToast = useCallback((type: ToastType, message: string) => {
      const id = ++toastId;
      setToasts((prev) => [...prev, { id, type, message }]);
      setTimeout(() => {
        setToasts((prev) => prev.filter((t) => t.id !== id));
      }, 3000);
    }, []);

    const removeToast = useCallback((id: number) => {
      setToasts((prev) => prev.filter((t) => t.id !== id));
    }, []);

    return (
      <ToastContext.Provider value={{ toasts, addToast, removeToast }}>
        {children}
        <ToastContainer toasts={toasts} onRemove={removeToast} />
      </ToastContext.Provider>
    );
  }

  export function useToast() {
    const ctx = useContext(ToastContext);
    if (!ctx) throw new Error('useToast must be used within ToastProvider');
    return ctx;
  }
  ```

### Step 2: ToastContainer 렌더링 컴포넌트
- 대상 파일: `apps/frontend/src/contexts/ToastContext.tsx` (같은 파일 하단)
- 작업 내용:
  ```typescript
  function ToastContainer({ toasts, onRemove }: { toasts: Toast[]; onRemove: (id: number) => void }) {
    const colors: Record<ToastType, { bg: string; border: string }> = {
      success: { bg: '#f0fff4', border: '#38a169' },
      error: { bg: '#fff5f5', border: '#e53e3e' },
      warning: { bg: '#fffff0', border: '#d69e2e' },
      info: { bg: '#ebf8ff', border: '#2b6cb0' },
    };

    return (
      <div style={{ position: 'fixed', top: 16, right: 16, zIndex: 2000, display: 'flex', flexDirection: 'column', gap: 8 }}>
        {toasts.map((toast) => (
          <div key={toast.id} onClick={() => onRemove(toast.id)}
            style={{ padding: '12px 16px', background: colors[toast.type].bg, borderLeft: `4px solid ${colors[toast.type].border}`,
              borderRadius: 4, boxShadow: '0 2px 8px rgba(0,0,0,0.15)', cursor: 'pointer', maxWidth: 360, fontSize: 14 }}>
            {toast.message}
          </div>
        ))}
      </div>
    );
  }
  ```

### Step 3: App.tsx에 ToastProvider 추가
- 대상 파일: `apps/frontend/src/App.tsx`
- 작업 내용:
  ```typescript
  import { ToastProvider } from './contexts/ToastContext';

  // AuthProvider 내부에 ToastProvider 감싸기
  <AuthProvider>
    <ToastProvider>
      <Routes>...</Routes>
    </ToastProvider>
  </AuthProvider>
  ```

### Step 4: 기존 alert()/인라인 에러를 Toast로 교체
- 대상 파일 (점진적 교체):
  - 모든 `alert(err?.message || ...)` → `addToast('error', ...)`
  - 성공 콜백에 `addToast('success', '등록되었습니다.')` 추가
  - 예시:
  ```typescript
  const { addToast } = useToast();

  // 성공 시
  addToast('success', '회사가 등록되었습니다.');

  // 에러 시
  addToast('error', err?.message || '등록에 실패했습니다.');
  ```
