# Permission 시드데이터

## 예상 소요시간
| 구분 | 시간 |
|------|------|
| 사람 (숙련 개발자) | ~20분 |
| AI 에이전트 | ~8분 |

## What (무엇을)
Permission 테이블에 12개 Resource와 11개 Action의 조합으로 정의된 31개의 권한(permission) 데이터를 삽입합니다.

## Why (왜)
모든 API의 권한 체크가 이 Permission 데이터를 기반으로 하므로, 정확한 권한 목록 정의가 필수입니다. Role과 Permission의 매핑을 통해 각 역할별 권한을 제어합니다.

### Step 2: Seed 파일에 Permission 시드데이터 추가
- 대상 파일: `apps/backend/prisma/seed.ts`
- 작업 내용: 기존 seed.ts의 main() 함수에 Permission 삽입 로직 추가
  ```typescript
  async function main() {
    // ... Role 삽입 코드 (앞서 작성한 코드) ...

    // Permission 데이터 삽입
    const permissions = [
      { resource: 'WBS', action: 'VIEW' },
      { resource: 'WBS', action: 'EDIT' },
      { resource: 'REPORT', action: 'VIEW' },
      { resource: 'REPORT', action: 'CREATE' },
      { resource: 'REPORT', action: 'DOWNLOAD' },
      { resource: 'TIMESHEET', action: 'VIEW' },
      { resource: 'TIMESHEET', action: 'EDIT' },
      { resource: 'TIMESHEET', action: 'DOWNLOAD' },
      { resource: 'MEMBER', action: 'VIEW' },
      { resource: 'MEMBER', action: 'INVITE' },
      { resource: 'MEMBER', action: 'REMOVE' },
      { resource: 'MEMBER', action: 'PERMISSION' },
      { resource: 'COMPANY', action: 'VIEW' },
      { resource: 'COMPANY', action: 'MANAGE' },
      { resource: 'PROJECT', action: 'VIEW' },
      { resource: 'PROJECT', action: 'EDIT' },
      { resource: 'PROJECT', action: 'SETTING' },
      { resource: 'DELIVERABLE', action: 'VIEW' },
      { resource: 'DELIVERABLE', action: 'UPLOAD' },
      { resource: 'DELIVERABLE', action: 'APPROVE' },
      { resource: 'ISSUE', action: 'VIEW' },
      { resource: 'ISSUE', action: 'MANAGE' },
      { resource: 'RISK', action: 'VIEW' },
      { resource: 'RISK', action: 'MANAGE' },
      { resource: 'CHANGE', action: 'VIEW' },
      { resource: 'CHANGE', action: 'REQUEST' },
      { resource: 'CHANGE', action: 'APPROVE' },
      { resource: 'AUDIT', action: 'VIEW' },
      { resource: 'AUDIT', action: 'MANAGE' },
      { resource: 'CLOSE', action: 'VIEW' },
      { resource: 'CLOSE', action: 'MANAGE' },
    ];

    for (let i = 0; i < permissions.length; i++) {
      const perm = permissions[i];
      await prisma.permission.upsert({
        where: {
          resource_action: {
            resource: perm.resource as any,
            action: perm.action as any,
          },
        },
        update: {},
        create: {
          resource: perm.resource as any,
          action: perm.action as any,
          description: `${perm.resource} ${perm.action}`,
        },
      });
    }

    console.log('Permission 시드데이터 삽입 완료');
  }
  ```

## 참고
- Permission 테이블의 unique 제약조건은 (resource, action) 조합입니다.
- upsert를 사용하여 동일 시드 실행 시에도 중복 생성되지 않습니다.
- 이 31개 Permission은 고정값이며, 추후 비즈니스 요구사항 변경 시에만 수정됩니다.
