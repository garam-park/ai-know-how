# 로그인 JWT 발급 API

## 예상 소요시간
| 구분 | 시간 |
|------|------|
| 사람 (숙련 개발자) | ~40분 |
| AI 에이전트 | ~15분 |

## What (무엇을)
POST /api/v1/auth/login 엔드포인트를 구현하여 사용자가 이메일과 비밀번호로 로그인하면 accessToken(15분 만료)과 refreshToken(7일 만료)을 발급합니다.

## Why (왜)
인증된 API 요청의 기반이 되는 JWT 토큰을 발급합니다. accessToken은 단기, refreshToken은 장기로 설정하여 보안과 사용성의 균형을 맞춥니다.

### Step 3: AuthService에 login 메서드 추가
- 대상 파일: `apps/backend/src/modules/auth/auth.service.ts`
- 작업 내용: 로그인 비즈니스 로직 추가
  ```typescript
  import { Injectable, ConflictException, UnauthorizedException } from '@nestjs/common';
  import { JwtService } from '@nestjs/jwt';
  import * as bcrypt from 'bcrypt';
  import { PrismaService } from '../prisma/prisma.service';
  import { CreateUserDto } from './dto/create-user.dto';
  import { LoginDto } from './dto/login.dto';
  import { UserDto } from './dto/user.dto';
  import { JwtPayload } from './types/jwt-payload.type';

  @Injectable()
  export class AuthService {
    constructor(
      private prisma: PrismaService,
      private jwtService: JwtService,
    ) {}

    // ... register 메서드 ...

    async login(loginDto: LoginDto) {
      const { email, password } = loginDto;

      // 1. 이메일로 사용자 조회
      const user = await this.prisma.user.findUnique({
        where: { email },
      });

      if (!user) {
        throw new UnauthorizedException({
          code: 401000,
          message: '이메일 또는 비밀번호가 올바르지 않습니다',
        });
      }

      // 2. 비밀번호 검증
      const isPasswordValid = await bcrypt.compare(password, user.password);
      if (!isPasswordValid) {
        throw new UnauthorizedException({
          code: 401000,
          message: '이메일 또는 비밀번호가 올바르지 않습니다',
        });
      }

      // 3. JWT 토큰 생성
      const jwtPayload: JwtPayload = {
        sub: user.id,
        email: user.email,
      };

      // accessToken: 15분 만료
      const accessToken = this.jwtService.sign(jwtPayload, {
        expiresIn: '15m',
      });

      // refreshToken: 7일 만료
      const refreshToken = this.jwtService.sign(jwtPayload, {
        expiresIn: '7d',
      });

      return {
        accessToken,
        refreshToken,
        user: this.toUserDto(user),
      };
    }

    private toUserDto(user: any): UserDto {
      return {
        id: user.id,
        email: user.email,
        name: user.name,
        tel: user.tel,
        createdAt: user.createdAt,
      };
    }
  }
  ```

### Step 4: AuthController에 login 엔드포인트 추가
- 대상 파일: `apps/backend/src/modules/auth/auth.controller.ts`
- 작업 내용: 로그인 API 엔드포인트 추가
  ```typescript
  import { Controller, Post, Body } from '@nestjs/common';
  import { AuthService } from './auth.service';
  import { CreateUserDto } from './dto/create-user.dto';
  import { LoginDto } from './dto/login.dto';

  @Controller('api/v1/auth')
  export class AuthController {
    constructor(private readonly authService: AuthService) {}

    @Post('register')
    async register(@Body() createUserDto: CreateUserDto) {
      const user = await this.authService.register(createUserDto);
      return {
        code: 201000,
        result: {
          user,
        },
      };
    }

    @Post('login')
    async login(@Body() loginDto: LoginDto) {
      const { accessToken, refreshToken, user } = await this.authService.login(loginDto);
      return {
        code: 200000,
        result: {
          accessToken,
          refreshToken,
          user,
        },
      };
    }
  }
  ```

### Step 6: AuthModule의 JwtModule 설정 업데이트
- 대상 파일: `apps/backend/src/modules/auth/auth.module.ts`
- 작업 내용: 환경 변수 기반 JwtModule 설정
  ```typescript
  import { Module } from '@nestjs/common';
  import { JwtModule } from '@nestjs/jwt';
  import { PassportModule } from '@nestjs/passport';
  import { ConfigModule, ConfigService } from '@nestjs/config';
  import { AuthController } from './auth.controller';
  import { AuthService } from './auth.service';
  import { PrismaModule } from '../prisma/prisma.module';

  @Module({
    imports: [
      PassportModule,
      JwtModule.registerAsync({
        imports: [ConfigModule],
        inject: [ConfigService],
        useFactory: (configService: ConfigService) => ({
          secret: configService.get<string>('JWT_SECRET') || 'dev-secret-key',
          signOptions: {
            expiresIn: configService.get<string>('JWT_EXPIRATION') || '15m',
          },
        }),
      }),
      PrismaModule,
    ],
    controllers: [AuthController],
    providers: [AuthService],
    exports: [AuthService, JwtModule],
  })
  export class AuthModule {}
  ```

### Step 9: JWT 토큰 검증 테스트
- 작업 내용: jwt.io 또는 온라인 JWT 디코더에서 토큰 검증

## 참고
- JWT 토큰은 Bearer 스키마로 사용됩니다: `Authorization: Bearer <accessToken>`
- refreshToken은 클라이언트에서 안전하게 저장되어야 합니다 (HttpOnly 쿠키 권장).
- 프로덕션 환경에서는 JWT_SECRET을 안전한 값으로 변경해야 합니다.
- 토큰 발급 후 데이터베이스에 별도로 저장할 필요는 없습니다 (stateless 인증).
