# 토큰 갱신 API

## 예상 소요시간
| 구분 | 시간 |
|------|------|
| 사람 (숙련 개발자) | ~30분 |
| AI 에이전트 | ~10분 |

## What (무엇을)
POST /api/v1/auth/refresh 엔드포인트를 구현하여 유효한 refreshToken을 제출하면 새로운 accessToken과 refreshToken을 발급합니다.

## Why (왜)
accessToken이 만료되면 사용자가 다시 로그인하지 않고도 refreshToken으로 새 토큰을 발급받아 세션을 지속할 수 있습니다. 이는 사용자 경험을 개선하고 보안을 강화합니다.

## 수행 가이드

### Step 1: RefreshDto 작성
- 대상 파일: `apps/backend/src/modules/auth/dto/refresh.dto.ts`
- 작업 내용: 토큰 갱신 요청 DTO 정의
  ```typescript
  import { IsString } from 'class-validator';

  export class RefreshDto {
    @IsString()
    refreshToken: string;
  }
  ```

### Step 2: AuthService에 refresh 메서드 추가
- 대상 파일: `apps/backend/src/modules/auth/auth.service.ts`
- 작업 내용: 토큰 갱신 비즈니스 로직 추가
  ```typescript
  import { Injectable, UnauthorizedException } from '@nestjs/common';
  import { JwtService } from '@nestjs/jwt';
  import { PrismaService } from '../prisma/prisma.service';
  import { JwtPayload } from './types/jwt-payload.type';

  @Injectable()
  export class AuthService {
    constructor(
      private prisma: PrismaService,
      private jwtService: JwtService,
    ) {}

    // ... register, login 메서드 ...

    async refresh(refreshToken: string) {
      try {
        // 1. refreshToken 검증
        const decoded = this.jwtService.verify(refreshToken) as JwtPayload;

        // 2. 사용자 존재 여부 확인
        const user = await this.prisma.user.findUnique({
          where: { id: decoded.sub },
        });

        if (!user) {
          throw new UnauthorizedException({
            code: 401001,
            message: '사용자를 찾을 수 없습니다',
          });
        }

        // 3. 새로운 JWT 토큰 생성
        const jwtPayload: JwtPayload = {
          sub: user.id,
          email: user.email,
        };

        const newAccessToken = this.jwtService.sign(jwtPayload, {
          expiresIn: '15m',
        });

        const newRefreshToken = this.jwtService.sign(jwtPayload, {
          expiresIn: '7d',
        });

        return {
          accessToken: newAccessToken,
          refreshToken: newRefreshToken,
        };
      } catch (error) {
        if (error instanceof UnauthorizedException) {
          throw error;
        }

        // JWT 검증 실패 (만료, 위변조 등)
        throw new UnauthorizedException({
          code: 401001,
          message: '유효하지 않거나 만료된 토큰입니다',
        });
      }
    }
  }
  ```

### Step 3: AuthController에 refresh 엔드포인트 추가
- 대상 파일: `apps/backend/src/modules/auth/auth.controller.ts`
- 작업 내용: 토큰 갱신 API 엔드포인트 추가
  ```typescript
  import { Controller, Post, Body } from '@nestjs/common';
  import { AuthService } from './auth.service';
  import { RefreshDto } from './dto/refresh.dto';

  @Controller('api/v1/auth')
  export class AuthController {
    constructor(private readonly authService: AuthService) {}

    // ... register, login 메서드 ...

    @Post('refresh')
    async refresh(@Body() refreshDto: RefreshDto) {
      const { accessToken, refreshToken } = await this.authService.refresh(
        refreshDto.refreshToken,
      );

      return {
        code: 200000,
        result: {
          accessToken,
          refreshToken,
        },
      };
    }
  }
  ```

### Step 4: API 테스트
- 작업 내용: 토큰 갱신 API 테스트 시나리오

  **시나리오 1: 정상 refresh token으로 새 토큰 발급**
  ```bash
  # 먼저 로그인하여 refreshToken 획득
  curl -X POST http://localhost:3000/api/v1/auth/login \
    -H "Content-Type: application/json" \
    -d '{
      "email": "test@example.com",
      "password": "password123"
    }'
  
  # 응답에서 refreshToken 값 복사 후 사용
  curl -X POST http://localhost:3000/api/v1/auth/refresh \
    -H "Content-Type: application/json" \
    -d '{
      "refreshToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
    }'
  ```

  정상 응답:
  ```json
  {
    "code": 200000,
    "result": {
      "accessToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
      "refreshToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
    }
  }
  ```

  **시나리오 2: 잘못된 refreshToken**
  ```bash
  curl -X POST http://localhost:3000/api/v1/auth/refresh \
    -H "Content-Type: application/json" \
    -d '{
      "refreshToken": "invalid.token.here"
    }'
  ```

  에러 응답:
  ```json
  {
    "code": 401001,
    "message": "유효하지 않거나 만료된 토큰입니다"
  }
  ```

  **시나리오 3: 만료된 refreshToken**
  ```bash
  # 이미 7일 이상 지난 refreshToken 사용
  curl -X POST http://localhost:3000/api/v1/auth/refresh \
    -H "Content-Type: application/json" \
    -d '{
      "refreshToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9... (expired)"
    }'
  ```

  에러 응답:
  ```json
  {
    "code": 401001,
    "message": "유효하지 않거나 만료된 토큰입니다"
  }
  ```

### Step 5: 토큰 갱신 흐름 검증
- 작업 내용: 토큰 갱신 흐름을 다음과 같이 검증합니다:
  1. 로그인 → accessToken (15m 만료), refreshToken (7d 만료) 획득
  2. accessToken이 만료되기 전에 refresh 호출 → 새 accessToken, refreshToken 발급
  3. 새 토큰들이 유효하고 exp 시간이 갱신되었는지 확인
  4. 이전 refreshToken으로 다시 refresh 호출 → 401001 에러 반환되어야 함 (선택사항: 이전 토큰 재사용 방지)

## 참고
- refreshToken 자체도 갱신되므로, 클라이언트는 매번 새 refreshToken을 저장해야 합니다.
- refreshToken이 만료되면 사용자는 다시 로그인해야 합니다.
- 토큰 갱신 시점은 accessToken 만료 전에 해야 합니다 (클라이언트에서 관리).
- 향후 Redis를 이용한 refreshToken 블랙리스트 기능을 추가할 수 있습니다 (토큰 강제 무효화).
