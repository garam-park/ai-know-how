# 회원가입 API

## 예상 소요시간
| 구분 | 시간 |
|------|------|
| 사람 (숙련 개발자) | ~30분 |
| AI 에이전트 | ~10분 |

## What (무엇을)
POST /api/v1/auth/register 엔드포인트를 구현하여 사용자가 이메일, 비밀번호, 이름, 전화번호로 계정을 생성할 수 있게 합니다.

## Why (왜)
사용자 계정 생성이 모든 기능의 전제입니다. 회원가입 API를 통해 새로운 사용자가 시스템에 등록되어 인증 및 권한 관리의 대상이 됩니다.

## 수행 가이드

### Step 1: CreateUserDto 보강
- 대상 파일: `apps/backend/src/modules/auth/dto/create-user.dto.ts`
- 작업 내용: validation decorators 추가
  ```typescript
  import { IsEmail, IsString, MinLength, IsOptional, IsMobilePhone } from 'class-validator';

  export class CreateUserDto {
    @IsEmail()
    email: string;

    @IsString()
    @MinLength(8, { message: '비밀번호는 최소 8자 이상이어야 합니다' })
    password: string;

    @IsString()
    name: string;

    @IsOptional()
    @IsMobilePhone('ko-KR', {}, { message: '유효한 한국 전화번호 형식이 아닙니다' })
    tel?: string;
  }
  ```

### Step 2: UserDto 보강
- 대상 파일: `apps/backend/src/modules/auth/dto/user.dto.ts`
- 작업 내용: 응답 DTO 정의

### Step 3: AuthService에 register 메서드 구현
- 대상 파일: `apps/backend/src/modules/auth/auth.service.ts`
- 작업 내용: 회원가입 비즈니스 로직 구현
  ```typescript
  import { Injectable, ConflictException, BadRequestException } from '@nestjs/common';
  import * as bcrypt from 'bcrypt';
  import { PrismaService } from '../prisma/prisma.service';
  import { CreateUserDto } from './dto/create-user.dto';
  import { UserDto } from './dto/user.dto';

  @Injectable()
  export class AuthService {
    constructor(private prisma: PrismaService) {}

    async register(createUserDto: CreateUserDto): Promise<UserDto> {
      const { email, password, name, tel } = createUserDto;

      // 1. 이메일 중복 체크
      const existingUser = await this.prisma.user.findUnique({
        where: { email },
      });

      if (existingUser) {
        throw new ConflictException({
          code: 409000,
          message: '이미 등록된 이메일입니다',
        });
      }

      // 2. 비밀번호 해싱
      const hashedPassword = await bcrypt.hash(password, 10);

      // 3. 사용자 생성
      const user = await this.prisma.user.create({
        data: {
          email,
          password: hashedPassword,
          name,
          tel: tel || null,
        },
      });

      // 4. 응답 DTO로 변환 (비밀번호 제외)
      return this.toUserDto(user);
    }

    private toUserDto(user: any): UserDto {
      return {
        id: user.id,
        email: user.email,
        name: user.name,
        tel: user.tel,
        createdAt: user.createdAt,
      };
    }
  }
  ```

### Step 4: AuthController에 register 엔드포인트 추가
- 대상 파일: `apps/backend/src/modules/auth/auth.controller.ts`
- 작업 내용: 회원가입 API 엔드포인트 구현
  ```typescript
  import { Controller, Post, Body, HttpStatus } from '@nestjs/common';
  import { AuthService } from './auth.service';
  import { CreateUserDto } from './dto/create-user.dto';
  import { UserDto } from './dto/user.dto';

  @Controller('api/v1/auth')
  export class AuthController {
    constructor(private readonly authService: AuthService) {}

    @Post('register')
    async register(@Body() createUserDto: CreateUserDto) {
      const user = await this.authService.register(createUserDto);
      return {
        code: 201000,
        result: {
          user,
        },
      };
    }
  }
  ```

  ```typescript
  import { NestFactory } from '@nestjs/core';
  import { ValidationPipe } from '@nestjs/common';
  import { AppModule } from './app.module';

  async function bootstrap() {
    const app = await NestFactory.create(AppModule);
    
    // 글로벌 ValidationPipe 적용
    app.useGlobalPipes(new ValidationPipe({ whitelist: true }));
    
    await app.listen(3000);
  }
  bootstrap();
  ```

## 참고
- 비밀번호는 평문으로 저장되지 않고 bcrypt로 해싱되어 저장됩니다.
- 응답에는 비밀번호를 절대로 포함시키지 않습니다.
- 이메일은 unique 제약조건을 가져야 합니다 (schema.prisma에서 @unique 지정).
- 추후 회원가입 시 기본 Company, Role 설정을 추가할 수 있습니다.
