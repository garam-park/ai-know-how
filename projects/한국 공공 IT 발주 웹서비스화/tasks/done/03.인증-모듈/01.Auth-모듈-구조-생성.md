# Auth 모듈 구조 생성

## 예상 소요시간
| 구분 | 시간 |
|------|------|
| 사람 (숙련 개발자) | ~15분 |
| AI 에이전트 | ~5분 |

## What (무엇을)
NestJS 모듈 구조를 기반으로 `apps/backend/src/modules/auth/` 디렉토리 구조를 생성하고, AuthModule을 AppModule에 등록합니다. 필요한 의존성 라이브러리도 설치합니다.

## Why (왜)
인증 모듈을 체계적으로 구조화하면 이후 API 구현이 일관되고 유지보수하기 쉬워집니다. NestJS의 모듈 기반 구조를 따라야 전체 애플리케이션의 일관성을 유지할 수 있습니다.

### Step 3: auth.module.ts 파일 생성
- 대상 파일: `apps/backend/src/modules/auth/auth.module.ts`
- 작업 내용: 기본 AuthModule 구현
  ```typescript
  import { Module } from '@nestjs/common';
  import { JwtModule } from '@nestjs/jwt';
  import { PassportModule } from '@nestjs/passport';
  import { AuthController } from './auth.controller';
  import { AuthService } from './auth.service';
  import { PrismaModule } from '../prisma/prisma.module';

  @Module({
    imports: [
      PassportModule,
      JwtModule.register({
        secret: process.env.JWT_SECRET || 'dev-secret-key',
        signOptions: { expiresIn: '15m' },
      }),
      PrismaModule,
    ],
    controllers: [AuthController],
    providers: [AuthService],
    exports: [AuthService],
  })
  export class AuthModule {}
  ```

### Step 7: AppModule에 AuthModule 등록
- 대상 파일: `apps/backend/src/app.module.ts`
- 작업 내용: AuthModule을 imports 배열에 추가
  ```typescript
  import { Module } from '@nestjs/common';
  import { AuthModule } from './modules/auth/auth.module';
  import { PrismaModule } from './modules/prisma/prisma.module';

  @Module({
    imports: [
      PrismaModule,
      AuthModule,
      // 다른 모듈들...
    ],
  })
  export class AppModule {}
  ```

## 참고
- JWT_SECRET은 프로덕션 환경에서는 환경 변수로 관리되어야 합니다.
- 현재는 개발 환경이므로 기본값 'dev-secret-key'를 사용합니다.
- PrismaModule은 이전 단계에서 생성되었다고 가정합니다. 없는 경우 별도로 생성해야 합니다.
