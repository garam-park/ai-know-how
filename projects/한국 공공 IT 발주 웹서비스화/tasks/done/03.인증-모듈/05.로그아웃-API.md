# 로그아웃 API

## 예상 소요시간
| 구분 | 시간 |
|------|------|
| 사람 (숙련 개발자) | ~20분 |
| AI 에이전트 | ~8분 |

## What (무엇을)
POST /api/v1/auth/logout 엔드포인트를 구현합니다. MVP 단계에서는 클라이언트 측에서 토큰을 삭제하는 방식을 채택하며, 추후 Redis 블랙리스트로 고도화합니다.

## Why (왜)
로그아웃 시 클라이언트에서 저장된 토큰을 삭제하여 더 이상 인증된 요청을 하지 못하도록 합니다. 보안 로그 기록 및 추후 서버 측 토큰 무효화 기능을 위한 기초를 마련합니다.

## 수행 가이드

### Step 1: @CurrentUser 데코레이터 임시 활용
- 작업 내용: 로그아웃 시 현재 사용자 정보가 필요하면 @CurrentUser 데코레이터 사용
- 참고: @CurrentUser 데코레이터는 다음 태스크에서 정식으로 구현됩니다.

### Step 2: AuthService에 logout 메서드 추가
- 대상 파일: `apps/backend/src/modules/auth/auth.service.ts`
- 작업 내용: 로그아웃 비즈니스 로직 추가
  ```typescript
  import { Injectable } from '@nestjs/common';
  import { PrismaService } from '../prisma/prisma.service';
  import { JwtService } from '@nestjs/jwt';

  @Injectable()
  export class AuthService {
    constructor(
      private prisma: PrismaService,
      private jwtService: JwtService,
    ) {}

    // ... register, login, refresh 메서드 ...

    async logout(userId: string) {
      // MVP 단계: 클라이언트에서 토큰 삭제
      // 추후 Redis 블랙리스트 기능으로 고도화 가능
      
      // 현재는 로그 기록만 수행 (선택사항)
      console.log(`User ${userId} logged out at ${new Date().toISOString()}`);

      return {
        message: '로그아웃 성공',
      };
    }
  }
  ```

### Step 3: AuthController에 logout 엔드포인트 추가
- 대상 파일: `apps/backend/src/modules/auth/auth.controller.ts`
- 작업 내용: 로그아웃 API 엔드포인트 추가 (현재는 인증 없이 간단히 구현)
  ```typescript
  import { Controller, Post, Body } from '@nestjs/common';
  import { AuthService } from './auth.service';

  @Controller('api/v1/auth')
  export class AuthController {
    constructor(private readonly authService: AuthService) {}

    // ... register, login, refresh 메서드 ...

    @Post('logout')
    async logout() {
      // MVP 단계: userId는 추후 @CurrentUser에서 가져올 예정
      // 현재는 클라이언트가 토큰을 삭제하도록 안내
      const result = await this.authService.logout('');

      return {
        code: 200000,
        result,
      };
    }
  }
  ```

### Step 4: API 테스트
- 작업 내용: 로그아웃 API 테스트
  ```bash
  curl -X POST http://localhost:3000/api/v1/auth/logout \
    -H "Content-Type: application/json"
  ```

  정상 응답:
  ```json
  {
    "code": 200000,
    "result": {
      "message": "로그아웃 성공"
    }
  }
  ```

### Step 5: 클라이언트 측 토큰 삭제 안내
- 작업 내용: 프론트엔드 개발자를 위한 로그아웃 가이드 작성
  
  **클라이언트 측 구현 예시 (React):**
  ```typescript
  // 로그아웃 함수
  const handleLogout = async () => {
    try {
      // 1. 서버에 로그아웃 요청 (선택사항)
      await fetch('/api/v1/auth/logout', {
        method: 'POST',
        headers: {
          'Authorization': `Bearer ${accessToken}`,
        },
      });

      // 2. 클라이언트 측에서 토큰 삭제
      localStorage.removeItem('accessToken');
      localStorage.removeItem('refreshToken');
      
      // 또는 secure/httpOnly 쿠키 삭제
      // 쿠키는 서버에서 Set-Cookie: max-age=0으로 삭제

      // 3. 로그인 페이지로 리다이렉트
      navigate('/login');
    } catch (error) {
      console.error('로그아웃 실패', error);
    }
  };
  ```

- [ ] 응답 구조: {code: 200000, result: {message: "로그아웃 성공"}}
- [ ] POST 요청 시 200 상태코드로 응답
- [ ] (선택) 로그아웃 시 사용자 정보 로깅 추가

## 참고
- MVP 단계에서는 클라이언트 측에서 토큰을 삭제하는 방식을 사용합니다.
- 추후 고도화 단계에서 다음을 추가할 수 있습니다:
  - Redis를 이용한 refreshToken 블랙리스트
  - accessToken도 블랙리스트에 추가
  - 로그아웃 이벤트 저장
  - 여러 디바이스에서 로그인한 경우 선택적 로그아웃
- 클라이언트에서 localStorage에 저장한 토큰은 XSS 공격에 취약하므로, 추후 secure/httpOnly 쿠키로 변경하는 것이 좋습니다.
- 로그아웃 후 accessToken으로 API 요청 시 여전히 성공할 수 있으므로, 클라이언트에서 토큰 존재 여부를 먼저 확인해야 합니다.
