# Prisma & PostgreSQL 연결

## 예상 소요시간
| 구분 | 시간 |
|------|------|
| 사람 (숙련 개발자) | ~30분 |
| AI 에이전트 | ~10분 |

## What (무엇을)
백엔드(`apps/backend`)에 Prisma 6 ORM을 설치하고, PostgreSQL 데이터베이스 연결을 구성하며, 프로젝트 전체 데이터 모델을 정의하는 `schema.prisma` 파일을 작성합니다. 또한 PrismaService와 PrismaModule을 생성하여 NestJS와 통합합니다.

## Why (왜)
Prisma는 타입 안정성이 있는 데이터베이스 ORM으로, 자동 마이그레이션과 타입 생성을 지원합니다. 이를 통해 데이터 모델 변경 시 데이터베이스와 코드 간 일관성을 유지할 수 있습니다. PrismaService는 NestJS 의존성 주입과 생명주기 관리를 제공합니다.

## 수행 가이드

### Step 1: Prisma 및 관련 패키지 설치
```bash
cd project-root/apps/backend
pnpm add @prisma/client
pnpm add -D prisma
```

### Step 2: Prisma 초기화
```bash
cd project-root/apps/backend
npx prisma init
```

이 명령은 `prisma/schema.prisma` 파일과 `.env` 파일을 생성합니다.

### Step 3: 환경변수 설정
- 대상 파일: `project-root/.env` (루트 디렉토리)
- 작업 내용:

```env
# Database
DATABASE_URL="postgresql://user:password@localhost:5432/korean_public_it?schema=public"

# JWT
JWT_SECRET="your-secret-key-change-in-production"
JWT_EXPIRES_IN="1h"
JWT_REFRESH_EXPIRES_IN="7d"

# Server
PORT=3000
NODE_ENV=development
```

**주의:** `.env` 파일은 `.gitignore`에 추가되어야 합니다.

### Step 4: Prisma Schema 파일 작성
- 대상 파일: `project-root/apps/backend/prisma/schema.prisma`
- 전체 내용 (설계 문서의 스키마를 그대로 사용):

```prisma
// This is your Prisma schema file,
// learn more about it in the docs: https://pris.ly/d/prisma-schema

generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}

model User {
  id        String     @id @default(cuid())
  email     String     @unique
  password  String
  name      String
  phone     String?
  createdAt DateTime   @default(now())
  updatedAt DateTime   @updatedAt

  companyId String?
  company   Company?   @relation(fields: [companyId], references: [id])

  projectMembers ProjectMember[]
  projectRolePermissions ProjectRolePermission[]

  @@index([email])
  @@index([companyId])
}

model Company {
  id                    String     @id @default(cuid())
  name                  String
  registrationNumber    String     @unique
  representativeName    String
  representativeEmail   String?
  representativePhone   String?
  address               String
  phone                 String
  createdAt             DateTime   @default(now())
  updatedAt             DateTime   @updatedAt

  users               User[]
  projects            ProjectCompany[]
  projectMembers      ProjectMember[]

  @@index([registrationNumber])
}

model Project {
  id              String     @id @default(cuid())
  name            String
  description     String?
  status          String     @default("PLANNING")
  startDate       DateTime
  endDate         DateTime
  budget          Float?
  createdAt       DateTime   @default(now())
  updatedAt       DateTime   @updatedAt

  companies       ProjectCompany[]
  members         ProjectMember[]
  roles           Role[]
  wbsNodes        WbsNode[]
  rolePermissions ProjectRolePermission[]

  @@index([status])
}

model ProjectCompany {
  id        String     @id @default(cuid())
  projectId String
  companyId String
  role      String     // OWNER, PRIME, PARTNER, SUB
  joinedAt  DateTime   @default(now())

  project   Project    @relation(fields: [projectId], references: [id], onDelete: Cascade)
  company   Company    @relation(fields: [companyId], references: [id], onDelete: Cascade)

  @@unique([projectId, companyId])
  @@index([projectId])
  @@index([companyId])
}

model ProjectMember {
  id        String     @id @default(cuid())
  projectId String
  userId    String
  roleId    String
  joinedAt  DateTime   @default(now())

  project   Project    @relation(fields: [projectId], references: [id], onDelete: Cascade)
  user      User       @relation(fields: [userId], references: [id], onDelete: Cascade)
  role      Role       @relation(fields: [roleId], references: [id])

  @@unique([projectId, userId])
  @@index([projectId])
  @@index([userId])
  @@index([roleId])
}

model Role {
  id          String     @id @default(cuid())
  projectId   String
  name        String
  description String?
  createdAt   DateTime   @default(now())
  updatedAt   DateTime   @updatedAt

  project     Project    @relation(fields: [projectId], references: [id], onDelete: Cascade)
  members     ProjectMember[]
  permissions RolePermission[]
  projectRolePermissions ProjectRolePermission[]

  @@unique([projectId, name])
  @@index([projectId])
}

model Permission {
  id          String     @id @default(cuid())
  resource    String     // WBS, REPORT, TIMESHEET, MEMBER, COMPANY, PROJECT, DELIVERABLE, ISSUE, RISK, CHANGE, AUDIT, CLOSE
  action      String     // VIEW, EDIT, CREATE, DELETE, INVITE, REMOVE, APPROVE, REQUEST, DOWNLOAD, MANAGE, SETTING
  description String?

  roles       RolePermission[]

  @@unique([resource, action])
}

model RolePermission {
  id           String     @id @default(cuid())
  roleId       String
  permissionId String

  role       Role       @relation(fields: [roleId], references: [id], onDelete: Cascade)
  permission Permission @relation(fields: [permissionId], references: [id], onDelete: Cascade)

  @@unique([roleId, permissionId])
  @@index([roleId])
  @@index([permissionId])
}

model ProjectRolePermission {
  id           String     @id @default(cuid())
  projectId    String
  roleId       String
  userId       String
  permissionId String
  scope        String     // ALL, OWN_COMPANY, OWN_TASK, NONE
  createdAt    DateTime   @default(now())
  updatedAt    DateTime   @updatedAt

  project    Project     @relation(fields: [projectId], references: [id], onDelete: Cascade)
  role       Role        @relation(fields: [roleId], references: [id], onDelete: Cascade)
  user       User        @relation(fields: [userId], references: [id], onDelete: Cascade)

  @@unique([projectId, roleId, userId, permissionId])
  @@index([projectId])
  @@index([roleId])
  @@index([userId])
}

model WbsNode {
  id                 String     @id @default(cuid())
  projectId          String
  parentId           String?
  name               String
  type               String     // CATEGORY, TASK
  description        String?
  plannedStartDate   DateTime
  plannedEndDate     DateTime
  actualStartDate    DateTime?
  actualEndDate      DateTime?
  createdAt          DateTime   @default(now())
  updatedAt          DateTime   @updatedAt

  project            Project    @relation(fields: [projectId], references: [id], onDelete: Cascade)
  parent             WbsNode?   @relation("WbsNodeParent", fields: [parentId], references: [id], onDelete: SetNull)
  children           WbsNode[]  @relation("WbsNodeParent")

  @@unique([projectId, name, parentId])
  @@index([projectId])
  @@index([parentId])
}
```

### Step 6: PostgreSQL 데이터베이스 마이그레이션 생성
```bash
cd project-root/apps/backend
npx prisma migrate dev --name init
```

이 명령은:
1. 데이터베이스가 없으면 생성
2. 마이그레이션 파일을 생성
3. 마이그레이션을 적용
4. Prisma Client를 생성

### Step 7: PrismaService 생성
- 대상 파일: `project-root/apps/backend/src/prisma/prisma.service.ts`

```typescript
import { Injectable, OnModuleInit, OnModuleDestroy } from '@nestjs/common';
import { PrismaClient } from '@prisma/client';

@Injectable()
export class PrismaService extends PrismaClient implements OnModuleInit, OnModuleDestroy {
  async onModuleInit() {
    await this.$connect();
  }

  async onModuleDestroy() {
    await this.$disconnect();
  }
}
```

### Step 8: PrismaModule 생성
- 대상 파일: `project-root/apps/backend/src/prisma/prisma.module.ts`

```typescript
import { Module, Global } from '@nestjs/common';
import { PrismaService } from './prisma.service';

@Global()
@Module({
  providers: [PrismaService],
  exports: [PrismaService],
})
export class PrismaModule {}
```

### Step 9: AppModule에 PrismaModule 추가
- 대상 파일: `project-root/apps/backend/src/app.module.ts` 수정

```typescript
import { Module } from '@nestjs/common';
import { ConfigModule } from '@nestjs/config';
import { PrismaModule } from './prisma/prisma.module';
import { AppController } from './app.controller';
import { AppService } from './app.service';

@Module({
  imports: [
    ConfigModule.forRoot({
      isGlobal: true,
      envFilePath: '../../.env',
    }),
    PrismaModule,
  ],
  controllers: [AppController],
  providers: [AppService],
})
export class AppModule {}
```

cd project-root/apps/backend

## 참고
- PostgreSQL이 설치되어 있어야 함 (로컬 또는 Docker)
- Docker로 PostgreSQL 실행: `docker run --name postgres -e POSTGRES_PASSWORD=password -p 5432:5432 -d postgres`
- 마이그레이션은 `prisma/migrations/` 디렉토리에 저장되며 Git으로 관리됨
- 스키마 변경 후 항상 새로운 마이그레이션 생성 필요: `npx prisma migrate dev --name <description>`
- Prisma Studio로 데이터 확인: `npx prisma studio`
