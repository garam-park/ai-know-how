# 인증 시나리오 E2E 테스트

## 예상 소요시간
| 구분 | 시간 |
|------|------|
| 사람 (숙련 개발자) | ~30분 |
| AI 에이전트 | ~10분 |

## What (무없을)
회원가입, 로그인, 토큰 만료/갱신, 잘못된 로그인 등 인증 관련 E2E 시나리오를 작성합니다.

## Why (왜)
가장 기본적인 사용자 플로우를 검증합니다. 인증이 실패하면 전체 애플리케이션이 동작하지 않습니다.

## 선행 작업
- `13.통합테스트-E2E/01.E2E-테스트-환경설정.md`

## 수행 가이드

### Step 1: 회원가입 시나리오 테스트
`e2e/auth.signup.spec.ts` 파일을 생성합니다:

```typescript
import { test, expect } from '@playwright/test';
import { expectToast, waitForLoading } from './utils/helpers';

test.describe('회원가입', () => {
  test('새 사용자가 성공적으로 회원가입할 수 있다', async ({ page }) => {
    // 회원가입 페이지로 이동
    await page.goto('/register');

    // 폼 작성
    const newEmail = `test-${Date.now()}@example.com`;
    await page.fill('input[name="email"]', newEmail);
    await page.fill('input[name="name"]', '테스트 사용자');
    await page.fill('input[name="password"]', 'Test1234!');
    await page.fill('input[name="confirmPassword"]', 'Test1234!');
    await page.fill('input[name="tel"]', '010-1234-5678');

    // 제출
    await page.click('button:has-text("회원가입")');

    // 성공 확인
    await expectToast(page, '회원가입이 완료되었습니다');
    await page.waitForURL('/projects');
    await expect(page.locator('text=내 프로젝트')).toBeVisible();
  });

  test('이미 존재하는 이메일로 가입하면 에러가 발생한다', async ({ page }) => {
    await page.goto('/register');

    // 기존 사용자 이메일로 가입 시도
    await page.fill('input[name="email"]', 'admin@example.com');
    await page.fill('input[name="name"]', '테스트');
    await page.fill('input[name="password"]', 'Test1234!');
    await page.fill('input[name="confirmPassword"]', 'Test1234!');

    await page.click('button:has-text("회원가입")');

    // 에러 메시지 확인
    await expectToast(page, '이메일이 이미 존재합니다');
    await page.waitForURL('/register');
  });

  test('비밀번호 불일치 시 가입할 수 없다', async ({ page }) => {
    await page.goto('/register');

    await page.fill('input[name="email"]', `test-${Date.now()}@example.com`);
    await page.fill('input[name="name"]', '테스트');
    await page.fill('input[name="password"]', 'Test1234!');
    await page.fill('input[name="confirmPassword"]', 'Different1234!');

    // 폼 검증 에러 표시
    const errorMessage = page.locator(
      'text=비밀번호가 일치하지 않습니다'
    );
    await expect(errorMessage).toBeVisible();

    // 제출 버튼이 비활성화되지는 않지만, 제출해도 에러 발생
    const submitButton = page.locator('button:has-text("회원가입")');
    // 클릭 후 에러 메시지 확인
  });

  test('필수 필드 미입력 시 에러 메시지가 표시된다', async ({ page }) => {
    await page.goto('/register');

    // 이메일만 입력하고 제출 시도
    await page.fill('input[name="email"]', `test-${Date.now()}@example.com`);
    await page.click('button:has-text("회원가입")');

    // 필드 검증 에러 확인
    const nameError = page.locator('text=이름은 필수입니다');
    const passwordError = page.locator('text=비밀번호는 필수입니다');
    await expect(nameError).toBeVisible();
    await expect(passwordError).toBeVisible();
  });
});
```

### Step 2: 로그인 시나리오 테스트
`e2e/auth.login.spec.ts` 파일을 생성합니다:

```typescript
import { test, expect } from '@playwright/test';
import { login, expectToast } from './utils/helpers';

test.describe('로그인', () => {
  test('올바른 이메일/비밀번호로 로그인할 수 있다', async ({ page }) => {
    await login(page, 'pm@example.com', 'Test1234!');

    // 프로젝트 목록 페이지로 이동되었는지 확인
    await expect(page).toHaveURL(/\/projects$/);
    await expect(page.locator('text=내 프로젝트')).toBeVisible();
  });

  test('잘못된 비밀번호로 로그인할 수 없다', async ({ page }) => {
    await page.goto('/login');

    await page.fill('input[type="email"]', 'pm@example.com');
    await page.fill('input[type="password"]', 'WrongPassword');
    await page.click('button:has-text("로그인")');

    // 에러 메시지 확인
    await expectToast(page, '이메일 또는 비밀번호가 잘못되었습니다');

    // 여전히 로그인 페이지에 있음
    await expect(page).toHaveURL(/\/login$/);
  });

  test('존재하지 않는 이메일로 로그인할 수 없다', async ({ page }) => {
    await page.goto('/login');

    await page.fill('input[type="email"]', 'nonexistent@example.com');
    await page.fill('input[type="password"]', 'Test1234!');
    await page.click('button:has-text("로그인")');

    // 에러 메시지 확인
    await expectToast(page, '이메일 또는 비밀번호가 잘못되었습니다');
  });

  test('빈 필드로는 로그인할 수 없다', async ({ page }) => {
    await page.goto('/login');

    // 아무것도 입력하지 않고 제출
    await page.click('button:has-text("로그인")');

    // 폼 검증 에러 확인
    const emailError = page.locator('text=이메일은 필수입니다');
    const passwordError = page.locator('text=비밀번호는 필수입니다');
    await expect(emailError).toBeVisible();
    await expect(passwordError).toBeVisible();
  });

  test('로그인 후 회원가입 페이지로 접근할 수 없다', async ({ page }) => {
    await login(page, 'pm@example.com', 'Test1234!');

    // 로그인 상태에서 /register로 접근 시도
    await page.goto('/register');

    // 프로젝트 목록으로 리다이렉트되어야 함 (설정에 따라)
    // 또는 /register에 머물렀더라도 로그인 폼이 아닌 다른 콘텐츠 표시
  });
});
```

### Step 3: 토큰 갱신 시나리오 테스트
`e2e/auth.tokenRefresh.spec.ts` 파일을 생성합니다:

```typescript
import { test, expect } from '@playwright/test';
import { login } from './utils/helpers';

test.describe('토큰 갱신', () => {
  test('세션이 유지되어야 한다', async ({ page }) => {
    // 로그인
    await login(page, 'pm@example.com', 'Test1234!');

    // 프로젝트 목록 페이지에서
    await expect(page).toHaveURL(/\/projects$/);

    // 특정 프로젝트로 이동
    await page.click('text=E2E 테스트 프로젝트');
    await page.waitForURL(/\/projects\/\w+$/);

    // 페이지 새로고침
    await page.reload();

    // 여전히 로그인 상태여야 함
    await expect(page.locator('text=E2E 테스트 프로젝트')).toBeVisible();

    // 다시 프로젝트 목록으로
    await page.click('text=프로젝트 목록');
    await expect(page).toHaveURL(/\/projects$/);
  });

  test('API 요청 후에도 세션이 유지된다', async ({ page }) => {
    await login(page, 'pm@example.com', 'Test1234!');

    // 프로젝트 로드
    await page.waitForSelector('[data-testid="project-card"]');

    // 5분 대기 시뮬레이션 (실제 테스트에서는 시간을 조작)
    // 이 부분은 실제로 5분을 대기하기보다는 모의(mock) 또는
    // 백엔드의 토큰 만료 시간을 짧게 설정하여 테스트

    // 추가 API 요청 (WBS 조회 등)
    await page.click('text=WBS');
    await page.waitForSelector('[data-testid="wbs-table"]');

    // 성공적으로 로드되었으면 토큰 갱신이 작동함
    await expect(page.locator('[data-testid="wbs-table"]')).toBeVisible();
  });

  test('refresh token 만료 시 로그인 페이지로 리다이렉트된다', async ({
    page,
  }) => {
    await login(page, 'pm@example.com', 'Test1234!');

    // 쿠키/로컬스토리지에서 refresh token 제거 (시뮬레이션)
    // 또는 백엔드에서 모든 토큰 무효화

    // API 요청 트리거
    await page.click('text=프로젝트 목록');
    await page.reload();

    // 로그인 페이지로 리다이렉트
    await page.waitForURL('/login');
    await expect(page.locator('text=로그인')).toBeVisible();
  });
});
```

### Step 4: 세션 관리 시나리오 테스트
`e2e/auth.logout.spec.ts` 파일을 생성합니다:

```typescript
import { test, expect } from '@playwright/test';
import { login, logout, expectToast } from './utils/helpers';

test.describe('로그아웃', () => {
  test('로그아웃 후 로그인 페이지로 리다이렉트된다', async ({ page }) => {
    await login(page, 'pm@example.com', 'Test1234!');

    // 로그아웃 버튼 클릭
    await logout(page);

    // 로그인 페이지에 있는지 확인
    await expect(page).toHaveURL('/login');
    await expect(page.locator('text=로그인')).toBeVisible();
  });

  test('로그아웃 후 /projects에 접근할 수 없다', async ({ page }) => {
    await login(page, 'pm@example.com', 'Test1234!');
    await logout(page);

    // 로그인 상태 없이 /projects 접근 시도
    await page.goto('/projects');

    // 로그인 페이지로 리다이렉트되어야 함
    await page.waitForURL('/login');
    await expect(page.locator('text=로그인')).toBeVisible();
  });

  test('로그아웃 후 브라우저 뒤로가기로도 로그인 페이지로 갈 수 없다', async ({
    page,
  }) => {
    await login(page, 'pm@example.com', 'Test1234!');

    // 프로젝트 페이지 방문
    await page.click('text=E2E 테스트 프로젝트');
    await page.waitForURL(/\/projects\/\w+$/);

    // 로그아웃
    await logout(page);

    // 뒤로가기 시도
    await page.goBack();

    // 여전히 로그인 페이지여야 함
    await expect(page).toHaveURL('/login');
  });

  test('동시에 여러 탭에서 로그아웃하면 모두 영향을 받는다', async ({
    browser,
  }) => {
    // 첫 번째 탭에서 로그인
    const page1 = await browser.newPage();
    await login(page1, 'pm@example.com', 'Test1234!');

    // 두 번째 탭 오픈 (같은 세션 공유)
    const page2 = await browser.newPage();
    await page2.goto('/projects');
    // 자동으로 로그인되어 있어야 함
    await expect(page2.locator('text=내 프로젝트')).toBeVisible();

    // 첫 번째 탭에서 로그아웃
    await logout(page1);

    // 두 번째 탭도 영향받아 로그인 페이지로 리다이렉트되어야 함
    await page2.click('text=프로젝트 목록');
    await page2.waitForURL('/login');

    await page1.close();
    await page2.close();
  });
});
```

## 참고
- 각 시나리오는 독립적으로 실행 가능해야 함
- 테스트 데이터는 매 테스트마다 새로 생성되어야 함
- 시간 기반 테스트(토큰 만료)는 시간 모의화 필요
- 에러 메시지는 정확하게 매칭해야 함
- 모든 시나리오에서 페이지 로딩 완료를 기다려야 함
