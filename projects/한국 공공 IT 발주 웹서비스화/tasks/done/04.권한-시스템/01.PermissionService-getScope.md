# PermissionService getScope 구현

## 예상 소요시간
| 구분 | 시간 |
|------|------|
| 사람 (숙련 개발자) | ~40분 |

## What (무엇을)
PermissionService의 getScope(projectId, userId, resource, action) 메서드를 구현하여 주어진 조건에서 사용자가 접근할 수 있는 범위(Scope)를 반환합니다. 이는 모든 API의 데이터 범위 제어의 핵심 함수입니다.

## Why (왜)
API에서 데이터를 조회할 때 사용자의 권한 범위(ALL, OWN_COMPANY, OWN_TASK, NONE)를 정확히 파악해야 쿼리를 올바르게 필터링할 수 있습니다. 이 함수가 모든 권한 체크의 기초가 됩니다.

### Step 2: getScope 메서드 전체 구현
- 대상 파일: `apps/backend/src/modules/permission/permission.service.ts`
- 작업 내용: 우선순위 기반 권한 조회 로직 구현
  ```typescript
  import { Injectable } from '@nestjs/common';
  import { PrismaService } from '../prisma/prisma.service';
  import { Scope, Resource, Action } from '@shared/enums';

  @Injectable()
  export class PermissionService {
    constructor(private prisma: PrismaService) {}

    async getScope(
      projectId: string,
      userId: string,
      resource: Resource,
      action: Action,
    ): Promise<Scope> {
      // 1. ProjectMember에서 사용자의 Role 조회
      const projectMember = await this.prisma.projectMember.findUnique({
        where: {
          userId_projectId: {
            userId,
            projectId,
          },
        },
        select: {
          roleId: true,
        },
      });

      if (!projectMember) {
        // 프로젝트 멤버가 아니면 권한 없음
        return Scope.NONE;
      }

      // 2. Permission ID 조회 (resource + action 조합)
      const permission = await this.prisma.permission.findUnique({
        where: {
          resource_action: {
            resource,
            action,
          },
        },
        select: {
          id: true,
        },
      });

      if (!permission) {
        // 해당 Permission이 없으면 권한 없음
        return Scope.NONE;
      }

      // 3. ProjectRolePermission에서 프로젝트 커스텀 권한 조회 (우선순위 높음)
      const projectRolePermission = await this.prisma.projectRolePermission.findUnique({
        where: {
          projectId_roleId_permissionId: {
            projectId,
            roleId: projectMember.roleId,
            permissionId: permission.id,
          },
        },
        select: {
          scope: true,
        },
      });

      if (projectRolePermission) {
        return projectRolePermission.scope as Scope;
      }

      // 4. RolePermission에서 기본 권한 조회
      const rolePermission = await this.prisma.rolePermission.findUnique({
        where: {
          roleId_permissionId: {
            roleId: projectMember.roleId,
            permissionId: permission.id,
          },
        },
        select: {
          scope: true,
        },
      });

      if (rolePermission) {
        return rolePermission.scope as Scope;
      }

      // 5. 어디에도 없으면 권한 없음
      return Scope.NONE;
    }
  }
  ```

### Step 5: PermissionService 테스트
- 대상 파일: `apps/backend/src/modules/permission/permission.service.spec.ts`
- 작업 내용: 단위 테스트 작성
  ```typescript
  import { Test, TestingModule } from '@nestjs/testing';
  import { PermissionService } from './permission.service';
  import { PrismaService } from '../prisma/prisma.service';
  import { Scope, Resource, Action } from '@shared/enums';

  describe('PermissionService', () => {
    let service: PermissionService;
    let prisma: PrismaService;

    beforeEach(async () => {
      const module: TestingModule = await Test.createTestingModule({
        providers: [
          PermissionService,
          {
            provide: PrismaService,
            useValue: {
              projectMember: {
                findUnique: jest.fn(),
              },
              permission: {
                findUnique: jest.fn(),
              },
              projectRolePermission: {
                findUnique: jest.fn(),
              },
              rolePermission: {
                findUnique: jest.fn(),
              },
            },
          },
        ],
      }).compile();

      service = module.get<PermissionService>(PermissionService);
      prisma = module.get<PrismaService>(PrismaService);
    });

    it('주관사PM이 WBS VIEW → ALL scope 반환', async () => {
      const projectId = 'project-1';
      const userId = 'user-1';

      jest.spyOn(prisma.projectMember, 'findUnique').mockResolvedValue({
        roleId: 1, // 주관사PM
      } as any);

      jest.spyOn(prisma.permission, 'findUnique').mockResolvedValue({
        id: 1,
      } as any);

      jest.spyOn(prisma.projectRolePermission, 'findUnique').mockResolvedValue(null);

      jest.spyOn(prisma.rolePermission, 'findUnique').mockResolvedValue({
        scope: Scope.ALL,
      } as any);

      const scope = await service.getScope(projectId, userId, Resource.WBS, Action.VIEW);
      expect(scope).toBe(Scope.ALL);
    });

    it('프로젝트 커스텀 권한이 기본 권한을 덮어쓰기', async () => {
      const projectId = 'project-1';
      const userId = 'user-1';

      jest.spyOn(prisma.projectMember, 'findUnique').mockResolvedValue({
        roleId: 2, // 참여사PM
      } as any);

      jest.spyOn(prisma.permission, 'findUnique').mockResolvedValue({
        id: 1,
      } as any);

      // ProjectRolePermission에 OWN_TASK로 커스터마이징됨
      jest.spyOn(prisma.projectRolePermission, 'findUnique').mockResolvedValue({
        scope: Scope.OWN_TASK,
      } as any);

      const scope = await service.getScope(projectId, userId, Resource.WBS, Action.VIEW);
      expect(scope).toBe(Scope.OWN_TASK);
    });

    it('프로젝트 멤버가 아니면 NONE 반환', async () => {
      jest.spyOn(prisma.projectMember, 'findUnique').mockResolvedValue(null);

      const scope = await service.getScope('project-1', 'user-1', Resource.WBS, Action.VIEW);
      expect(scope).toBe(Scope.NONE);
    });
  });
  ```

## 참고
- getScope는 순수하게 Scope 값만 반환하며, 권한 없음(NONE)도 예외 발생 대신 NONE을 반환합니다.
- 권한 체크 로직은 이 getScope 값을 활용하여 Guard/Middleware에서 구현됩니다.
- ProjectRolePermission이 없으면 RolePermission을 사용하므로, 기본값이 항상 존재하도록 시드데이터가 준비되어야 합니다.
- 성능 최적화: 향후 캐싱 또는 사전로딩이 필요할 수 있습니다.
