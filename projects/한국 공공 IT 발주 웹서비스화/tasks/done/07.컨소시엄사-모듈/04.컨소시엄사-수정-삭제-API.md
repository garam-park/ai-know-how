# 컨소시엘사 수정 및 삭제 API

## 예상 소요시간
| 구분 | 시간 |
|------|------|
| 사람 (숙련 개발자) | ~20분 |
| AI 에이전트 | ~8분 |

## What (무엇을)
PATCH `/api/v1/projects/:projectId/companies/:projectCompanyId`와 DELETE `/api/v1/projects/:projectId/companies/:projectCompanyId` 엔드포인트를 구현합니다. 컨소시엘사의 역할(role)이나 상위사(parentId)를 변경하고, 컨소시엘사를 삭제합니다.

## Why (왜)
프로젝트 진행 중 협력 구조가 변경될 수 있습니다. 참여사의 역할 변경, 하청사의 상위사 변경, 또는 퇴사 회사 제거 등이 필요합니다. 순환 참조 방지로 트리 구조의 무결성을 보장합니다.

## 수행 가이드

### Step 1: ProjectCompanyService에 update 메서드 구현
- 대상 파일: `apps/backend/src/modules/project-company/project-company.service.ts`
- 작업 내용:
  ```typescript
  async update(
    projectId: string,
    projectCompanyId: string,
    dto: UpdateProjectCompanyDto,
  ): Promise<ProjectCompanyDto> {
    // 대상 ProjectCompany 존재 확인
    const target = await this.prisma.projectCompany.findFirst({
      where: { id: projectCompanyId, projectId, deletedAt: null },
    });

    if (!target) {
      throw new NotFoundException({
        code: 404000,
        message: '컨소시엘사 정보를 찾을 수 없습니다.',
      });
    }

    // parentId 변경 시 순환 참조 방지
    if (dto.parentId !== undefined) {
      // 자기 자신을 parent로 지정 불가
      if (dto.parentId === projectCompanyId) {
        throw new BadRequestException({
          code: 400000,
          message: '자기 자신을 상위 회사로 지정할 수 없습니다.',
        });
      }

      // 자신의 하위 노드를 parent로 지정 불가
      const descendants = await this.findDescendants(projectCompanyId);
      if (dto.parentId && descendants.some((d) => d.id === dto.parentId)) {
        throw new BadRequestException({
          code: 400000,
          message: '하위 노드를 상위 회사로 지정할 수 없습니다.',
        });
      }

      // parent가 같은 프로젝트에 속하는지 확인
      if (dto.parentId) {
        const parent = await this.prisma.projectCompany.findFirst({
          where: { id: dto.parentId, projectId, deletedAt: null },
        });

        if (!parent) {
          throw new BadRequestException({
            code: 400000,
            message: '상위 회사가 같은 프로젝트에 속하지 않습니다.',
          });
        }
      }
    }

    // role 변경 시 PRIME 중복 검사
    if (dto.role === 'PRIME' && target.role !== 'PRIME') {
      const existing = await this.prisma.projectCompany.findFirst({
        where: {
          projectId,
          role: 'PRIME',
          parentId: null,
          deletedAt: null,
          NOT: { id: projectCompanyId },
        },
      });

      if (existing) {
        throw new ConflictException({
          code: 409001,
          message: '주관사(PRIME)는 중복으로 등록할 수 없습니다.',
        });
      }
    }

    const updated = await this.prisma.projectCompany.update({
      where: { id: projectCompanyId },
      data: {
        ...(dto.role && { role: dto.role }),
        ...(dto.parentId !== undefined && { parentId: dto.parentId }),
      },
      include: { company: true },
    });

    return {
      id: updated.id,
      projectId: updated.projectId,
      companyId: updated.companyId,
      role: updated.role,
      parentId: updated.parentId,
      createdAt: updated.createdAt,
      updatedAt: updated.updatedAt,
    };
  }

  // 하위 노드를 재귀적으로 조회
  private async findDescendants(projectCompanyId: string): Promise<any[]> {
    const children = await this.prisma.projectCompany.findMany({
      where: { parentId: projectCompanyId, deletedAt: null },
    });

    let descendants = [...children];
    for (const child of children) {
      descendants = [...descendants, ...(await this.findDescendants(child.id))];
    }

    return descendants;
  }
  ```

### Step 2: ProjectCompanyService에 delete 메서드 구현
- 대상 파일: `apps/backend/src/modules/project-company/project-company.service.ts`

  ```typescript
  async delete(
    projectId: string,
    projectCompanyId: string,
  ): Promise<void> {
    // 대상 ProjectCompany 존재 확인
    const target = await this.prisma.projectCompany.findFirst({
      where: { id: projectCompanyId, projectId, deletedAt: null },
    });

    if (!target) {
      throw new NotFoundException({
        code: 404000,
        message: '컨소시엘사 정보를 찾을 수 없습니다.',
      });
    }

    // 하위 자식이 있는 경우: 함께 삭제 또는 경고
    const children = await this.prisma.projectCompany.findMany({
      where: { parentId: projectCompanyId, deletedAt: null },
    });

    if (children.length > 0) {
      // 선택 1: 함께 삭제 (현재 구현)
      // 선택 2: 에러 반환 (호출자 선택)
      // 여기서는 함께 삭제
    }

    // Transaction: 자신과 모든 하위 노드를 soft delete
    await this.prisma.$transaction(async (tx) => {
      const allDescendants = [target.id, ...(await this.findDescendants(projectCompanyId)).map((d) => d.id)];

      await tx.projectCompany.updateMany({
        where: { id: { in: allDescendants } },
        data: { deletedAt: new Date() },
      });
    });
  }
  ```

### Step 3: ProjectCompanyController에 PATCH/:projectCompanyId 엔드포인트 추가
- 대상 파일: `apps/backend/src/modules/project-company/project-company.controller.ts`
- 작업 내용:
  ```typescript
  import { Patch, Delete, Body, Param, UseGuards } from '@nestjs/common';
  import { PermissionGuard } from '@/auth/guards/permission.guard';
  import { RequirePermission } from '@/common/decorators/require-permission.decorator';

  @Patch(':projectCompanyId')
  @UseGuards(JwtAuthGuard, PermissionGuard)
  @RequirePermission('COMPANY', 'MANAGE')
  async update(
    @Param('projectId') projectId: string,
    @Param('projectCompanyId') projectCompanyId: string,
    @Body() dto: UpdateProjectCompanyDto,
  ): Promise<{
    code: number;
    result: { projectCompany: ProjectCompanyDto };
  }> {
    const projectCompany = await this.projectCompanyService.update(
      projectId,
      projectCompanyId,
      dto,
    );
    return {
      code: 200000,
      result: { projectCompany },
    };
  }
  ```

### Step 4: ProjectCompanyController에 DELETE/:projectCompanyId 엔드포인트 추가

  @UseGuards(JwtAuthGuard, PermissionGuard)
  @RequirePermission('COMPANY', 'MANAGE')
  async delete(
    @Param('projectId') projectId: string,
    @Param('projectCompanyId') projectCompanyId: string,
  ): Promise<{
    code: number;
    message: string;

    };
  }

## 참고
- 순환 참조 방지: findDescendants로 모든 하위 노드 조회하여 검사
- Soft delete: 하위 노드도 함께 삭제하여 고아 노드 방지
- Transaction: 여러 작업을 원자성 있게 처리
