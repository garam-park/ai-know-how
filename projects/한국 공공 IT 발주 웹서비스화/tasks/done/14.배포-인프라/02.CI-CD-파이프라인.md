# CI/CD 파이프라인

## 예상 소요시간
| 구분 | 시간 |
|------|------|
| 사람 (숙련 개발자) | ~40분 |
| AI 에이전트 | ~15분 |

## What (무엇을)
GitHub Actions를 사용한 CI/CD 파이프라인을 구축합니다. PR 생성 시 자동으로 린트/빌드/테스트를 실행하고, main 브랜치 merge 시 배포를 트리거합니다.

## Why (왜)
코드 품질 관리와 자동 배포로 개발 속도를 높이고 휴먼 에러를 줄입니다.

## 수행 가이드

### Step 1: CI 파이프라인 생성
`.github/workflows/ci.yml` 파일을 생성합니다:

```yaml
name: CI

on:
  pull_request:
    branches: [main, develop]
    paths:
      - 'apps/backend/**'
      - 'apps/frontend/**'
      - 'packages/**'
      - '.github/workflows/ci.yml'
  push:
    branches: [main, develop]

concurrency:
  group: ${{ github.ref }}
  cancel-in-progress: true

jobs:
  # 린트 검사
  lint:
    name: Lint
    runs-on: ubuntu-latest
    strategy:
      matrix:
        package: [backend, frontend]

    steps:
      - uses: actions/checkout@v3

      - name: Setup Node
        uses: actions/setup-node@v3
        with:
          node-version: '18'
          cache: 'npm'

      - name: Install dependencies
        run: npm ci

      - name: Run ESLint (${{ matrix.package }})
        run: npm run lint --workspace=apps/${{ matrix.package }}

  # 유닛 테스트
  unit-test:
    name: Unit Tests
    runs-on: ubuntu-latest
    needs: lint

    steps:
      - uses: actions/checkout@v3

      - name: Setup Node
        uses: actions/setup-node@v3
        with:
          node-version: '18'
          cache: 'npm'

      - name: Install dependencies
        run: npm ci

      - name: Run unit tests
        run: npm run test:unit

      - name: Upload coverage
        uses: codecov/codecov-action@v3
        with:
          flags: unittests
          fail_ci_if_error: false

  # 빌드 검사
  build:
    name: Build
    runs-on: ubuntu-latest
    needs: lint

    strategy:
      matrix:
        package: [backend, frontend]

    steps:
      - uses: actions/checkout@v3

      - name: Setup Node
        uses: actions/setup-node@v3
        with:
          node-version: '18'
          cache: 'npm'

      - name: Install dependencies
        run: npm ci

      - name: Build (${{ matrix.package }})
        run: npm run build --workspace=apps/${{ matrix.package }}

      - name: Upload build artifacts
        uses: actions/upload-artifact@v3
        with:
          name: build-${{ matrix.package }}
          path: apps/${{ matrix.package }}/dist
          retention-days: 1

  # E2E 테스트 (선택)
  e2e-test:
    name: E2E Tests
    runs-on: ubuntu-latest
    needs: build
    if: github.event_name == 'pull_request'

    services:
      postgres:
        image: postgres:15-alpine
        env:
          POSTGRES_USER: test_user
          POSTGRES_PASSWORD: test_password
          POSTGRES_DB: test_db
        options: >-
          --health-cmd pg_isready
          --health-interval 10s
          --health-timeout 5s
          --health-retries 5
        ports:
          - 5432:5432

    steps:
      - uses: actions/checkout@v3

      - name: Setup Node
        uses: actions/setup-node@v3
        with:
          node-version: '18'
          cache: 'npm'

      - name: Install dependencies
        run: npm ci

      - name: Install Playwright browsers
        run: npx playwright install --with-deps

      - name: Setup test database
        run: |
          npx prisma migrate deploy
          npm run seed:test
        env:
          DATABASE_URL: postgresql://test_user:test_password@localhost:5432/test_db

      - name: Start backend
        run: |
          npm run dev --workspace=apps/backend &
          npx wait-on http://localhost:3001/health
        env:
          DATABASE_URL: postgresql://test_user:test_password@localhost:5432/test_db
          NODE_ENV: test

      - name: Run E2E tests
        run: npm run test:e2e
        env:
          NODE_ENV: test
          DATABASE_URL: postgresql://test_user:test_password@localhost:5432/test_db

      - name: Upload Playwright report
        if: always()
        uses: actions/upload-artifact@v3
        with:
          name: playwright-report
          path: playwright-report/
          retention-days: 30

  # 보안 검사 (선택)
  security:
    name: Security Scan
    runs-on: ubuntu-latest

    steps:
      - uses: actions/checkout@v3

      - name: Run Snyk security scan
        uses: snyk/actions/node@master
        env:
          SNYK_TOKEN: ${{ secrets.SNYK_TOKEN }}
        with:
          args: --severity-threshold=high

  # 결과 종합
  ci-result:
    name: CI Result
    runs-on: ubuntu-latest
    needs: [lint, unit-test, build]
    if: always()

    steps:
      - name: Check CI status
        run: |
          if [[ "${{ needs.lint.result }}" != "success" ]] || \
             [[ "${{ needs.unit-test.result }}" != "success" ]] || \
             [[ "${{ needs.build.result }}" != "success" ]]; then
            echo "CI failed"
            exit 1
          fi
          echo "CI passed"
```

### Step 2: CD 파이프라인 생성
`.github/workflows/cd.yml` 파일을 생성합니다:

```yaml
name: CD

on:
  push:
    branches: [main]

jobs:
  # 배포 전 검사
  pre-deploy-checks:
    name: Pre-Deploy Checks
    runs-on: ubuntu-latest

    steps:
      - uses: actions/checkout@v3

      - name: Setup Node
        uses: actions/setup-node@v3
        with:
          node-version: '18'
          cache: 'npm'

      - name: Install dependencies
        run: npm ci

      - name: Run linter
        run: npm run lint

      - name: Run tests
        run: npm run test:unit

  # 이미지 빌드 및 푸시
  build-and-push:
    name: Build and Push Docker Images
    runs-on: ubuntu-latest
    needs: pre-deploy-checks
    if: success()

    outputs:
      backend-image: ${{ steps.meta-backend.outputs.tags }}
      frontend-image: ${{ steps.meta-frontend.outputs.tags }}

    steps:
      - uses: actions/checkout@v3

      - name: Set up Docker Buildx
        uses: docker/setup-buildx-action@v2

      - name: Login to Docker Hub
        uses: docker/login-action@v2
        with:
          username: ${{ secrets.DOCKER_USERNAME }}
          password: ${{ secrets.DOCKER_PASSWORD }}

      # Backend 이미지 빌드
      - name: Build and push Backend image
        uses: docker/build-push-action@v4
        with:
          context: ./apps/backend
          push: true
          tags: |
            ${{ secrets.DOCKER_USERNAME }}/app-backend:latest
            ${{ secrets.DOCKER_USERNAME }}/app-backend:${{ github.sha }}
          cache-from: type=registry,ref=${{ secrets.DOCKER_USERNAME }}/app-backend:buildcache
          cache-to: type=registry,ref=${{ secrets.DOCKER_USERNAME }}/app-backend:buildcache,mode=max

      # Frontend 이미지 빌드
      - name: Build and push Frontend image
        uses: docker/build-push-action@v4
        with:
          context: ./apps/frontend
          push: true
          tags: |
            ${{ secrets.DOCKER_USERNAME }}/app-frontend:latest
            ${{ secrets.DOCKER_USERNAME }}/app-frontend:${{ github.sha }}
          cache-from: type=registry,ref=${{ secrets.DOCKER_USERNAME }}/app-frontend:buildcache
          cache-to: type=registry,ref=${{ secrets.DOCKER_USERNAME }}/app-frontend:buildcache,mode=max

  # 스테이징 배포
  deploy-staging:
    name: Deploy to Staging
    runs-on: ubuntu-latest
    needs: build-and-push
    environment:
      name: staging
      url: https://staging.example.com

    steps:
      - uses: actions/checkout@v3

      - name: Deploy to Staging Server
        uses: appleboy/ssh-action@master
        with:
          host: ${{ secrets.STAGING_HOST }}
          username: ${{ secrets.STAGING_USERNAME }}
          key: ${{ secrets.STAGING_SSH_KEY }}
          port: 22
          script: |
            cd /app
            docker pull ${{ secrets.DOCKER_USERNAME }}/app-backend:${{ github.sha }}
            docker pull ${{ secrets.DOCKER_USERNAME }}/app-frontend:${{ github.sha }}

            # docker-compose 업데이트 및 재시작
            docker-compose \
              -f docker-compose.yml \
              -e BACKEND_IMAGE=${{ secrets.DOCKER_USERNAME }}/app-backend:${{ github.sha }} \
              -e FRONTEND_IMAGE=${{ secrets.DOCKER_USERNAME }}/app-frontend:${{ github.sha }} \
              up -d

            # 마이그레이션 실행
            docker-compose exec -T backend npx prisma migrate deploy

      - name: Health Check
        run: |
          for i in {1..30}; do
            if curl -f https://staging.example.com/health; then
              echo "Health check passed"
              exit 0
            fi
            sleep 10
          done
          echo "Health check failed"
          exit 1

      - name: Smoke Tests
        run: |
          npx playwright test e2e/smoke.spec.ts
        env:
          FRONTEND_URL: https://staging.example.com

  # 프로덕션 배포 (수동 승인)
  deploy-production:
    name: Deploy to Production
    runs-on: ubuntu-latest
    needs: deploy-staging
    environment:
      name: production
      url: https://example.com

    steps:
      - uses: actions/checkout@v3

      - name: Deploy to Production Server
        uses: appleboy/ssh-action@master
        with:
          host: ${{ secrets.PROD_HOST }}
          username: ${{ secrets.PROD_USERNAME }}
          key: ${{ secrets.PROD_SSH_KEY }}
          port: 22
          script: |
            cd /app
            docker pull ${{ secrets.DOCKER_USERNAME }}/app-backend:${{ github.sha }}
            docker pull ${{ secrets.DOCKER_USERNAME }}/app-frontend:${{ github.sha }}

            # 백업 생성
            docker-compose exec -T postgres pg_dump -U app_user app_db > backup_$(date +%Y%m%d_%H%M%S).sql

            # docker-compose 업데이트 및 재시작
            docker-compose \
              -f docker-compose.yml \
              -e BACKEND_IMAGE=${{ secrets.DOCKER_USERNAME }}/app-backend:${{ github.sha }} \
              -e FRONTEND_IMAGE=${{ secrets.DOCKER_USERNAME }}/app-frontend:${{ github.sha }} \
              up -d

            # 마이그레이션 실행
            docker-compose exec -T backend npx prisma migrate deploy

      - name: Health Check
        run: |
          for i in {1..30}; do
            if curl -f https://example.com/health; then
              echo "Health check passed"
              exit 0
            fi
            sleep 10
          done
          echo "Health check failed"
          exit 1

      - name: Notify Deployment
        if: success()
        uses: 8398a7/action-slack@v3
        with:
          status: custom
          custom_payload: |
            {
              text: `🚀 Production deployment successful!`,
              attachments: [{
                color: 'good',
                text: `Version: ${{ github.sha }}\nURL: https://example.com`
              }]
            }
          webhook_url: ${{ secrets.SLACK_WEBHOOK }}

      - name: Notify Failure
        if: failure()
        uses: 8398a7/action-slack@v3
        with:
          status: custom
          custom_payload: |
            {
              text: `❌ Production deployment failed!`,
              attachments: [{
                color: 'danger',
                text: `Version: ${{ github.sha }}`
              }]
            }
          webhook_url: ${{ secrets.SLACK_WEBHOOK }}

  # 롤백 (선택)
  rollback:
    name: Rollback
    runs-on: ubuntu-latest
    needs: deploy-production
    if: failure()
    environment:
      name: production

    steps:
      - uses: actions/checkout@v3

      - name: Rollback to Previous Version
        uses: appleboy/ssh-action@master
        with:
          host: ${{ secrets.PROD_HOST }}
          username: ${{ secrets.PROD_USERNAME }}
          key: ${{ secrets.PROD_SSH_KEY }}
          port: 22
          script: |
            cd /app
            # 이전 버전으로 되돌리기 (docker-compose의 이전 .env 사용)
            docker-compose down
            git revert HEAD --no-edit
            docker-compose up -d
            echo "Rollback completed"

      - name: Notify Rollback
        uses: 8398a7/action-slack@v3
        with:
          status: custom
          custom_payload: |
            {
              text: `🔄 Production deployment rolled back`,
              attachments: [{
                color: 'warning'
              }]
            }
          webhook_url: ${{ secrets.SLACK_WEBHOOK }}
```

### Step 3: GitHub Secrets 설정
Repository Settings → Secrets and variables → Actions에서 다음 항목 추가:

```
DOCKER_USERNAME: your-docker-username
DOCKER_PASSWORD: your-docker-password
STAGING_HOST: staging.example.com
STAGING_USERNAME: deploy_user
STAGING_SSH_KEY: (private key)
PROD_HOST: production.example.com
PROD_USERNAME: deploy_user
PROD_SSH_KEY: (private key)
SLACK_WEBHOOK: https://hooks.slack.com/services/...
SNYK_TOKEN: (snyk token)
```

### Step 4: 필수 체크 설정
Repository Settings → Branches → main 브랜치 보호 규칙:
- "Require a pull request before merging" 체크
- "Require status checks to pass before merging" 체크
- 필수 체크: lint, unit-test, build 선택
