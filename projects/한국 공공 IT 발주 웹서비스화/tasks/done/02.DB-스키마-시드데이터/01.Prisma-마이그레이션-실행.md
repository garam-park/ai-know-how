# Prisma 마이그레이션 실행

## 예상 소요시간
| 구분 | 시간 |
|------|------|
| 사람 (숙련 개발자) | ~15분 |
| AI 에이전트 | ~5분 |

## What (무엇을)
schema.prisma에 정의된 스키마를 기반으로 PostgreSQL에 첫 마이그레이션을 실행하여 10개 테이블과 enum 타입들을 데이터베이스에 생성하는 작업입니다.

## Why (왜)
모든 백엔드 API와 서비스가 데이터베이스 스키마에 의존하고 있으므로, 마이그레이션 실행을 통해 데이터베이스 구조를 확정하는 것이 필수입니다. 이후 모든 개발 작업이 이 스키마 위에서 진행됩니다.

## 선행 작업
- `01.모노레포-프로젝트-세팅/05.Prisma-PostgreSQL-연결.md`

## 수행 가이드

### Step 1: PostgreSQL 서버 실행 확인
- PostgreSQL 데이터베이스 서버가 실행 중인지 확인합니다.
- 터미널에서 PostgreSQL 상태 확인:
  ```bash
  sudo systemctl status postgresql
  # 또는
  pg_isready -h localhost
  ```

### Step 2: 새 데이터베이스 생성
- 대상 파일: `apps/backend/.env.local` (DATABASE_URL 기준)
- 작업 내용: PostgreSQL에 새 데이터베이스 생성
  ```bash
  createdb -U postgres public_it_service_db
  ```
  또는 psql로 접속하여 생성:
  ```bash
  psql -U postgres
  # psql 프롬프트에서
  CREATE DATABASE public_it_service_db;
  \q
  ```

### Step 3: Prisma 마이그레이션 실행
- 대상 파일: `apps/backend/prisma/schema.prisma`
- 작업 내용: prisma migrate dev 명령으로 초기 마이그레이션 실행
  ```bash
  cd /sessions/fervent-charming-hopper/mnt/한국\ 공공\ IT\ 발주\ 웹서비스화
  pnpm exec prisma migrate dev --name init
  ```
  또는 apps/backend 디렉토리에서:
  ```bash
  cd apps/backend
  npx prisma migrate dev --name init
  ```
  명령어 실행 후 migration 파일이 `apps/backend/prisma/migrations/` 디렉토리에 생성됩니다.

### Step 4: 생성된 테이블 확인
- 작업 내용: PostgreSQL에서 실제 테이블이 생성되었는지 확인
  ```bash
  psql -U postgres -d public_it_service_db -c "\dt"
  ```
  다음 10개 테이블이 생성되어야 합니다:
  - User
  - Company
  - Project
  - ProjectCompany
  - ProjectMember
  - Role
  - Permission
  - RolePermission
  - ProjectRolePermission
  - WbsNode

### Step 5: Enum 타입 확인
- 작업 내용: PostgreSQL에서 생성된 enum 타입 확인
  ```bash
  psql -U postgres -d public_it_service_db -c "\dT"
  ```
  다음 enum 타입들이 생성되어야 합니다:
  - "Resource" (12개 값)
  - "Action" (11개 값)
  - "Scope" (4개 값: ALL, OWN_COMPANY, OWN_TASK, NONE)
  - "CompanyRole" (4개 값: OWNER, ADMIN, MEMBER, VIEWER)
  - "WbsType" (2개 값: PHASE, TASK)

### Step 6: 마이그레이션 상태 확인
- 작업 내용: 마이그레이션 상태 정상 확인
  ```bash
  cd apps/backend
  npx prisma migrate status
  ```

## 완료 기준
- [ ] PostgreSQL 서버가 정상 실행 중
- [ ] 데이터베이스 `public_it_service_db` 생성 완료
- [ ] `npx prisma migrate dev --name init` 명령 성공 실행
- [ ] `npx prisma migrate status` 에서 "Database schema is up to date" 메시지 확인
- [ ] PostgreSQL에서 10개 테이블 모두 생성 확인
- [ ] 5개 enum 타입 생성 확인
- [ ] `apps/backend/prisma/migrations/` 디렉토리에 타임스탬프가 포함된 마이그레이션 폴더 생성됨

## 참고
- Prisma 마이그레이션은 `schema.prisma`의 변경사항을 추적하므로, 향후 스키마 수정 시 동일 과정으로 마이그레이션을 실행합니다.
- 로컬 개발 환경에서는 `prisma migrate dev`를 사용하고, 프로덕션 환경에서는 `prisma migrate deploy`를 사용합니다.
- 마이그레이션 파일은 버전 제어(git)에 포함되어 팀원이 동일한 스키마를 유지합니다.
