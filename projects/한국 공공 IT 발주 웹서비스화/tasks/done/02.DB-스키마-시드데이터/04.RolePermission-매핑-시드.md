# RolePermission 매핑 시드

## 예상 소요시간
| 구분 | 시간 |
|------|------|
| 사람 (숙련 개발자) | ~30분 |
| AI 에이전트 | ~10분 |

## What (무엇을)
RolePermission 테이블에 Role별 기본 Permission과 Scope 매핑을 삽입합니다. MVP 단계에서 멤버 초대 시 Role 선택만으로 권한이 자동 부여되는 핵심 로직입니다.

## Why (왜)
각 Role이 어떤 Permission을 가지고, 어떤 Scope(범위)로 접근할 수 있는지 정의하는 것이 권한 제어의 기반입니다. 이를 통해 멤버 초대 시 Role 선택만으로 권한 설정이 완료됩니다.

### Step 4: RolePermission 매핑 확인
- 작업 내용: 각 Role별 Permission 매핑 수 확인
  ```bash
  psql -U postgres -d public_it_service_db
  # psql 프롬프트에서
  SELECT r.id, r.name, COUNT(rp.id) as permission_count
  FROM "Role" r
  LEFT JOIN "RolePermission" rp ON r.id = rp."roleId"
  GROUP BY r.id, r.name
  ORDER BY r.id;
  ```
  결과:
  ```

    2 | 참여사PM |                9

  (4 rows)
  ```

### Step 5: 구체적 권한 확인
- 작업 내용: 특정 Role의 권한 상세 확인
  ```bash
  psql -U postgres -d public_it_service_db
  # 참여사PM의 권한 조회
  SELECT p.resource, p.action, rp.scope
  FROM "RolePermission" rp
  JOIN "Role" r ON rp."roleId" = r.id
  JOIN "Permission" p ON rp."permissionId" = p.id
  WHERE r.id = 2
  ORDER BY p.resource, p.action;
  ```

## 참고
- RolePermission 매핑은 기본값이며, ProjectRolePermission으로 프로젝트 단위 커스터마이징 가능합니다.
- Scope 값: ALL(전체), OWN_COMPANY(자사), OWN_TASK(할당된 작업), NONE(거부)
- MVP 단계에서는 Role별 기본 매핑만 사용하며, 추후 프로젝트별 권한 커스터마이징 기능을 추가합니다.
