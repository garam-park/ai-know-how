# 04-D-17 도메인 연결 & 자동 SSL 확인

> 유형: 실습 | 예상 소요 시간: 15~20분
> 이전: Caddyfile 작성 (Reverse Proxy 설정) | 다음: Studio 초기 접근 & 관리자 설정

---

## 이 파일에서 하는 것

Cloudflare DNS에서 도메인을 서버 IP에 연결하고, Caddy가 자동으로 SSL 인증서를 발급받아 HTTPS가 정상 작동하는지 확인합니다. Vercel에서 커스텀 도메인을 추가하는 과정과 유사합니다.

---

## 사전 조건

- Cloudflare에 도메인이 등록되어 있음
- Cloudflare 계정에 접근 가능
- 서버의 공인 IP 주소를 알고 있음
- Caddy가 설치되어 있고 Caddyfile이 작성됨 (04-D-16 완료)
- 터미널에서 `curl`, `dig`, `nslookup`, `openssl` 명령어 사용 가능

---

## 실습

### Step 1: Cloudflare에서 A 레코드 추가

**목표**: 도메인을 서버 공인 IP에 연결

1. Cloudflare 대시보드 → 해당 도메인 선택
2. 좌측 메뉴에서 **DNS** 클릭
3. **레코드 추가** 버튼 클릭
4. 다음과 같이 입력:
   - **유형**: A
   - **이름**: `api` (예: api.your-domain.com 사용 시)
   - **IPv4 주소**: 서버의 공인 IP (예: `203.0.113.42`)
   - **프록시 상태**: DNS only (중요! ⭐)
   - **TTL**: Auto

5. **저장** 클릭

> **FE 비유**: Vercel의 "도메인 추가" 기능처럼, 도메인과 서버를 연결하는 DNS 설정입니다.

> **⚠️ 중요**: **프록시 상태는 반드시 "DNS only"로 설정하세요**. Cloudflare의 프록시(주황색 클라우드)를 활성화하면 Caddy의 자동 SSL 인증서 발급이 실패할 수 있습니다.

---

### Step 2: DNS 전파 확인

**목표**: A 레코드가 전 세계 DNS 서버에 반영되었는지 확인

DNS 전파는 최대 48시간 걸리지만, 보통 5~30분 내에 완료됩니다.

#### 방법 1: dig 명령어 사용 (Linux/Mac)

```bash
dig api.your-domain.com
```

**예상 출력**:
```
api.your-domain.com.    300    IN    A    203.0.113.42
```

#### 방법 2: nslookup 명령어 사용 (Windows/Linux/Mac)

```bash
nslookup api.your-domain.com
```

**예상 출력**:
```
Name:    api.your-domain.com
Address: 203.0.113.42
```

#### 방법 3: Cloudflare 대시보드에서 확인

DNS 설정 화면에서 A 레코드 옆에 파란색 체크 아이콘이 표시되면 전파 완료

> **완료 기준**: 모든 조회에서 올바른 IP 주소가 반환되어야 합니다.

---

### Step 3: Caddy 재시작

**목표**: Caddy가 도메인으로 자동 SSL 인증서를 발급받도록 재시작

```bash
sudo systemctl restart caddy
```

또는 Docker 사용 시:

```bash
docker restart caddy-container
```

Caddy 로그 확인:

```bash
sudo journalctl -u caddy -f
```

또는 Docker 로그:

```bash
docker logs -f caddy-container
```

**예상 로그 내용**:
```
{
  "level": "info",
  "ts": 1743638400.5,
  "msg": "autosaved config",
  "encoding": "json"
}
[notice] policy violated; refusing to issue certificate for api.your-domain.com
```

또는 성공 시:

```
[info] Issued certificate [...]
```

> **완료 기준**: 에러가 없거나, Let's Encrypt 인증서 발급 관련 메시지가 표시되어야 합니다.

---

### Step 4: HTTPS 접속 테스트

**목표**: SSL이 정상 작동하는지 확인

#### 방법 1: curl로 응답 코드 확인

```bash
curl -I https://api.your-domain.com
```

**예상 출력** (HTTP 200 또는 307):
```
HTTP/2 200
server: Caddy
content-length: 0
date: Wed, 01 Apr 2026 10:30:15 GMT
```

#### 방법 2: 브라우저에서 확인

1. 브라우저 주소창에 `https://api.your-domain.com` 입력
2. **주소 표시줄의 자물쇠 아이콘**이 초록색으로 표시되어야 함
3. 자물쇠 클릭 → "인증서" 또는 "Connection secure" 표시

> **완료 기준**: 자물쇠 아이콘이 초록색이고, 보안 경고가 없어야 합니다.

---

### Step 5: SSL 인증서 정보 확인

**목표**: 발급된 인증서의 세부 정보 확인 (선택사항, 디버깅용)

```bash
openssl s_client -connect api.your-domain.com:443 -servername api.your-domain.com 2>/dev/null | grep -A 5 "Issuer\|Subject\|Not Before\|Not After"
```

**예상 출력**:
```
subject=CN = api.your-domain.com
issuer=C = US, O = Let's Encrypt, CN = R3
...
Not Before: Apr  1 10:00:00 2026 GMT
Not After : Jun 30 10:00:00 2026 GMT
```

> **확인 항목**:
> - **Issuer**: Let's Encrypt 또는 다른 CA여야 함
> - **CN (Common Name)**: api.your-domain.com이어야 함
> - **Valid Period**: 현재 날짜가 "Not Before"와 "Not After" 사이에 있어야 함

---

### Step 6: Cloudflare 프록시 모드 vs DNS Only 이해하기

**이 부분은 필독하세요**. 많은 문제가 여기서 발생합니다.

#### DNS Only (현재 설정)

```
도메인 → Cloudflare DNS 서버 → 실제 서버(당신의 IP)
      ↓
DNS 조회만 수행 (IP 주소만 반환)
```

- **장점**:
  - Caddy가 자동으로 Let's Encrypt 인증서 발급 가능
  - 설정이 간단함
- **단점**:
  - Cloudflare의 DDoS 방어, 캐싱 등 혜택 없음
  - SSL 인증서는 당신의 서버에만 설치됨

#### Cloudflare 프록시 모드 (주황색 클라우드)

```
도메인 → Cloudflare의 서버 → 실제 서버(당신의 IP)
      ↓
Cloudflare이 모든 요청을 받음 (자신의 인증서 제공)
```

- **장점**:
  - Cloudflare의 보안 기능 활성화
  - 캐싱으로 성능 향상
- **단점**:
  - Caddy의 자동 SSL 발급 실패 가능
  - 복잡한 SSL 체인 설정 필요

> **결론**: 이 단계에서는 **DNS Only**를 사용하세요. 이후 필요하면 프록시 모드로 변경할 수 있습니다.

---

## 확인 방법

다음 체크리스트를 모두 확인하세요:

- [ ] Cloudflare DNS 레코드 추가됨 (A 레코드, DNS only)
- [ ] `dig` 또는 `nslookup`에서 올바른 IP 반환됨
- [ ] `curl -I https://api.your-domain.com`에서 HTTP 200 응답
- [ ] 브라우저 자물쇠 아이콘이 초록색 표시됨
- [ ] Caddy 로그에 에러가 없음 (또는 정상적인 인증서 발급 메시지 있음)

**모두 확인되면 완료입니다!**

## 자주 겪는 오류

| 오류 메시지 | 원인 | 해결 |
|-----------|------|------|
| `curl: (6) Could not resolve host` | DNS가 아직 전파되지 않음 | 5~30분 대기 후 재시도. `dig` 명령어로 전파 상태 확인 |
| `curl: (60) SSL certificate problem` | SSL 인증서가 유효하지 않음 | Caddy 로그 확인. Let's Encrypt rate limit 도달했을 수 있음 |
| `curl: (35) error:14094410` | 자동 SSL 발급 실패 | Cloudflare 프록시 모드 활성화 여부 확인. DNS only로 변경하세요 |
| 브라우저에서 "net::ERR_CERT_AUTHORITY_INVALID" | 자체 서명 인증서 또는 인증서 미발급 | Caddy 재시작 대기. `curl -k` 테스트로 일단 연결 확인 |
| Caddy 로그: "policy violated; refusing to issue" | Let's Encrypt rate limit (주로 테스트 중복) | 다른 도메인 사용 또는 24시간 대기. staging 환경에서 테스트 |
| DNS 조회는 되지만 HTTPS 접속 실패 | 방화벽에서 443 포트 차단 | 서버 방화벽 설정 확인: `sudo ufw allow 443/tcp` |
| `connection refused` | Caddy가 실행 중이 아님 | `sudo systemctl status caddy` 확인. `sudo systemctl start caddy` |
